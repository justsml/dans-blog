## लड़ाई...?  

### प्रमिसेज बनाम एसिंक/अवेट!  

रुको, यह एक लड़ाई है? यह निश्चित रूप से है ना? हम अब कॉलबैक्स के बारे में बात करते नहीं दिखते हैं?

नहीं, यह एक लड़ाई नहीं है। अंततः यह आपके उपकरणों के टूलबॉक्स में एक अन्य संभावित उपकरण है। हालांकि, क्योंकि `async`/`await` सभी प्रमिस फ़ंक्शनलिटी को नहीं बदलता (विशेष रूप से `Promise.all`, `.race`) **इसे प्रतिस्थापन के रूप में प्रस्तुत करना भ्रामक है।**

अनेक प्रभावशाली लोग इस भ्रम को बढ़ावा दे रहे हैं कि `async`/`await` प्रमिसेज का [प्रतिस्थापन](https://developers.google.com/web/fundamentals/primers/async-functions) [सभी के](https://hackernoon.com/6-reasons-why-javascripts-async-await-blows-promises-away-tutorial-c7ec10518dd9) [लिए](https://x.com/umaar/status/1045655069478334464) [इंतजार](http://2ality.com/2017/08/promise-try.html#why-not-just-use-async-functions) [किया](https://dzone.com/articles/javascript-promises-and-why-asyncawait-wins-the-ba) [जा रहा है।](https://dzone.com/articles/javascript-promises-and-why-asyncawait-wins-the-ba)

> **संकेत: नहीं, नहीं, और ना थोड़ा भी।**

मैंने पहले Promises को नापसंद किया था। अब, मैं पूरी तरह से उनके प्रशंसक हो गया हूँ। **Promises शानदार हैं।** वे आपको **फ़ंक्शन संरचना का लाभ उठाने** में सक्षम/प्रोत्साहित कर सकते हैं।

मैंने अपने Promise तकनीक को आगे बढ़ाने के लिए दो क्षेत्रों पर ध्यान देने की सिफारिश की है:

1. [नामित फ़ंक्शन (अनाम नहीं)](#rule-1)
1. [एकल-उद्देश्य वाले फ़ंक्शन](#rule-2)

<h2 id="rule-1">#1: नामित फ़ंक्शन!</h2>

अपने अनाम विधियों को मार डालो। **नामित फ़ंक्शन** का उपयोग करने से कोड आपकी आवश्यकताओं की कविता की तरह पढ़ता है।

चलो एक सामान्य उदाहरण पर नज़र डालते हैं:

एचटीटीपी जीईटी अनुरोध बनाना `fetch` का उपयोग करके:

### अंतर्निहित पैटर्न

```js
// ❌ अनाम अंतर्निहित फ़ंक्शन का उपयोग 💩
fetch(url)
  .then(response => response.status < 400
    ? response
    : Promise.reject(new Error('Request Failed: ' + response.ststus)))
  .then(response => response.text())
```

### समाधान: नामित विधियाँ

```js
// ✅ स्पष्टता उभरती है: नामित फ़ंक्शन
fetch(url)
  .then(checkResponse)
  .then(getText)


// पुनः उपयोग योग्य सामान्य-उद्देश्य वाले फ़ंक्शन
function checkResponse(response) {
  return response.status < 400
    ? response
    : Promise.reject(new Error('Request Failed: ' + response.ststus))
}
function getText(response) {
  return response.text()
}
```

> इस दृष्टिकोण के लाभ आपको डीआरवाई-र कोड प्राप्त करने के साथ बढ़ते हैं।

**अतिरिक्त संसाधन:** मेरे **1 मिनट के वीडियो** देखें [मूल लॉगिंग](https://youtu.be/xR_MZE1SIkk) और [उन्नत डीबगिंग](https://youtu.be/P_tghqWj72M) इस तकनीक का उपयोग करके।

<h2 id="rule-2">#2: एकल उद्देश्य (फ़ंक्शन)</h2>

यह _गुमराहकर तरीके से सटीक_ लगता है: एकल उद्देश्य।

लेकिन यह इतना व्यक्तिगत, आकस्मिक, और निश्चित रूप से, कभी-कभी अर्थहीन होता है।

अजीब तरह से, अधिकांश विकासकर्ता बताते हैं कि वे **एकल उद्देश्य** वाले कोड के लिए _काफी अच्छे_ हैं। असंबंधित नहीं है: वे अपने ड्राइविंग कौशल के बारे में भी अच्छे बताते हैं!

<!-- यह **Promises के साथ एक अद्वितीय समस्या नहीं है**, सरणी विधियाँ और सभी अन्य HoF-आधारित (उच्च क्रम फ़ंक्शन) API के समान अर्थव्यवस्था है। -->

चलिए [जेक अर्चिबाल्ड](https://x.com/jaffathecake) (अत्यंत प्रतिभाशाली) द्वारा गूगल विकासकर्ता साइट के अपने async/await लेख में दिए गए एक उदाहरण पर नज़र डालते हैं (नोट: 2024, लिंक हटा दिया गया है)।

<!--
चलिए एक ऐसे उदाहरण पर नज़र डालते हैं जिसे "❌ अनिर्माणीय" कहा गया है। (वर्णन है "मान लें कि हम कुछ श्रृंखला के URL फ़ेच करना चाहते हैं और उन्हें सही क्रम में जितनी जल्दी हो सके लॉग करते हैं।") -->

```js
// source: https://developers.google.com/web/fundamentals/primers/async-functions
function logInOrder(urls) {
  // fetch all the URLs
  const textPromises = urls.map(url => {
    return fetch(url).then(response => response.text());
  });

  // log them in order
  textPromises.reduce((chain, textPromise) => {
    return chain.then(() => textPromise)
      .then(text => console.log(text));
  }, Promise.resolve());
}
```

### एकल उद्देश्य?

मैं कहूंगा नहीं। `logInOrder` क्या कर रहा है?

1. `urls` की सूची के माध्यम से लूप करें  
1. इन्हें एक इनलाइन HTTP GET पर लागू करें:  
  1. HTTP `fetch`  
  1. प्रतिक्रिया पाठ शरीर लौटाएं  
1. `textPromises` में प्रत्येक प्रमिस के बाद `.then(text => console.log(text))` जोड़ें  
  1. परिणामों को क्रमिक रूप से प्रिंट करें  

इस एकल फ़ंक्शन में 5 अज्ञात विधियां परिभाषित की गई हैं। जेक ने भी इस बात पर ध्यान दिया है कि `.reduce` बहुत जटिल है। अपने पूरे कोड में ऐसे नुकसानदायक तंत्रों को हाथ से लिखना बनता नहीं है। अन्य तरीके से कहें तो, हम अंतहीन `document.createElement()`, `element.setAttribute()` आदि के साथ DOM निर्माण कोड नहीं लिखते हैं। बजाय इसके हम विभिन्न विकल्पों में से सबसे अच्छा उपकरण चुनते हैं: सहायक/उपयोगिता फ़ंक्शन, पुस्तकालय या फ्रेमवर्क।
