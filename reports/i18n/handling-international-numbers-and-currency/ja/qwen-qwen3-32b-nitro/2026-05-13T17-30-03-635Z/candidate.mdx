## 通貨: 地域化 (L10n) と国際化 (i18n)

これらはスクラブルの勝利を狙うためのものではなく、**製品が別の国で自然に感じられるようにする**プロセスを指します。

<p class="breakout quote">ローカルフォーマットに沿って通貨を表示しないことは明白なサインです。<br/>価格のフォーマットさえできないなら、輸送をどう扱いますか？</p>

国際化はテキスト翻訳から日付フォーマットまで幅広いトピックですが、本記事では特に**数値と通貨のフォーマット**に焦点を当てます。

ヨーロッパ通貨圏の3か国、アメリカ、インド間でのフォーマット違いを見てみましょう：

- `€1,234,567.89` アイルランド 🇮🇪  
- `1.234.567,89 €` ドイツ 🇩🇪  
- `1 234 567,89 €` フランス 🇫🇷  
- `$1,234,567.89` アメリカ 🇺🇸  
- `₹12,34,567.89` インド 🇮🇳  

混乱していますね！？右？記号や空白、句読点がいたるところに飛び散っています！EUが何かについて合意できるのが不思議です 😅  

## キーコンセプト  

解決策に移る前に、「**数値はローカル**」とはどういう意味でしょうか？  

### 数値はローカル 🏘️  

すべてのロケール（[ISO 3166で定義された国](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_codes)）は、数値のフォーマットルールを定義しています。

数値のフォーマットルールには以下が含まれます：

- 小数点：カンマ、ピリオド
- 千の位区切り：カンマ、ピリオド、スペース
- 通貨記号の位置とスペース

### 通貨はグローバル 🌎

`通貨` は特定の通貨単位を指します。（リストは [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217#Active_codes_(list_one)) を参照）

- 記号を指定：`$`、`€`、`£`、`¥`。（再利用されることが多い）
- 常に3文字のコードを持つ：`USD`、`EUR`、`GBP`、`JPY`
- 理論上は「どの」国でも使用/交換可能
- 通貨間変換には為替レートデータが必要
- ロケールに依存せず価値が変化しない

### ロケールが重要なのはいつか

ほとんどのEC/決済のREST APIは`price` + `currencyCode`を使用します。なぜロケールは使われないのですか？

ロケールは（通常）OS/デバイスレベルで設定され、ブラウザは`navigator.language`を通じてそれを提供します。すべてのユーザーが異なるロケールを持っている可能性があるため、数値や通貨のフォーマットはクライアントサイドで行うのが理にかなっています。

## 解決策

良いニュースです！現代のプログラミング言語にはこの機能のための組み込みサポートがあります。JavaScriptでは[`Intl`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl)クラスと`Intl.NumberFormat`が利用可能です！

コードを見てみましょう：

```javascript
const number = 1_234_567.89;

/**
 * ローカル通貨で数値をフォーマットする。
 * @param {number} amount - フォーマットする金額。
 * @param {string} currency - 3文字の通貨コード。
 * @param {string} [locale] - ユーザーのロケール文字列。
 */
const formatMoney = (amount, currency, locale = navigator.language) =>
  new Intl.NumberFormat(locale, { currency, style: 'currency' })
    .format(amount);

console.log('🇩🇪 ' + formatMoney(number, 'EUR', 'de-DE'));
console.log('🇮🇪 ' + formatMoney(number, 'EUR', 'ga-IE'));
console.log('🇫🇷 ' + formatMoney(number, 'EUR', 'fr-FR'));
```

税金の計算や割引の適用、通貨間の変換など、複雑な操作が必要な場合は、[dinero.js](https://v2.dinerojs.com/)などのライブラリを使用することをお勧めします。

## 次のステップ

特定の要件に応じて、関連する概念をさらに探求することをお勧めします：

- ユーザーのロケールに関するベストプラクティス。検出 + オーバーライドを許可する。（例：国選択のドロップダウン。）
- 整数の永続化（ドルではなくセントを保存。）
- 金銭計算。（例：`20% off`クーポンの適用、`subTotal + 税金`の計算など。）
- ライブ為替レート。（小売購入、外為/通貨交換の場合。）
