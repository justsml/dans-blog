## Money: Localization (L10n) and Internationalization (i18n)

Это не просто для победы в игре Scrabble: _локализация_ и _интернационализация_ означают процесс, позволяющий продукту **чувствовать себя как дома в другой стране**.

<p class="breakout quote">Показать валюту в неправильном локальном формате — явный сигнал: вы не приложили никаких усилий.<br/>Если вы не умеете форматировать цену, как вы справитесь с доставкой?</p>

Интернационализация — обширная тема, охватывающая всё от перевода текста до форматирования дат. В этой статье мы сосредоточимся на конкретном подпункте: **форматировании чисел и валюты**.

Рассмотрим форматирование в трёх странах еврозоны, США и Индии:

- `€1,234,567.89` Ireland 🇮🇪  
- `1.234.567,89 €` Germany 🇩🇪  
- `1 234 567,89 €` France 🇫🇷  
- `$1,234,567.89` USA 🇺🇸  
- `₹12,34,567.89` India 🇮🇳  

Хаос! Верно? Символы, пробелы и пунктуация летят повсюду! Удивительно, как ЕС может согласовать хоть что‑то! 😅  

## Критические концепции  

Прежде чем переходить к решениям, что мы подразумеваем под «Числа локальны»?  

### Числа локальны 🏘️  

Каждая локаль ([Country per ISO 3166](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_codes)) определяет правила форматирования чисел.

Правила форматирования чисел включают:

- Десятичный разделитель: запятая, точка.
- Разделитель тысяч: запятая, точка, пробел.
- Позиция символа валюты и пробелы.

### Валюта — глобальная 🌎

`currency` обозначает конкретную денежную единицу. (Список см. в [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217#Active_codes_(list_one)).)

- Задает символ: `$`, `€`, `£`, `¥`. (Часто переиспользуется.)
- Всегда имеет трёхбуквенный код: `USD`, `EUR`, `GBP`, `JPY`.
- Может использоваться/обмениваться в «любой» стране. Теоретически.
- Конвертация между валютами требует данных о курсах обмена.
- Значение не меняется в зависимости от локали.

### Когда локаль имеет значение

Большинство e‑commerce/платёжных REST API передают только `price` + `currencyCode`. Почему нет локалей?

Локали (как правило) задаются на уровне ОС/устройства, а браузеры делают их доступными через `navigator.language`. Поскольку каждый ваш пользователь может иметь свою локаль, имеет смысл выполнять форматирование чисел и валюты на клиенте.

## Решение

Хорошие новости! Современные языки программирования имеют встроенную поддержку этого. В JavaScript есть класс [`Intl`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl) и `Intl.NumberFormat`!

Посмотрим пример кода:

```javascript
const number = 1_234_567.89;

/**
 * Форматировать число в локальной валюте.
 * @param {number} amount - Сумма для форматирования.
 * @param {string} currency - Трёхбуквенный код валюты.
 * @param {string} [locale] - Строка локали пользователя.
 */
const formatMoney = (amount, currency, locale = navigator.language) =>
  new Intl.NumberFormat(locale, { currency, style: 'currency' })
    .format(amount);

console.log('🇩🇪 ' + formatMoney(number, 'EUR', 'de-DE'));
console.log('🇮🇪 ' + formatMoney(number, 'EUR', 'ga-IE'));
console.log('🇫🇷 ' + formatMoney(number, 'EUR', 'fr-FR'));
```

Если нужны более сложные операции — расчёт налогов, применение скидок или конвертация между валютами — стоит воспользоваться библиотекой вроде [dinero.js](https://v2.dinerojs.com/).

## Следующие шаги

В зависимости от ваших конкретных требований, возможно, захотите изучить смежные темы:

- Лучшие практики работы с пользовательской локалью. Обнаружение + возможность переопределения (например, выпадающий список стран).
- Сохранение целых чисел (храните центы, а не доллары).
- Операции с деньгами (например, применение купона «‑20 %», расчёт `subTotal + taxes` и т.п.).
- Актуальные курсы обмена (для розничных покупок, форекс/валютные обмены).
