## Argent : Localisation (L10n) et internationalisation (i18n)  

Ce ne sont pas seulement des termes pour dominer un jeu de Scrabble, la _localisation_ et l'_internationalisation_ désignent le processus de rendre un produit **compatible avec un autre pays**.  

<p class="breakout quote">Afficher une devise dans le mauvais format local est un aveu évident : vous n'avez fait aucun effort.<br/>Si vous ne pouvez pas formater un prix, comment pourriez-vous gérer l'expédition ?</p>  

L'internationalisation est un sujet vaste, couvrant tout, de la traduction des textes au formatage des dates. Dans cet article, nous nous concentrerons sur un sous-sujet particulier : **le formatage des nombres et de la monnaie**.  

Explorons les différences de format entre trois pays de la zone euro, les États-Unis et l'Inde :

- `€1,234,567.89` Irlande 🇮🇪  
- `1.234.567,89 €` Allemagne 🇩🇪  
- `1 234 567,89 €` France 🇫🇷  
- `$1,234,567.89` États-Unis 🇺🇸  
- `₹12,34,567.89` Inde 🇮🇳  

Le chaos, non ? Il y a des symboles, des espaces et des ponctuations partout ! C'est incroyable qu'ils arrivent à se mettre d'accord sur quoi que ce soit ! 😅

## Concepts clés  

Avant de plonger dans les solutions, qu'entendons-nous par « Les nombres sont locaux » ?  

### Les nombres sont locaux 🏘️  

Chaque locale ([Pays selon ISO 3166](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_codes)) définit des règles pour le formatage des nombres.  

Les règles de formatage des nombres incluent :  

- Décimales : virgule, point.  
- Milliers : virgule, point, espace.  
- Position et espacement du symbole monétaire.  

### La devise est globale 🌎  

Une `devise` désigne une unité monétaire spécifique. (Voir [ISO 4217](https://en.wikipedia.org/wiki/ISO_4217#Active_codes_(list_one)) pour la liste.)  

- Spécifie un symbole : `$`, `€`, `£`, `¥`. (Souvent réutilisés.)  
- Possède toujours un code à trois lettres : `USD`, `EUR`, `GBP`, `JPY`.  
- Peut être utilisée/échangée dans « n'importe quel » pays. En théorie.  
- La conversion entre devises nécessite des données sur les taux de change.  
- La valeur ne change pas en fonction de la locale.  

### Quand la locale est importante

La plupart des API REST pour le commerce électronique ou les paiements utilisent `price` + `currencyCode`. Pourquoi pas de locales ?

Les locales sont (généralement) définies au niveau du système d'exploitation/appareil, et les navigateurs les rendent disponibles via `navigator.language`. Comme chaque utilisateur peut avoir une locale différente, il est logique de formater les nombres et les devises côté client.

## Une Solution

Bonnes nouvelles ! Les langages de programmation modernes disposent d'un support intégré pour cela. En JavaScript, nous avons la classe [`Intl`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl) et `Intl.NumberFormat` !

Voyons un exemple de code :

```javascript
const number = 1_234_567.89;

/**
 * Formate un nombre en devise locale.
 * @param {number} amount - Le montant à formater.
 * @param {string} currency - Le code à trois lettres de la devise.
 * @param {string} [locale] - La chaîne de locale de l'utilisateur.
 */
const formatMoney = (amount, currency, locale = navigator.language) =>
  new Intl.NumberFormat(locale, { currency, style: 'currency' })
    .format(amount);

console.log('🇩🇪 ' + formatMoney(number, 'EUR', 'de-DE'));
console.log('🇮🇪 ' + formatMoney(number, 'EUR', 'ga-IE'));
console.log('🇫🇷 ' + formatMoney(number, 'EUR', 'fr-FR'));
```

Si vous avez besoin d'effectuer des opérations plus complexes, comme calculer des taxes, appliquer des remises ou convertir des devises, vous devriez utiliser une bibliothèque comme [dinero.js](https://v2.dinerojs.com/).

## Étapes suivantes

En fonction de vos besoins spécifiques, vous pourriez explorer des concepts connexes :

- Meilleures pratiques concernant la locale de l'utilisateur. Détecter et permettre des surcharges. (ex. un menu déroulant de pays.)
- Stocker des entiers (stocker les centimes, pas les dollars.)
- Calculs monétaires. (ex. appliquer un coupon `20% de réduction`, calculer `sous-total + taxes`, etc.)
- Taux de change en temps réel. (Pour les achats de détail, les changes/échanges de devises.)
