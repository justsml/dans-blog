<blockquote class="inset">
**अपडेट:** इस लेख ने [`llm://` URI स्कीम के लिए एक इंटरनेट-ड्राफ्ट](https://datatracker.ietf.org/doc/draft-levy-llm-uri-scheme/) को जन्म दिया।
</blockquote>

वे बुरे पुराने दिन याद हैं जब डेटाबेस से कनेक्ट करने का मतलब था एनवायरनमेंट वेरिएबल्स का एक बेतरतीब ढेर संभालना?

यह नाज़ुक कॉन्फ़िग का एक मीनार था। `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`... या रुको, क्या वह `DB_USERNAME` था? क्या यह `DB_PASS` है या `DB_PWD`? क्या इस बार मुझे `PG_*` प्रीफ़िक्स चाहिए? और टाइमआउट सेटिंग कहाँ जाती है?

यह ताश का एक नाज़ुक घर था, जो आपके प्रोडक्शन बिल्ड को इसलिए गिराने को तैयार था क्योंकि आप `HOST` को कैपिटलाइज़ करना भूल गए।

फिर, किसी के पास शानदार विचार आया कि बस एक URL¹ का उपयोग करें:

```bash
postgres://user:pass@host:5432/dbname
```

एक स्ट्रिंग। आपको जो कुछ चाहिए। सार्वभौमिक रूप से पार्स करने योग्य। पोर्टेबल। क्या मैं कह सकता हूँ... सुंदर?

तो हम LLM के साथ ऐसा क्यों कर रहे हैं जैसे 1999 है?

## एनव वार का विस्फोट

अभी, मेरी `.env` फ़ाइल परित्यक्त API कुंजियों के कब्रिस्तान जैसी दिखती है। `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `MISTRAL_API_KEY`, `GROQ_API_KEY`। और Azure के बारे में मुझे मत बताइए—आपको सिर्फ "हैलो" कहने के लिए एक एंडपॉइंट, एक डिप्लॉयमेंट नाम, एक API वर्जन और एक कुंजी चाहिए।

यह सिर्फ बदसूरत नहीं है; यह घर्षण है। हर बार जब मैं कोई मॉडल बदलना चाहता हूँ या किसी नए प्रदाता का परीक्षण करना चाहता हूँ, तो मैं इनिशियलाइज़ेशन कोड को फिर से लिख रहा हूँ, विशिष्ट पैरामीटर नामों के लिए दस्तावेज़ीकरण खोज रहा हूँ, और अपने एनवायरनमेंट कॉन्फ़िग में तीन और लाइनें जोड़ रहा हूँ।

क्या होगा अगर हम सिर्फ... DB URL के विचार को ~~चुरा लें~~ उधार ले लें?

## LLM कनेक्शन स्ट्रिंग्स का परिचय

कल्पना करें कि आप अपने पूरे मॉडल इंटरफ़ेस को एक ही लाइन से कॉन्फ़िगर कर रहे हैं:

```bash
llm://api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7&max_tokens=1500
llm://api.z.ai/glm-4.7?top_p=0.9&cache=true
```

---
<br />

### LLM कनेक्शन स्ट्रिंग की शारीरिक रचना

![LLM कनेक्शन स्ट्रिंग के भाग](./inline-url-diagram-dark.svg)

स्कीम `llm://` है। होस्ट प्रदाता का API बेस URL है। पथ मॉडल का नाम है। और क्वेरी पैरामीटर उन सभी रनटाइम विकल्पों को संभालते हैं जो आमतौर पर आपके कोड को अव्यवस्थित करते हैं।

## प्रमाणीकरण चाहिए? बढ़िया, इसे जोड़ें।

बिल्कुल `postgres://` की तरह, हम प्रमाणीकरण को सीधे बेक कर सकते हैं:

```bash
llm://app-name:sk-proj-123456@api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7
```

*नोट: हाँ, URL में क्रेडेंशियल डालना एक सुरक्षा जोखिम हो सकता है यदि आप उन्हें सार्वजनिक लॉग में चिपका रहे हैं। लेकिन आधुनिक लॉगिंग सेवाएँ इन पैटर्न को साफ करने में काफी अच्छी हैं, और ईमानदारी से, क्या आप अपनी `.env` फ़ाइल के साथ बेहतर व्यवहार कर रहे हैं? सत्यापित करें, स्वच्छ करें, और सावधानी से उपयोग करें।*

## लचीलापन? क्यों नहीं।

कई डेटाबेस लाइब्रेरीज़ एकाधिक होस्ट निर्दिष्ट करके राउंड-रॉबिन फेलओवर का समर्थन करती हैं। हमारे AI एजेंटों को भी वही विश्वसनीयता क्यों नहीं मिलनी चाहिए?

```bash
llms://primary.gpt,backup.gpt/gpt-6?temp=0.9
```

`llms://` में वह `s` कोई टाइपो नहीं है। यह बहुवचन है। यदि `primary.gpt` हैंग होता है, तो क्लाइंट स्वचालित रूप से `backup.gpt` पर पुनः प्रयास करता है। कोई जटिल राउटर लॉजिक आवश्यक नहीं।

<blockquote class="inset">एक स्ट्रिंग जिसमें आपके **प्रमाणीकरण** से लेकर आपके **एंडपॉइंट** तक आपके **हाइपरपैरामीटर** तक सब कुछ है।</blockquote>

## वैकल्पिक प्रारूप

मैं `llm://` से बंधा नहीं हूँ। विशिष्ट स्कीम मानक से कम मायने रखती है।

मैं एक ऐसी दुनिया की कल्पना कर सकता हूँ जहाँ हम संक्षिप्तता के लिए प्रदाता-विशिष्ट स्कीम का उपयोग करते हैं, जबकि मानक संरचना को बनाए रखते हैं:

```bash
ollama://localhost:11434/llama3
vercel://anthropic/sonnet-4.5?temp=0.8&web_search={"maxUses":3}
bedrock://us-west-2.aws/anthropic/sonnet-4.5?temp=0.8&cacheControl=ephemeral
```

सटीक सिंटैक्स चाहे जो भी हो, मुख्य लाभ निर्विवाद हैं:

1.  **पोर्टेबिलिटी:** अपने पूरे कॉन्फ़िग को स्थानीय स्क्रिप्ट से क्लाउड वर्कर पर कॉपी और पेस्ट करें।
2.  **CLI-अनुकूल:** अपनी स्क्रिप्ट में एक एकल तर्क पास करें। `my-agent --model "llm://..."` `my-agent --model gpt-4 --temp 0.7 --key $KEY --host ...` से बेहतर है।
3.  **भाषा-अज्ञेय:** हर प्रोग्रामिंग भाषा में एक मजबूत URL पार्सर होता है। हमें मुफ्त में सत्यापन, पार्सिंग और स्वच्छता मिलती है।

<blockquote class="ai-response inset">डेटाबेस की दुनिया को यह समझने में दशकों लग गए।<br /><b>अच्छी खबर, AI टाइमलाइन में, यह लगभग आधा वाइब-वर्ष पहले की बात है।</b></blockquote>

## फैसला

हमें किसी और जटिल कॉन्फ़िगरेशन मानक या नई YAML-आधारित मेनिफेस्ट फ़ाइल की आवश्यकता नहीं है। हमें बस उस एक उपकरण का उपयोग करने की आवश्यकता है जो पिछले 30 वर्षों से बाकी इंटरनेट के लिए काम कर रहा है।

चलिए पहिया को फिर से आविष्कार करना बंद करें और अपने LLM कनेक्शन के साथ वही सम्मान करना शुरू करें जो हम अपने डेटाबेस को देते हैं। आपकी `.env` फ़ाइल (और आपकी समझदारी) आपको धन्यवाद देगी।
