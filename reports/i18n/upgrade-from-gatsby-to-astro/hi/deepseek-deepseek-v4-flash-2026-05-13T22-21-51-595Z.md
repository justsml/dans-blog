## सही प्रौद्योगिकी स्टैक चुनना

तो, मेरे पास बहुत सारे विकल्प हैं, जिन्हें मैंने अंततः **Remix**, **Next.js** और **Astro** तक सीमित कर दिया,

मैं अपने मूल्यांकन प्रक्रिया पर एक पूरी ब्लॉग श्रृंखला लिख सकता था, लेकिन मैं इसे यहाँ संक्षेप में बताऊँगा:

<p class="breakout">मैंने [Astro](https://astro.build) को चुना क्योंकि मैं कितनी जल्दी _सार्थक चीजें_ कर सकता था।</p>

यह थोड़ा आश्वस्त करने वाला था कि Astro में कोई स्पष्ट क्लाउड पक्षपात या फ्रेमवर्क एजेंडा नहीं है।

Astro एकमात्र तकनीक नहीं थी जिसका मैंने उपयोग किया, यहाँ स्टैक का पूरा विवरण है:

- [Astro](https://astro.build): एक आधुनिक स्टैटिक साइट जनरेटर।
- [ShadcnUI](https://ui.shadcn.com): पुन: प्रयोज्य घटकों का एक संग्रह।
- [Tailwind CSS](https://tailwindcss.com): एक उपयोगिता-प्रथम CSS फ्रेमवर्क।
- [MDX](https://mdxjs.com): मार्कडाउन सामग्री + इनलाइन घटक।
- [Pagefind](https://pagefind.app): तेज़, स्टैटिक और ऑफलाइन साइट खोज लाइब्रेरी। Algolia की कोई आवश्यकता नहीं!
- [Utterances](https://utteranc.es): GitHub issues पर आधारित टिप्पणी प्रणाली।
- [Netlify](https://www.netlify.com): स्वचालित परिनियोजन, कैप्चा के साथ संपर्क फ़ॉर्म।

<p class="breakout quote">Astro जल्दी ही मेरे अपग्रेड का आधार बन गया।</p>

यहाँ कुछ प्रमुख विशेषताएँ दी गई हैं जो मुझे विशेष रूप से उपयोगी लगीं:

- `.astro` फ़ाइलें: पहली नज़र में, Astro घटक React JSX घटकों जैसे लग सकते हैं, हालाँकि ये काफी भिन्न हैं और अलग-अलग उद्देश्यों की पूर्ति करते हैं। (नीचे तुलना तालिका देखें।)
- अपने स्वयं के Golang [निर्माण उपकरणों](https://github.com/withastro/compiler) और Vite द्वारा संचालित: यह बस काम करता है। ESM/CJS, TypeScript, कोड बंडलिंग, स्टाइल, इमेज आदि को सहजता से संभालता है।
- [कोई फ्रेमवर्क पक्षपात](https://docs.astro.build/en/guides/framework-components/#official-ui-framework-integrations) या [क्लाउड पक्षपात](https://docs.astro.build/en/guides/deploy/) नहीं। (*खाँसी* Next.js, OpenNext)
- [स्टैटिक बनाम हाइब्रिड](https://docs.astro.build/en/basics/rendering-modes/) रेंडरिंग: Astro [अधिकांश क्लाउड प्लेटफार्मों को लक्षित करने की लचीलापन](https://docs.astro.build/en/guides/deploy/) प्रदान करता है: AWS, GCP, Firebase, Netlify, Vercel, Cloudflare Pages, Azure, Fly.io, और कई अन्य।
- सामग्री संग्रह: [`getCollection`](https://docs.astro.build/en/reference/api-reference/#getcollection) API सामग्री फ़ाइलों को डेटा स्रोत के रूप में उपयोग करना सरल बनाता है।
- फ़ाइल-आधारित रूटिंग: Astro की फ़ाइल-आधारित रूटिंग प्रणाली, `getStaticPaths` के साथ मिलकर, पेज जनरेट करना आसान बनाती है।

## आधुनिक CSS: वाह

फ्रंटएंड डेवलपमेंट पर वापस आकर, मैं नेटिव CSS में हुई प्रगति से बहुत प्रसन्न हुआ:

- CSS वेरिएबल्स: काफी समय से उपलब्ध हैं, और 202* से ब्राउज़रों में काफी स्थिर हैं।
- नेस्टिंग: आखिरकार स्पेक में आ गई, और पहले की अजीब सिंटैक्स के बिना। अब यह Less या SCSS जैसी है।
- नए सेलेक्टर: [`:is()`, `:where()`, और `:has()`](https://www.youtube.com/watch?v=3ncFpP8GP4g) एलिमेंट्स को अधिक सटीक रूप से टार्गेट करने की सुविधा देते हैं।
- `ch`, `vw` जैसी आधुनिक इकाइयाँ और `clamp()` जैसे फंक्शन लेआउट और टाइपोग्राफी पर बेहतर नियंत्रण प्रदान करते हैं।
- `-inline` और `-block` एट्रीब्यूट्स के साथ स्पेसिंग को अधिक स्वाभाविक रूप से सेट करें। पैडिंग या मार्जिन को क्षैतिज या ऊर्ध्वाधर अक्ष पर सेट करें। `margin: 0 1rem 0 1rem` के बजाय → `margin-inline: 1rem`।
- उन्नत लेआउट: CSS Grid को फिर से सीखना। वाह, इसमें बहुत सारी चीज़ें हैं। इसके उपयोग के अनगिनत तरीकों के साथ यह भारी लग सकता है। ध्यान रखें, आप 1 या 2 तरीके निकालकर काम चला सकते हैं। ये बेहतरीन संसाधन देखें जिन्होंने मुझे ग्रिड के साथ ट्रिक्स करने में मदद की: [Kevin Powell का वीडियो: Learn CSS Grid the easy way](https://www.youtube.com/watch?v=rg7Fvvl3taU), [मीडिया क्वेरी के बिना रिस्पॉन्सिव](https://ardilamorin.com/responsive-no-media-queries/), [CSS की एक पंक्ति में दस आधुनिक लेआउट](https://web.dev/articles/one-line-layouts)।

## खोज: Pagefind

तीसरे पक्ष की सेवाओं या डेटाबेस होस्टिंग के बिना **साइट सर्च** लागू करना एक मजेदार चुनौती लग रहा था। आखिरकार, ऐसा नहीं है कि मेरे पास इंडेक्स करने के लिए 10,000 पोस्ट हैं (अभी तक)।

[Astro के सामुदायिक इंटीग्रेशन](https://astro.build/integrations/?search=find) ब्राउज़ करते हुए मुझे एक शानदार टूल मिला जिसके बारे में मैं पहले जानना चाहता था: [Pagefind](https://pagefind.app/)।

<p class="breakout quote">कुछ ही उपकरण किसी समस्या को उतनी अच्छी तरह हल करते हैं जितना Pagefind स्थानीय साइट सर्च को हल करता है।</p>

Pagefind को लागू करने की सरलता एक आनंद है। इसे किसी भी स्टैटिक साइट कंटेंट के साथ एकीकृत किया जा सकता है, और आप चुन सकते हैं कि आप डिफ़ॉल्ट UI चाहते हैं, या आप चाहें तो कुछ भी कस्टम बना सकते हैं।

इसने मेरी सभी इच्छाओं को बड़ी सफाई से हल किया। इसे एकीकृत करने में केवल मिनट लगे, और अधिकांश काम में एक `<div id="search"></div>` टैग और कुछ स्टाइलिंग जोड़ना शामिल था!
