> אבטחת מפתחות API ואסימונים היא **חשובה קריטית**!

טעות אחת יכולה לגרום לאיבוד שליטה בשרת ובנתונים שלך לטובת האקרים!

לא צריך להיות כל כך קשה לקבוע אם אסימון מסוים חייב להיות מוסתר – אפילו בהתבסס על תיעוד רשמי!

הדבר מחמיר לעיתים עם הבלגן של מונחים קשורים שתיתקלו בהם: _אסימונים_, _מפתחות_, _אישורים_, _סודות_, _פרטי_, ו‑_ציבוריים_.

בואו ננסח זאת מחדש כ‑`סוד` ו‑`לא‑סוד`.

* 🔒 [`Secret keys`](#-secret-keys) חייבים להישאר מוסתרים. באופן כללי הם לעולם לא צריכים לעזוב את השרת הפרטי שלכם (או שירות – כמו heroku, netlify או travis-ci).
* 🌍 [`Non-secret keys`](#-non-secret-keys) מתארים מחרוזות שניתן לשתף בחופשיות ולהכליל בבקשות דפדפן.

## 🔒 `Secret keys`

** ‼️ חשוב:** `Secret keys` **חייבים** להיות מוסתרים מ‑Git _וגם_ לא לכלול בקוד של הדפדפן. [_איך להשתמש ב‑dotenv_](#-how-to-handle-secrets-safely)

<br />

_איך אתה יודע מתי אתה מתמודד עם `Secret key`?_

<br />

**👍 כלל אצבע:** שרתים שמחזירים `CORS errors` אינם נתמכים בדפדפן. זה מרמז בחוזקה שאתה **חייב** לפרוקס את השירות, ולטפל בו כאילו הוא `secret`.

**👍 כלל אצבע:** שירותים יקרים צריכים (כמעט) תמיד להיות פרוקסים או מוסתרים.

**👍 כלל אצבע:** אם אתה מבצע פעולה של כתיבה (**העלאת קובץ, הוספת שורה למסד נתונים**), ייתכן שאתה מתמודד עם `secret keys`.

<br />

**_מקרי שימוש ותכונות:_** `Secret` keys

- הרשאה לטווח ארוך (אישורים, אסימוני גישה, JSON Web Tokens)
- הרשאה לטווח קצר (OAuth tokens, אחסון סשן)
- גישה לשירותים בתשלום/יקרים (לאימות, קידוד גיאוגרפי, אחסון קבצים, וכו')
- החלק הפרטי של זוג ציבורי/פרטי (RECAPTCHA, Stripe, Auth0)
- אישורי שירות (Email/SMTP, LDAP/Directory Services)
- הצפנת נתונים ובדיקה של שלמות

### רשימת בדיקה: טיפול בטוחים בסודות

#### סקירה מהירה

השלם את הצעדים הבאים כדי **להסיר סודות מהקוד שלך:**

- [ ] החלף מפתחות מקודדים בקבועים במשתני סביבה. למשל `process.env.API_SECRET`
- [ ] השתמש בספרייה כמו [`dotenv`](https://github.com/motdotla/dotenv#dotenv) יחד עם קובץ `.env`. הוסף את הסודות שהיו מקודדים קודם לקובץ `.env`.
- [ ] הוסף שורה של `.env` לקובץ `.gitignore` שלך!

> **DON'T** ליצור קובץ `.env` על שרתים שמופעלים. השתמש בכלי ניהול משתני הסביבה שמספק שירות האחסון שלך (למשל [Heroku](https://devcenter.heroku.com/articles/config-vars), Netlify, AWS EC2): לדוגמה **לוח בקרה או שורת פקודה**.

## 🌍 `Non-secret keys`

**👍 כלל אצבע:** כאשר מפתח חייב להישלח לדפדפן בקוד או באופן מוטמע (למשל באמצעות תג `<script src="https://my-api/?apiKey=123-abc-456">`), **זה בהחלט `non-secret`**. דוגמה נפוצה היא Google Maps.

<br />

**_מקרים ושימושים:_** מפתחות `Non-secret`

- גישה קצרה טווח (מזהי סשן משתמש, JSON Web Tokens)
- הגבלת גישה ל‑API לפי אפליקציה/מפתח (לאימות, קידוד גיאוגרפי, וכו')
- החלק הציבורי של זוג ציבורי/פרטי (RECAPTCHA, Stripe, Auth0)
- מזהי אנליטיקה

#### ✅ טיפול במפתחות שאינם סודיים:

> **אפשר לקודד במפורש מפתחות שאינם סודיים (ציבוריים)!**

הפכו זאת לניהול נוח בטווח הארוך עם קובץ `config.js` משותף לאפליקציה שלכם.

**דוגמה:**

```js
// config.js
module.exports = {
  googleMapsKey: '123-abc'
};
```

```js
// load-map.js
const config = require('./config.js');
const key = config.googleMapsKey;
const src = `//maps.googleapis.com/maps/api/js?key=${key}`;
// ...
```
