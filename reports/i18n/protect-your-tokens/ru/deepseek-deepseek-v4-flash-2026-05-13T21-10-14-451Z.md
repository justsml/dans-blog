## Когда защищать ваши токены?

<!--  For the purpose of this article we'll treat the following terms as related: . **They are not interchangable** despite most documentation and [StackOverflow](https://stackoverflow.com/questions/51698672/how-to-secure-my-api-key) answers using them as such. -->

> Защита API-ключей и токенов **критически важна**!

Одна ошибка может привести к потере контроля над вашим сервером и данными хакерам!

Не должно быть так сложно определить, нужно ли скрывать конкретный токен — даже на основе официальной документации!

Ситуацию часто усугубляет каша из связанных терминов, с которыми вы столкнетесь: _токены_, _ключи_, _учетные данные_, _секреты_, _приватные_ и _публичные_.

Давайте переформулируем это как различие между `secret` (секретными) и `non-secret` (несекретными).

* 🔒 [`Секретные ключи`](#-secret-keys) ДОЛЖНЫ быть скрыты. Как правило, они НИКОГДА не должны покидать ваш приватный сервер (или сервис — например, Heroku, Netlify или Travis CI).
* 🌍 [`Несекретные ключи`](#-non-secret-keys) описывают строки, которыми можно свободно делиться и включать в браузерные запросы.

<br />

## 🔒 `Секретные ключи`

** ‼️ Важно:** `Секретные ключи` **ДОЛЖНЫ** игнорироваться Git'ом _И_ отсутствовать во всем браузерном коде. [_Как использовать dotenv_](#-how-to-handle-secrets-safely)

<br />

_Как понять, что вы имеете дело с `секретным ключом`?_

<br />

**👍 Практическое правило:** серверы, возвращающие `CORS-ошибки`, не поддерживаются браузерами. Это настоятельно указывает на то, что вам **НЕОБХОДИМО** проксировать сервис, обращаясь с ним как с `секретным`.

**👍 Практическое правило:** дорогостоящие сервисы следует (почти) всегда проксировать или скрывать.

**👍 Практическое правило:** если вы выполняете операцию записи (**загрузка файла, вставка строки в БД**), вы, вероятно, имеете дело с `секретными ключами`.

<br />

**_Варианты использования и особенности:_** `Секретные` ключи

- Долгосрочная авторизация (учётные данные, токены доступа, JSON Web Tokens)
- Краткосрочная авторизация (OAuth-токены, хранилище сессий)
- Доступ к платным/дорогим сервисам (для аутентификации, геокодирования, файлового хранилища и т.д.)
- Приватная часть пары публичный/приватный ключ (RECAPTCHA, Stripe, Auth0)
- Сервисные учётные данные (Email/SMTP, LDAP/Службы каталогов)
- Шифрование данных и проверка целостности

### Контрольный список: безопасная работа с секретами

#### Краткий обзор

Выполните следующие шаги, чтобы **исключить секреты из вашего кода:**

- [ ] Замените жёстко заданные ключи на переменные окружения. Например: `process.env.API_SECRET`
- [ ] Используйте библиотеку [`dotenv`](https://github.com/motdotla/dotenv#dotenv) вместе с файлом `.env`. Добавьте ранее жёстко заданные секреты в файл `.env`.
- [ ] Добавьте строку `.env` в ваш файл `.gitignore`!

> **НЕ** создавайте файл `.env` на развёрнутых серверах. Используйте инструменты управления переменными окружения, предоставляемые вашим хостингом (например, [Heroku](https://devcenter.heroku.com/articles/config-vars), Netlify, AWS EC2): **панель управления или командную строку.**

## 🌍 `Несекретные ключи`

**👍 Практическое правило:** если ключ должен быть отправлен в браузер в коде или инлайн (например, через тег `<script src="https://my-api/?apiKey=123-abc-456">`), **это точно `несекретный`**. Типичный пример — Google Maps.

<br />

**_Варианты использования и особенности:_** `Несекретные` ключи

- Краткосрочный доступ (идентификаторы сессий пользователей, JSON Web Tokens)
- Ограничение доступа к API по приложению/разработчику (для аутентификации, геокодирования и т.д.)
- Публичная часть пары публичный/приватный (RECAPTCHA, Stripe, Auth0)
- Идентификаторы аналитики

#### ✅ Обработка несекретных ключей:

> **Несекретные (публичные) ключи можно безопасно зашивать в код!**

Чтобы упростить долгосрочное управление, используйте общий `config.js` для вашего приложения.

**Пример:**

```js
// config.js
module.exports = {
  googleMapsKey: '123-abc'
};
```

```js
// load-map.js
const config = require('./config.js');
const key = config.googleMapsKey;
const src = `//maps.googleapis.com/maps/api/js?key=${key}`;
// ...
```
