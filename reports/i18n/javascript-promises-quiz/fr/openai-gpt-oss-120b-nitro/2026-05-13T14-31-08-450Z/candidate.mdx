>
  <slot name="question">
  <div className="question">
    Quel sera le résultat du code suivant ?
    ```js
        var p = new Promise((resolve, reject) => {
          return Promise.reject(Error('The Fails!'))
        })
        p.catch(error => console.log(error.message))
        p.catch(error => console.log(error.message))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lors de l'utilisation du constructeur Promise, vous devez appeler l'un des callbacks `resolve()` ou `reject()`. Le constructeur Promise ignore la valeur de retour de l'exécuteur, ainsi la Promise supplémentaire créée avec `Promise.reject()` n'est pas chaînée à `p`. Les deux gestionnaires sont attachés à `p`, qui reste en attente, tandis que la promesse rejetée retournée est signalée comme non gérée par l'environnement d'exécution.
  </div>
  </slot>
</Challenge>

>
  <slot name="question">
  <div className="question">
    Quel sera le résultat du code suivant ?
    ```js
        var p = new Promise((resolve, reject) => {
          reject(Error('The Fails!'))
        })
        .catch(error => console.log(error))
        .then(error => console.log(error))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lors du chaînage des `.then` et des `.catch`, il est utile de les imaginer comme une série d'étapes. Chaque `.then` reçoit la valeur renvoyée par le `.then` précédent (en tant qu'argument). Cependant, si votre « étape » rencontre une erreur, toutes les « étapes » `.then` suivantes seront sautées jusqu'à ce qu'un `.catch` soit rencontré. Si vous voulez écraser une erreur, il suffit de renvoyer une valeur non‑erreur. Celle‑ci pourra être récupérée dans n'importe quel `.then` suivant.
  </div>
  </slot>
</Challenge>
