  <slot name="question">
  <div className="question">
    Quelle sera la sortie du code suivant ?
    ```js
        var p = new Promise((resolve, reject) => {
          return Promise.reject(Error('The Fails!'))
        })
        p.catch(error => console.log(error.message))
        p.catch(error => console.log(error.message))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lorsqu'on utilise le constructeur Promise, il faut appeler soit `resolve()`, soit `reject()`. Le constructeur Promise ignore la valeur de retour de l'exécuteur, donc la promesse supplémentaire créée avec `Promise.reject()` n'est pas chaînée à `p`. Les deux gestionnaires sont attachés à `p`, qui reste en attente, tandis que la promesse rejetée retournée est signalée comme non gérée par l'environnement hôte.
  </div>
  </slot>
</Challenge>

    ```js
        var p = new Promise((resolve, reject) => {
          reject(Error('The Fails!'))
        })
        .catch(error => console.log(error))
        .then(error => console.log(error))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lorsqu'on enchaîne des `.then` et des `.catch`, il est utile de les considérer comme une série d'étapes. Chaque `.then` reçoit la valeur renvoyée par le `.then` précédent (en tant qu'argument). Cependant, si votre "étape" rencontre une erreur, toutes les `.then` suivantes seront ignorées jusqu'à ce qu'un `.catch` soit rencontré. Si vous souhaitez écraser une erreur, il vous suffit de renvoyer une valeur non-erreur. Elle pourra être récupérée par tous les `.then` suivants.
  </div>
  </slot>
</Challenge>
