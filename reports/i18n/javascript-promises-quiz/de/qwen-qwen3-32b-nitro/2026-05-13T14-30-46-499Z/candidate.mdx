    Was wird die Ausgabe des folgenden Codes sein?
    ```js
        var p = new Promise((resolve, reject) => {
          return Promise.reject(Error('The Fails!'))
        })
        p.catch(error => console.log(error.message))
        p.catch(error => console.log(error.message))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Beim Verwenden des Promise-Konstruktors müssen Sie entweder `resolve()` oder `reject()` Callbacks aufrufen. Der Promise-Konstruktor ignoriert den Rückgabewert des Executors, daher ist der zusätzliche Promise, der mit `Promise.reject()` erstellt wird, nicht an `p` gekettet. Die beiden Handler sind an `p` angehängt, welcher weiterhin ausstehend bleibt, während der zurückgegebene abgelehnte Promise als nicht behandelt vom Hostumgebung gemeldet wird.
  </div>
  </slot>
</Challenge>

<Challenge

  index={2}
  group="Fehlerbehandlung"
  title="Kettenbildung von `.then` und `.catch`"
  options={[
    {text: 'Fehler und `undefined` ausgeben', isAnswer: true},
    {text: 'Fehler zweimal ausgeben'},
    {text: 'UnhandledPromiseRejectionWarning'},

  ]}
>
  <slot name="question">
  <div className="question">
    Was wird die Ausgabe des folgenden Codes sein?
    ```js
        var p = new Promise((resolve, reject) => {
          reject(Error('The Fails!'))
        })
        .catch(error => console.log(error))
        .then(error => console.log(error))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Bei der Kettenbildung von `.then` und `.catch` ist es hilfreich, sie als eine Reihe von Schritten zu betrachten. Jeder `.then`-Aufruf erhält den Wert, der vom vorherigen `.then` zurückgegeben wurde (als Argument). Wenn jedoch ein Schritt einen Fehler erzeugt, werden alle nachfolgenden `.then`-Schritte übersprungen, bis ein `.catch` auftritt. Um einen Fehler zu überschreiben, musst du einfach einen nicht-fehlerhaften Wert zurückgeben. Dieser kann dann von jedem nachfolgenden `.then` weiterverarbeitet werden.
  </div>
  </slot>
</Challenge>

  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">

In diesem Beispiel fängt der erste `.catch` den Fehler ab und gibt `console.log` zurück, der nur über einen zusätzlichen `.then()` nach beiden `.catch`-Blöcken erfasst werden könnte.
  </div>
  </slot>
</Challenge>

  options={[

  <slot name="question">
  <div className="question">
    Was wird die Ausgabe des folgenden Codes sein?
    ```js
        new Promise((resolve, reject) => {
            resolve('Success!')
          })
          .then(() => {
            throw Error('Oh noes!')
          })
          .catch(error => {
            return "actually, that worked"
          })
          .catch(error => console.log(error.message))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    **Hinweis:** `.catch`-Blöcke können dazu verwendet werden, Fehler zu ignorieren (oder zu überschreiben), indem einfach ein regulärer Wert zurückgegeben wird.

    Dieser Trick funktioniert nur, wenn ein nachfolgender `.then` vorhanden ist, der den Wert empfangen kann.
  </div>
  </slot>
</Challenge>
