## AngularJS 也能有趣！

> 适用于：AngularJS v1.x

1.  AngularJS 开发者很快会发现，他们的中大型应用在散落的 `$watch` 和常常臃肿的 `$scope` 支撑下，承受不住重量。
2.  保持 `$scope` 不被多余的 UI 状态占满，尽量限制整体层级的大小和深度。

### 双向数据绑定：双向之剑

单单双向绑定就已经让从 Backbone 等其他框架转来的开发者感到 **惊为天人**。

问题在于：很多站点 **长期过度使用** Angular 的设计模式。  
这会导致指令蔓延，`$scope/rootScope` 轻易拥有成千上万的实例，并且会黏住巨大的对象，彻底破坏垃圾回收的可能性。

你知道接下来会怎样：浏览器被耗尽！永远被迫以 **疯狂的速度** 执行无休止且冗余的 UI/DOM 重新编译。

### 停止过度 Angular 化

> “如果你唯一的工具是一把锤子，那么所有问题看起来都是钉子。”
>
> — 古老箴言

你的应用在指令上有问题吗？

```jade
current-user-status-label
  div(ng-if='loggedIn')
    view-user-surplusage(ng-if='!editMode')
      .head: contact-details(user='user')
      .tool: contact-buttons(loggedIn='loggedIn')
      a.edit-icon(ng-click='editMode = true')
    edit-user-surplusage(ng-if='editMode')
      .head: avatar-edit(user='user')
      .body: edit-contact-details(user='user')
      a.save-icon(ng-click='editMode = false')
```

让我们设计一个灵活的 user‑widget 来帮助：

1.  具备可复用性的 DRY Angular 代码组件化  
2.  可理解的指令，保持指令体积和深度最小（注意你的 ng‑repeat）  
3.  简单的 Service 层  
4.  实现所需的实际代码极少——只需 HTML/View 代码  

```jade
// jade
user-widget
  div(ng-if='loggedIn')
    div.edit(ng-if='editMode')
      h4.email-icon: input(type='email', ng-model='user.email')
      h4.phone-icon: input(type='email', ng-model='user.phone')
      a.save-icon(ng-click='editMode = false')
    div.show(ng-if='!editMode')
      h1.users-icon {{ user.name  }}
      h4.email-icon {{ user.email }}
      h4.phone-icon {{ user.phone }}
      a.edit-icon(ng-click='editMode = true')
  div(ng-if='!loggedIn')
    h5: i Welcome User
    a.btn(href='/login') Login
```

## 解决方案

### Angular 小技巧

1.  使用单向绑定（例如 `{ :: title }`）  
2.  限制指令的递归嵌套  
3.  必须嵌套指令时，**绝不能**在 `ng-repeat` 内部——性能会逼近 `O(n^2)^3` ;)  
    I. 在工厂模式中使用原生 JS/DOM 代码创建基础 DOM/UI 片段，例如模态框、状态栏。可在指令或控制器中调用这些 UI 工厂。  
4.  **额外提示：**了解 [浏览器渲染生命周期](https://developers.google.com/web/fundamentals/performance/rendering/index?hl=en) 的成本与触发点：动画、合成渲染、回流  

### 使用 Browserify 组织项目

并非专门针对 Angular 本身，但对简单的依赖解析至关重要。

[Browserify](https://github.com/substack/browserify-handbook/blob/master/readme.markdown#exports) 让 JavaScript 项目几乎没有额外代码开销（好吧，几百字符）即可可管理。

[只阅读此章节](https://github.com/substack/browserify-handbook/blob/master/readme.markdown#exports)（来自[Browserify Handbook](https://github.com/substack/browserify-handbook/)）。

## 替代方案

### 来自 Facebook 的 [ReactJS](https://facebook.github.io/react/)

如果你拥有大量小型可复用的 UI 组件——ReactJS 可能是更好的选择：

- 如果你的项目...?：
  - 对 UI/DOM 实现有别于 Angular 的哲学
  - 已经使用某种“框架”——可以 **在 AngularJS、Ember、Backbone 旁边使用 ReactJS**（但尽量避免）。
  - 在自己的代码中频繁处理数据模型变更，使用 ReactJS 可以规避 Angular 那种注意力缺陷多动症式的 digest/loop 模式带来的负担

### 来自 Google 的 [Polymer Project](http://www.Polymer-Project.org/)

### 更纯粹的 JS 方法

- 顺带一提，这里展示的是如何编写与框架无关的代码（+1 可测试性，+1 可复用性）
  1. 使用普通的 JavaScript 类加载数据（AJAX/JSONP/页面内嵌等）。
  1. 使用 Mustache 模板生成 HTML 字符串（或直接生成 DOM）。
  1. 如果可能，将渲染后的内容缓存到 `localStorage`。
  1. （可选）再添加一个事件监听器以重新渲染内容。我统一使用的事件名为 `refresh.<class-name>`。
