## ¡AngularJS puede ser divertido!

> Para: AngularJS v1.x

1.  Los desarrolladores de AngularJS descubren rápidamente que sus aplicaciones medianas‑grandes se están hundiendo bajo el peso de los `$watch` dispersos y el a menudo inflado apoyo conocido como `$scope`.
2.  Mantén tu `$scope` libre de estado UI excesivo, intenta limitar el tamaño y la profundidad de tu jerarquía global.

### Enlace de datos bidireccional: espada bidireccional

El enlace bidireccional por sí solo hace que, viniendo de otros frameworks como Backbone, sea, bueno, **frickin amazeballs**.

El problema es: muchos sitios **sobreutilizan crónicamente** los patrones de diseño de Angular.  
Esto genera una expansión de directivas y un `$scope/rootScope` que fácilmente llega a miles de instancias, y puede aferrarse a objetos enormes impidiendo cualquier esperanza de una recolección de basura eficaz.

Sabes a dónde va esto: ¡un navegador exhausto! Condenado a trabajar a un **ritmo frenético** ejecutando recompilaciones UI/DOM interminables y redundantes.

### Detén la SOBRE‑Angular.JSificación

> “Si tu única herramienta es un martillo, entonces todo problema parece un clavo.”
>
> – viejo adagio

¿Tu aplicación tiene problemas con las directivas?

```jade
current-user-status-label
  div(ng-if='loggedIn')
    view-user-surplusage(ng-if='!editMode')
      .head: contact-details(user='user')
      .tool: contact-buttons(loggedIn='loggedIn')
      a.edit-icon(ng-click='editMode = true')
    edit-user-surplusage(ng-if='editMode')
      .head: avatar-edit(user='user')
      .body: edit-contact-details(user='user')
      a.save-icon(ng-click='editMode = false')
```

Diseñemos un widget de usuario flexible que ayude a:

1.  Componentización versátil con código Angular DRY  
2.  Directivas comprensibles, con tamaño y profundidad mínima (cuidado con los `ng-repeat`)  
3.  Capa de servicio sencilla  
4.  Poco código real para implementar – solo HTML/Vista

```jade
// jade
user-widget
  div(ng-if='loggedIn')
    div.edit(ng-if='editMode')
      h4.email-icon: input(type='email', ng-model='user.email')
      h4.phone-icon: input(type='email', ng-model='user.phone')
      a.save-icon(ng-click='editMode = false')
    div.show(ng-if='!editMode')
      h1.users-icon {{ user.name  }}
      h4.email-icon {{ user.email }}
      h4.phone-icon {{ user.phone }}
      a.edit-icon(ng-click='editMode = true')
  div(ng-if='!loggedIn')
    h5: i Welcome User
    a.btn(href='/login') Login
```

## Soluciones

### Consejos para Angular

1.  Usa enlaces unidireccionales (p. ej. `{ :: title }` )
1.  Limita la anidación recursiva de directivas
1.  Y si necesitas anidar directivas, **NUNCA** lo hagas dentro de un `ng-repeat` – el rendimiento empezará a comportarse como `O(n^2)^3` ;)
    I. Utiliza código nativo JS/DOM en un patrón de fábrica para crear fragmentos básicos de DOM/UI, por ejemplo: cuadro modal, barra de estado. Llama a las fábricas UI desde directivas o controladores.
1.  **Bonus:** Comprende el costo y los disparadores del [ciclo de vida de renderizado del navegador](https://developers.google.com/web/fundamentals/performance/rendering/index?hl=en): animaciones, composición de renderizado, reflows

### Usa Browserify para organizar el proyecto

No es exclusivo de Angular, pero es esencial para una resolución de dependencias sencilla.

[Browserify](https://github.com/substack/browserify-handbook/blob/master/readme.markdown#exports) permite que los proyectos JS sean manejables con prácticamente sin sobrecarga de código (ok, unos pocos cientos de caracteres).

[Simplemente lea esta sección](https://github.com/substack/browserify-handbook/blob/master/readme.markdown#exports) del [Manual de Browserify](https://github.com/substack/browserify-handbook/).

## Alternativas

### [ReactJS](https://facebook.github.io/react/) de Facebook

Si tiene un montón de componentes UI pequeños y reutilizables, ReactJS podría ser una mejor opción:

- Si su proyecto...?:
  - Tiene una filosofía diferente de implementación UI/DOM que Angular
  - Ya cuenta con algún tipo de “framework” – Puede **usar ReactJS junto a** AngularJS, Ember, Backbone. (Evítelo si es posible).
  - Maneja cambios frecuentes en el modelo de datos en su propio código, se beneficiará al evitar la naturaleza hiperactiva del patrón digest/bucle de Angular

### [Proyecto Polymer](http://www.Polymer-Project.org/) de Google

### Enfoque JS más puro

- Por cierto, aquí es donde intento crear código agnóstico al framework (+1 testabilidad, +1 reutilización)
  1.  Use plain javascript class para cargar datos (AJAX/JSONP/Embedded in page, et al.)
  1.  Use mustache templating para crear HTML strings (or DOM directly)
  1.  Cache rendered content en localStorage si puede
  1.  (Optional) Now add an event listener para re-renderizar el content. He estandarizado el nombre del evento `refresh.<class-name>`
