#### #1 Promiseは何かを保持する必要があります

**関数から常に`return`する**ことが不可欠です。

Promiseのコールバック関数は、`.then(callback)`や`.catch(callback)`で特定のパターンに従います。

各戻り値は次の`.then()`のコールバックに渡されます。

```js
function addTen(number) {
  return number + 10;
}

Promise.resolve(10)  // 10
  .then(addTen)      // 20
  .then(addTen)      // 30
  .then(addTen)      // 40
  .then(console.log) // logs "40"
```

> "常に返す"のボーナス：コードはユニットテストがはるかに簡単になります。

#### #2 実際の`Error`インスタンスを使用する

JavaScriptにはエラーに関する興味深い挙動があります（これは非同期**および**同期コードに適用されます）。

[<i>repl.itで例を確認: `throwing errors in javascript`</i>](https://repl.it/@justsml/throwing-errors-in-javascript)
<img alt="JavaScriptでのエラー投げ方" src="../throwing-errors-in-javascript.webp" />

**行番号やコールスタックの詳細を得るには**、`Error`インスタンスを使用する必要があります。文字列を投げることでPythonやRubyのように動作しません。

JavaScriptでは**`throw "string"`が処理されると見える**かもしれませんが、`catch`ハンドラでは文字列しか取得できません。*すべての情報はこれだけです。* 以前の[スタックフレーム](https://en.wikipedia.org/wiki/Call_stack#Stack_and_frame_pointers)は含まれません。

正しい`new Error`の例:

```js
throw new Error('message')           // ✅
Promise.reject(new Error('message')) // ✅
throw Error('message')               // ✅
Promise.reject(Error('message'))     // ✅
```

よくある反パターン:

```js
throw 'error message'  // ❌
Promise.reject(-42)    // ❌
```

#### #3 エラーを意味のある場所で処理する

Promiseは`.catch()`を使用して、エラーをすっきりと処理する方法を提供します。これは基本的に特殊な`.then()`のようなものです—前の`.then()`で発生したエラーを処理します。例を見てみましょう...

```js
Promise.resolve(42)
  .then(() => 'hello')
  .catch(() => console.log('will not get hit'))
  .then(() => throw new Error('totes fail'))
  .catch(() => console.log('WILL get hit'))
```

`.catch()`がDOMイベントハンドラ（例: `click`、`keypress`）のように見えるかもしれませんが、配置が重要です。なぜなら、エラーを**自分より上にある**場所のエラーしかキャッチできないからです。

**エラーの上書きは比較的単純です** `.catch()`コールバックで非エラー値を返すと、Promiseチェーンは`.then()`コールバックを順番に実行します。（実質的に。）

次の例の流れを追ってみましょう：

```js
Promise.resolve(42)
  .then(() => 'hello')
  .then(() => throw new Error('totes fail'))
  .catch(() => {
    return 99
  })
  .then(num => num + 1)
  .then(console.log) // 期待される出力: 100
```

**重要なのはこの流れです。**

馬鹿げた例ですが、Promiseにおける**エラーやデータの流れ**を示すために設計されています。

シーケンスの概要は以下の通りです：

1. 42は初期値です。
1. 次のメソッドは常に`hello`を返します。
1. 以前の値は無視し、`'totes fail'`というメッセージでエラーをスローします。
1. `.catch()`がエラーをインターセプトし、代わりに`99`を返します。これはその後の`.then()`によって処理されます。
1. `num`をインクリメントし、`100`を返します。
1. `console.log`メソッドは`100`を受け取り、それを出力します！:tada:

#### #4 名前付き関数で明確さを追加する 🦄✨

**読みやすさ**の観点から、以下の2つの例を比較してみましょう：

**匿名関数:** ❌

```js
Promise.resolve(10)          // 10
  .then(x => x * 2)          // 20
  .then(x => x / 4)          // 5
  .then(x => x * x)          // 25
  .then(x => x.toFixed(2))   // "25.00"
  .then(x => console.log(x)) // 予期される出力: "25.00"
```

**名前付き関数:** ✅

```js
Promise.resolve(10) // 10
  .then(double)     // 20
  .then(quarter)    // 5
  .then(square)     // 25
  .then(format)     // "25.00"
  .then(log)        // 予期される出力: "25.00"

const double = x => x * 2
const quarter = x => x / 4
const square = x => x * x
const format = x => x.toFixed(2)
const log = x => console.log(x)
```

**ボーナス:** ✅

> 配列メソッドと互換性あり!!!

名前付き関数は`Array.prototype.`の仲間たちと再利用できます。`.map()`、`.filter()`、`.every()`、`.some()`、`.find()`なども含め、自由に活用可能です！

コレクションパイプライン #FTW:

```js
// IT'S LIKE THE SAME THING :mindblown:

[10, 20]           // [ 10, 20 ]
  .map(double)     // [ 20, 40 ]
  .map(quarter)    // [ 5, 1:0 ]
  .map(square)     // [ 25, 100 ]
  .map(format)     // [ "25.00", "100.00" ]
  .map(log)        // 予期される出力: "25.00", "100.00"

```

線形スタイルのコーディングが望ましくない場合でも...シンプルな関数があります！

必要に応じて自由に使用できます:

```js
// ネストパターン
// ❌ これはやらないでください

const result = format(square(quarter(double(10))))

log(result)
// 予期される出力: "25.00"
```
