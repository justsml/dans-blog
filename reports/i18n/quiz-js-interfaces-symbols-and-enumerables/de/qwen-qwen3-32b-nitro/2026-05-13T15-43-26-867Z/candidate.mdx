  <slot name="question">
  <div className="question">
    Wird die Eigenschaft `age` bei einer `for...in`-Iteration aufgelistet werden?
    ```js
        const person = {};
        Object.defineProperty(person, 'age', {
          value: 25,
          enumerable: true
        });
        for (let key in person) {
          console.log(key);
        }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Die `enumerable`-Eigenschaft in `Object.defineProperty()` steuert, ob die Eigenschaft in Enumerationsmethoden wie `for...in` angezeigt wird. In diesem Beispiel, da `enumerable: true` gesetzt ist, wird die `age`-Eigenschaft bei der Iteration aufgelistet.
  </div>
  </slot>
</Challenge>

        const car = {};

  </div>

    Was wird das Ergebnis der folgenden Vergleichsoperation sein?
    ```js
        const sym1 = Symbol('id');
        const sym2 = Symbol('id');
        console.log(sym1 === sym2);
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Jeder Aufruf von `Symbol()` erzeugt einen eindeutigen und unveränderlichen Wert, selbst wenn die Beschreibung identisch ist. In diesem Fall sind `sym1` und `sym2` unterschiedliche Symbole, daher gibt der Vergleich `false` zurück.
  </div>
  </slot>
