    ```js
        const person = {};
        Object.defineProperty(person, 'age', {
          value: 25,
          enumerable: true
        });

        }
    ```
  </div>

    La propriété `enumerable` dans `Object.defineProperty()` contrôle si la propriété apparaîtra dans les méthodes d'énumération comme `for...in`. Dans cet exemple, puisque `enumerable: true`, la propriété `age` sera répertoriée lors de l'itération.
  </div>

    Quelle est l'enumerabilité par défaut d'une propriété lors de l'utilisation de `Object.defineProperty()` sans spécifier `enumerable`?
    ```js
        const car = {};
        Object.defineProperty(car, 'make', {

        });
        console.log(Object.keys(car));
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lorsque vous utilisez `Object.defineProperty()` sans spécifier `enumerable`, sa valeur par défaut est `false`. Cela signifie que la propriété `make` ne s'affichera pas dans `Object.keys()` ou d'autres méthodes d'énumération.
  </div>
  </slot>
</Challenge>
