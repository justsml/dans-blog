    - 先頭列、または列の左端のプレフィックス。

    - 前方一致、`LIKE 'prefix%'`。


  <slot name='explanation'>
  <div className="explanation">
    PostgreSQLでは識別子の引用に二重引用符を使用します:
    - 識別子（列名、テーブル名）には二重引用符 `"..."` を使用します
    - 文字列リテラルには一重引用符 `'...'` を使用します
    - `first.name` の形式は `テーブル名.列名` として解釈されます

    引用符がないと、列名内のピリオドはスキーマ/テーブルの区切り文字として解釈されてしまいます！さらに、引用符で囲まれていない識別子はデフォルトで小文字に変換されます。

    驚きを減らすために、`snake_case` を使用し、特殊文字を避けるのが最善です。

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={8}
  group="データサンプリング"
  title="TABLESAMPLE 構文"
  options={[
  ]}
>
  <slot name="question">
  <div className="question">
    学生の **10% をランダムにサンプリング** する正しい構文はどれですか？
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    これは難しい問題です！おおよそ6つの罠が仕掛けられています。その一部を紹介します：

    1. 高レベルの説明と構文の断片が混在。不完全な情報をどう適応・組み立てるかを試します。
    2. `TABLESAMPLE` 構文の知識が必要です。なぜなら、どの `TABLESAMPLE` オプションも正しい構文を使っていないからです！（意地悪ですよね！でも大丈夫、あなたならできると信じています！！！ ❤️）
    3. `ROW_NUMBER() OVER (ORDER BY RAND())` と `ORDER BY RANDOM() > 10` はさらにミスリードです。一見正しそうに見えますが、違います。`RAND()` は存在しません。そして `RANDOM() > 10` は常に偽となるブール式です。なぜなら `RANDOM()` は常に `0.0-1.0` の範囲だからです。
    もう一つ問題があります。`ORDER BY RANDOM() > 10` はブール値でソートするのであって、乱数ではありません。でも `> 10` はパーセンテージのように感じられますよね？
    4. `BERNOULLI` は有効ですが、`SAMPLETABLE` は無効です。
    5. `WHERE RANDOM() >= 0.1` はトリックです！`RANDOM()` は `0.0 <= x < 1.0` の範囲の値を返すので、`>= 0.1` は約90%の行を返し、10%ではありません。

    `TABLESAMPLE` メソッドについて、PostgreSQLは2つのサンプリング方法をサポートしています：
    - `BERNOULLI`: 各行が等しい確率で選択される
    - `SYSTEM`: ブロックレベルのサンプリング（高速だがランダム性は低い）

    サンプリングについてもっと学ぶには、[Render](https://render.com/blog/postgresql-random-samples-big-tables) の記事をチェックしてください。

    例：
    ```sql
        -- Sample 10% of rows
        SELECT * FROM students TABLESAMPLE BERNOULLI (10);

        -- For repeatable results, use REPEATABLE:
        SELECT * FROM students
        TABLESAMPLE BERNOULLI (10) REPEATABLE (42);
    ```
    また、`TABLESAMPLE` は[あまりランダムではないと報告されています](https://stackoverflow.com/questions/8674718/best-way-to-select-random-rows-postgresql)。もっとランダムな固定サイズのサンプルが必要な場合は、`ORDER BY RANDOM() LIMIT 10` の使用を検討してください。ただし、これは10%ではなく10行を返し、大きなテーブルでは遅くなる可能性があることに注意してください。

    より良いランダムサンプリング方法を提供するサードパーティの拡張機能もありますが、それは今回の範囲外です！ 😅
  </div>
  </slot>
</Challenge>


  <slot name='explanation'>
  <div className="explanation">
    さて、これは少し厄介でした。落とし穴は、`x = NULL` は決して行にマッチせず、常にカウントが `0` の1行を返すことです。

    `NULL` は未知の値であるため、それ自体を含め、何とも比較できません。

    `NULL` 値をチェックする正しい方法は `IS NULL` を使うことです。

    例：
    ```sql
        postgres=# SELECT count(*)
          FROM students
          WHERE age = null;
        count 
        -------
            0
        (1 row)

        postgres=# 
        postgres=# SELECT count(*)
          FROM students
          WHERE age is null;
        count 
        -------
            3
        (1 row)
    ```
