    - 前导列，或列的最左前缀。
    - 诸如 `=`、`>`、`<`、`BETWEEN` 和 `IN` 之类的运算符，具体取决于查询形状。
    - 前缀匹配，如 `LIKE 'prefix%'`。

    哪个是有效的语法，用于**随机采样 10%** 的学生？
  </div>

  <slot name='explanation'>
  <div className="explanation">
    这道题有点狡猾！它大约设置了 6 个陷阱，包括：

    1. 高级描述加语法片段。测试你如何适应并组装不完整的信息。
    2. 需要熟悉 `TABLESAMPLE` 语法，因为所有 `TABLESAMPLE` 选项都没有使用正确的语法！（阴险！我知道！继续加油，我相信你！！！❤️）
    3. `ROW_NUMBER() OVER (ORDER BY RAND())` 和 `ORDER BY RANDOM() > 10` 是更多的烟雾弹。它们看起来可能是正确的，但实际上不是。`RAND()` 并不存在。而 `RANDOM() > 10` 是一个布尔表达式，由于 `RANDOM()` 总是在 `0.0-1.0` 范围内，所以它永远为假。
    还有一个问题，`ORDER BY RANDOM() > 10` 会按布尔值排序，而不是随机数。但那个 `> 10` 确实让人感觉它可能是一个百分比，对吧？
    4. `BERNOULLI` 是有效的，但 `SAMPLETABLE` 不是。
    5. `WHERE RANDOM() >= 0.1` 是个陷阱！`RANDOM()` 返回的值在 `0.0 <= x < 1.0` 范围内，所以 `>= 0.1` 会返回大约 90% 的行，而不是 10%。

    至于 `TABLESAMPLE` 方法，PostgreSQL 支持两种采样方法：
    - `BERNOULLI`：每行有相等的概率
    - `SYSTEM`：块级采样（更快但随机性较差）

    要了解更多关于采样的信息，请查看 [Render](https://render.com/blog/postgresql-random-samples-big-tables) 上的这篇文章。

    示例：
    ```sql
        -- Sample 10% of rows
        SELECT * FROM students TABLESAMPLE BERNOULLI (10);

        -- For repeatable results, use REPEATABLE:
        SELECT * FROM students
        TABLESAMPLE BERNOULLI (10) REPEATABLE (42);
    ```
    另外值得一提的是，`TABLESAMPLE` [已被报告](https://stackoverflow.com/questions/8674718/best-way-to-select-random-rows-postgresql) 随机性不够好。如果你需要更随机的固定大小样本，可以考虑使用 `ORDER BY RANDOM() LIMIT 10`。但请注意，这会返回 10 行，而不是 10%，并且在大型表上可能会很慢。

    还有一些第三方扩展提供了更好的随机采样方法，但这超出了本问题的范围！😅
  </div>
  </slot>
</Challenge>

  </div>

  <slot name='explanation'>
  <div className="explanation">
    好吧，这有点棘手。陷阱在于 `x = NULL` 永远不会匹配任何行，并且它总是返回一行计数为 `0`。

    由于 `NULL` 是未知值，它不能与任何东西比较，甚至不能与自己比较。

    检查 `NULL` 值的正确方法是使用 `IS NULL`。

    示例：
    ```sql
        postgres=# SELECT count(*)
          FROM students
          WHERE age = null;
        count 
        -------
            0
        (1 row)

        postgres=# 
        postgres=# SELECT count(*)
          FROM students
          WHERE age is null;
        count 
        -------
            3
        (1 row)
    ```
  </div>
  </slot>


  <slot name='explanation'>
  <div className="explanation">
    `EXPLAIN ANALYZE` 实际上会执行查询！分析写操作时要小心，特别是 `UPDATE`、`DELETE` 和 `INSERT` 语句。每次 `EXPLAIN ANALYZE` 后可能需要运行清理操作以获得可重复的结果。

    最佳实践：
    - 对 UPDATE/DELETE/INSERT 使用 `EXPLAIN`（不带 ANALYZE）
    - 将修改操作包装在事务中：
    ```sql
        BEGIN;
        EXPLAIN ANALYZE ...;
        ROLLBACK;
    ```
