## Docker Rocks

> Aggiornamenti settembre 2016, 2018  
> Boot2Docker è stato sostituito da Docker per Mac

> Nota storica: questo post è volontariamente conservato come snapshot di Docker per Mac del 2015. I consigli sulle prestazioni e i nomi degli strumenti riflettono quel momento; i flussi di lavoro attuali di Docker Desktop e Compose sono avanzati.

1. Docker è fantastico, veloce e flessibile.  
1. Gli strumenti precedenti, in particolare boot2docker, erano lenti e propensi a crash.

Docker può attualmente eseguire in modo nativo un kernel Linux v3.4+ - e la VM boot2docker attuale esegue effettivamente v4.

Miglior utilizzo del tuo hardware: installa l'ultima versione di Debian o Ubuntu sul tuo Mac/Windows.

Su, quei giochi non aiutano il tuo codice...

### Controlla la tua configurazione

Analizza l'output del comando `docker info`.

1. Sicurezza: Controlla il server per porte aperte inaspettatamente (usando `nmap` da una rete remota)  
1. DNS: Usa un server DNS con cache locale o a bassa latenza.  
1. Archiviazione: Usa il driver di archiviazione corretto (`overlay2` è probabile)

Aggiornato 2024:

- Docker Desktop è un software proprietario, ma gratuito per uso personale. È un ottimo modo per iniziare con Docker su macOS o Windows.  
- Se stai cercando una soluzione più open source, controlla [Rancher Desktop](https://rancherdesktop.io/).
