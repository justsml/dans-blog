## Docker ist großartig

> Aktualisierungen September 2016, 2018  
> Boot2Docker wurde durch Docker for Mac ersetzt

> Historische Notiz: Dieser Beitrag wird absichtlich als Snapshot von Docker-for-Mac aus dem Jahr 2015 beibehalten. Die Leistungs- und Tooling-Ratschläge spiegeln diesen Moment wider; aktuelle Docker Desktop- und Compose-Workflows haben sich weiterentwickelt.

1. Docker ist großartig, schnell und flexibel.  
1. Vorherige Tools, insbesondere boot2docker, waren langsam und stürzten häufig ab.

Docker kann derzeit direkt auf einem Linux-Kernel v3.4+ laufen – und die aktuelle boot2docker-VM läuft tatsächlich mit v4.

Beste Nutzung Ihrer Hardware: Installieren Sie die neueste Debian- oder Ubuntu-Version auf Ihrem Mac/Windows-System,

...kommen, diese Spiele helfen Ihrem Code nicht weiter...

### Überprüfen Sie Ihre Einrichtung

Prüfen Sie die Ausgabe des Befehls `docker info`.

1. Sicherheit: Überprüfen Sie den Server auf unerwartet offene Ports (mit `nmap` von einem entfernten Netzwerk aus)  
1. DNS: Nutzen Sie einen lokalen Cache oder Niedrig-Latenz-DNS-Server.  
1. Speicher: Verwenden Sie den richtigen Speicher-Treiber (`overlay2` ist wahrscheinlich der beste Kandidat)

Aktualisiert 2024:

- Docker Desktop ist zwar proprietär, aber für private Nutzung kostenlos. Es ist eine hervorragende Möglichkeit, mit Docker auf OSX oder Windows zu starten.  
- Wenn Sie eine offeneren Lösung suchen, schauen Sie sich [Rancher Desktop](https://rancherdesktop.io/) an.
