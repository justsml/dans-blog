## Article Summary
The article argues that "generative UI" is not a single technology but a layered stack—product shell, UI composition model, runtime/transport, and agent/tool backend—and that conflating these layers leads to confusion. It introduces a control spectrum from tool-to-component rendering (safe, developer-owned) to open-ended generated HTML (high agent freedom, risky), recommending the smallest amount of freedom that solves the user problem.
