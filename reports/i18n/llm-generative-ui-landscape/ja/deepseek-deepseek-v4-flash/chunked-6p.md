## Article Summary
The article argues that "generative UI" is not a single technology but a layered stack—product shell, UI composition model, runtime/transport, and agent/tool backend—and that conflating these layers leads to confusion. It maps the current ecosystem (AG-UI, A2UI, json-render, CopilotKit, OpenAI Apps SDK, etc.) onto this framework, then introduces a control spectrum from tool-to-component rendering to open-ended generated HTML, advocating for the minimal agent freedom that solves the user problem. The intended audience is developers and architects building LLM-powered user interfaces.
