## Заметки о языках программирования

#### Уверен, мои Разнообразные наблюдения уже где‑то звучали, но вот мой список самых интересных языков:

### JavaScript

Моя единственная истинная любовь, предельно гибкий и вездесущий — универсальный, удивительно мощный чемпион!  
Это язык № 1 по активности/популярности на GitHub.com уже _годы_ подряд.

Признаюсь, годами я относился к нему с презрением и насмешкой, а теперь **это мой любимый язык**.

**ES6** лишь усилил моё ~~~зависимость~~~ чувство любви. Хотя чистый ES5 навсегда останется в моём сердце, каждый раз, когда я использую **ES6**, ощущаю укус радиоактивного паука…

Было четыре причины, которые подтолкнули меня к **ES6‑лагерю**:

1.  Это весело. Серьёзно. Видны ощутимые выигрыши в красоте, ясности и продуктивности.

- Субъективные утверждения, говорите? Позвольте показать вам немного ES6:
- `let expired = users.filter(u => Date.now() > u.trialDate)`
- Теперь не нужно притворяться, что вы умеете пользоваться `Object.create` или `Object.defineProperty`
- Смотрите примеры ниже

1.  По состоянию на июль 2015 года ES6 официально завершённый стандарт!  
1.  Поддержка фактически 100 % *! … Хорошо, BabelJS нужен, чтобы «подправить» ваш код до совместимости с ES5. Транспилеры JavaScript исторически вызывали недоверие. Однако в последние годы (2014‑15) ситуация изменилась: BabelJS стал ключевым драйвером развития языка. Его используют десятки компаний, включая Microsoft и Facebook, на крупнейших сайтах.  
1.  [Последние версии Node](https://nodejs.org/en/blog/release/v4.0.0/) используют тот же движок V8, что и Chrome v45; это версия 4.5  

#### Examples  

> Я покажу, что именно заставило меня наконец‑то «попить» ES6‑коктейль.  

В моём недавнем опыте ES6 ускоряет написание кода. Точнее.  
Благодаря более лаконичному синтаксису требуется заметно меньше умственных ресурсов, чтобы пробежаться по старому коду (или по коду коллег).  

Я регулярно наблюдаю экономию в 20‑50 % KLOC. Это как «похудеть» до уровня Кейт Мосс!

```js
// /services/users.js
class Users {
  constructor(data) {
    this.users = data || [];
  }
  expired() {
    return this.users.filter(u => Date.now() > u.trialDate);
  }
}
```

### Rust

##### [Official Site](http://www.rust-lang.org/)

- **Pros**

- Представьте язык, быстрый как C и мощный как Python/C++, но без типичных сложностей и подводных камней, в которые попадают даже самые опытные разработчики.
  - На самом деле я бы сказал, что Rust примерно так же сложен, как спецификация ES6.
  - В него включено множество дополнительных возможностей:
    1.  По сути Rust транспилирует полусемантический синтаксис в **чистый C‑код**!
    1.  Включая \***\*все лучшие практики\*\*** в C, которые вы, вероятно, нарушаете, я ~~в конечном итоге~~ всегда делаю.
    - Автоматически вы получаете:
    - Автоматическое управление памятью (не нужен медленный сборщик мусора!)
    - Идеальное владение/блокировка объектов (минимизированные мьютексы и переключения контекста)
    - Времена жизни объектов (реализованы автоматически\*, и кодируются так, будто вы знали каждый крайний случай)
    - Предотвращает практически все ошибки времени выполнения (серьёзно, ваши пути кода становятся явными: нельзя просто пропустить ветку)
  - О, да, он также предлагает истинную расширяемость языка через разумную функцию «macro».
    - Нужны comprehensions? [Стиль Scala? Готово](https://gist.github.com/hanny24/5749688), и [как в Python? Готово](https://gist.github.com/JeffBelgum/5e762761cd63c796e803).
    1.  Слишком хорошо, чтобы быть правдой? Нет, становится лучше:
    - Показатели передовых технологий (статистика github.com) показывают, что Rust сильно конкурирует, а иногда и обгоняет Go (новый язык от Google)
      - Около 4 000 звёзд больше, чем у Go (примерно 12 200 сейчас)
      - Вдвое больше вкладчиков (2×! — 1 071 vs 479 у Go)
      - В три раза больше форков (3×! — 2 343 vs 765)
      - Число открытых задач отстает лишь слегка (2 000 vs 1 730 у Go)
      - Pull Request’ов (Rust 70+ vs 1 у Go)
    - Я тоже трижды проверил эти цифры.
  - Другие библиотеки очень стабильны благодаря конструкциям и правилам Rust.
  - Модель потоков пригодна даже для обычных смертных

- **Cons**
  - Приличные **веб‑фреймворки** ещё новы, мало проверены и обычно плохо документированы (хотя они **становятся** весьма впечатляющими — по состоянию на март 2015).
  - Много ранних изменений до версии 1.0, ломающих совместимость

### Python

- **Pros**
  - Огромный набор готовых алгоритмов уже реализован в Python (см.: scilearnkit, numpy, matplotlib, pil/pillow и т.д.)
  - Очень весело писать! Comprehensions и Decomposition — отличные возможности, заставляющие другие языки выглядеть раздутыми.
  - Массивы, «Sequences», кортежи и прочее относительно просты

```python
## dummy code: defines a color + pixel-coord -
def pixel(x, y, r, g, b): return dict(x=x, y=y, r=r, g=g, b=b)
## Create a new pixel object and apply to set of vars
x, y, r, g, b = pixel(10, 20, 255, 255, 255)
## Now we can call pixel
```

- **Недостатки**
  - Досадно, что Python 2.x и 3.x несовместимы. Великий раскол продолжается даже спустя многие годы.
  - Некоторые ключевые библиотеки не всегда понятны разработчикам (numpy)

### Haskell

- **Плюсы**
  - Очень удовлетворительно, когда наконец запоминаешь достаточно синтаксиса, чтобы собрать выразительные шаблоны на основе comprehensions
  - Вы изучаете умопомрачительные паттерны кода — часто применимые и к другим языкам.
- **Минусы**
  - Синтаксис и паттерны могут быть трудны для привыкания.
