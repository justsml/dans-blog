## प्रोग्रामिंग भाषाओं के नोट्स

#### मुझे यकीन है कि मेरे विविध अवलोकन पहले भी किए जा चुके हैं, लेकिन यहाँ मेरी सबसे रोचक भाषाओं की सूची है:

### जावास्क्रिप्ट

मेरी एकमात्र सच्ची मोहब्बत, अत्यंत बहुमुखी और सर्वव्यापी – सर्वांगीण, अद्भुत‑शक्तिशाली चैंपियन!  
यह GitHub.com पर लगातार _सालों_ से #1 सबसे सक्रिय/लोकप्रिय भाषा है।

मैं इसे स्वीकार करने से बचता रहा, लेकिन कई सालों तक मैं केवल तिरस्कार और उपहास से ही इस भाषा को देखता रहा, जो अब **मेरी पसंदीदा भाषा** बन गई है।

**ES6** ने मेरी ~~~आसक्ति~~~ प्रेम को और बढ़ा दिया है। जबकि शुद्ध ES5 हमेशा मेरे दिल में एक विशेष स्थान रखेगा, हर बार जब मैं कुछ **ES6** उपयोग करता हूँ, तो मुझे वह रेडियोधर्मी मकड़ी‑दंश जैसा एहसास होता है...

चार कारण थे जिन्होंने मुझे **ES6 कैंप** की ओर धकेला:

1.  यह मज़ेदार है। सच्चाई। सुंदरता, स्पष्टता और उत्पादकता में ठोस लाभ मिलते हैं।

- व्यक्तिपरक दावे, कह रहे हैं? चलिए आपको ES6 का एक छोटा नमूना दिखाते हैं:
- `let expired = users.filter(u => Date.now() > u.trialDate)`
- अब आपको `Object.create` या `Object.defineProperty` का उपयोग कैसे करना है, यह दिखावा करने की जरूरत नहीं।
- नीचे उदाहरण देखें

1.  जुलाई 2015 तक, ES6 आधिकारिक रूप से अंतिम मानक बन चुका है!  
1.  समर्थन प्रभावी रूप से 100 % *! … ठीक है, आपका कोड ES5‑संगत बनाने के लिए BabelJS की जरूरत होगी। ऐतिहासिक रूप से JS ट्रांसपाइलर को नापसंद किया जाता था। लेकिन हालिया (2014‑15) में चीज़ें बदल गईं; BabelJS भाषा प्रगति का मुख्य चालक बन गया है। माइक्रोसॉफ्ट, फेसबुक सहित कई बड़ी कंपनियां इसे अपने बड़े‑पैमाने के साइटों पर उपयोग करती हैं।  
1.  [Node के नवीनतम संस्करण](https://nodejs.org/en/blog/release/v4.0.0/) Chrome v45 के समान V8 JS इंजन को शामिल करते हैं; यह v4.5 है।

#### उदाहरण

> मैं आपको वह चीज़ दिखाने वाला हूँ जिसने आखिरकार मुझे ES6‑स्वाद वाले कूलएड पीना शुरू करवा दिया।

मेरे हालिया अनुभव में, ES6 आपको कोड तेज़ लिखने में मदद करता है। सीधे‑साधे।  
कोड छोटा होने के कारण, आपके पुराने कोड (या टीम‑मेट के कोड) को समझने में काफी कम मस्तिष्क‑शक्ति लगती है।

मैं नियमित रूप से 20‑50 % KLOC बचत देखता हूँ। यह लगभग केट मॉस की ट्रिम जैसा है!

**_ग़ायब छवि:_ EcmaScript 5 बनाम ES 2016 - डेमो: क्लासेज, डीस्ट्रक्चरिंग, स्लिक**  
{/* ](../images/screenshots/Diff-ES6-vs-ES5-Head-to-Head.png) */}

```js
// /services/users.js
class Users {
  constructor(data) {
    this.users = data || [];
  }
  expired() {
    return this.users.filter(u => Date.now() > u.trialDate);
  }
}
```

- फ़ंक्शन को पास किए गए फ़ील्ड्स को ‘एक्सट्रैक्ट’ और ‘चेक’ करने वाला थकाऊ कोड अब नहीं। उदाहरण `add()` देखें:

```js
// /services/users.js
class Users {
  constructor(data) {
    this.users = data || [];
  }
  add({ name, email, password }) {
    // पासवर्ड हैश स्टोर करें, हमें केवल 1 स्पष्ट `var/let` की ज़रूरत है – बाकी वेरिएबल्स ऊपर `{fields}` जादू से ‘डिफ़ाइंड’ होते हैं ^^^
    let hash = getSha256(password);
    return http
      .post("/users", {
        name: name,
        email: email,
        passwordHash: hash
      })
      .then(usr => this.users.push(usr)); // सेवा प्रतिक्रिया पर उपयोगकर्ता जोड़ें
  }
}
```

### Rust

##### [Official Site](http://www.rust-lang.org/)

- **Pros**

- कल्पना कीजिए अगर कोई भाषा C जितनी तेज़ और Python/C++ जितनी शक्तिशाली हो, फिर भी उन जटिलताओं/पिटफ़ॉल्स से मुक्त हो जो आमतौर पर सबसे कुशल डेवलपर्स को भी फँसाते हैं।
  - वास्तव में मैं अनुमान लगाता हूँ कि Rust की जटिलता ES6 स्पेसिफ़िकेशन के बराबर है।
  - इसमें बहुत सारे अतिरिक्त फीचर हैं:
    1.  मूलतः Rust सेमी‑डायनामिक सिंटैक्स को **शुद्ध C कोड** में ट्रांसपाइल करता है!
    1.  C में सभी बेहतरीन प्रैक्टिसेज़ को शामिल करके, जो आप संभवतः गड़बड़ करेंगे, मैं ~~आखिरकार~~ हमेशा करता हूँ।
    - स्वचालित रूप से आपको मिलते हैं:
    - ऑटो मेमोरी मैनेजमेंट (धीमी गार्बेज कलेक्टर की ज़रूरत नहीं!)
    - परफ़ेक्टली स्कोप्ड ऑब्जेक्ट ओनरशिप/लॉकिंग (म्यूटेक्सिंग और कॉन्टेक्स्ट स्विचिंग न्यूनतम)
    - ऑब्जेक्ट लाइफ़टाइम्स (स्वचालित रूप से लागू *, और कोड ऐसा लिखा जैसे आप हर एज केस जानते हों)
    - लगभग सभी रन‑टाइम एरर को रोकता है (सच में, आपके कोड‑पाथ स्पष्ट हो जाते हैं: आप कोई कोड‑पाथ अनदेखा नहीं कर सकते)
  - ओह हाँ, यह एक सेंसिबल ‘मैक्रो’ फीचर के साथ असली भाषा एक्स्टेंसिबिलिटी भी देता है।
    - कॉम्प्रिहेंशन चाहिए? [Scala स्टाइल? हो गया](https://gist.github.com/hanny24/5749688), और [Python जैसा? हो गया](https://gist.github.com/JeffBelgum/5e762761cd63c796e803)।
    1.  बहुत अच्छा लग रहा है? नहीं, और भी बेहतर:
    - ब्लीडिंग‑एज इंडिकेटर्स (github.com स्टैट्स) दिखाते हैं कि Rust बहुत प्रतिस्पर्धी है या यहाँ‑तक कि Go (Google की नई‑नई भाषा) को भी पीछे छोड़ रहा है
      - Go से लगभग 4K अधिक स्टार्स (वर्तमान में लगभग 12,200)
      - कुल योगदानकर्ता अधिक (2×! – 1,071 बनाम Go के 479)
      - फ़ोर्क्स अधिक (3×! – 2,343 बनाम 765)
      - ओपन इश्यूज़ की संख्या, थोड़ा कम (2,000 बनाम Go के 1,730)
      - पुल रीक्वेस्ट्स (Rust 70+ बनाम Go का 1)
    - मैंने भी इन नंबरों को तीन बार चेक किया।
  - अन्य लाइब्रेरीज़ बहुत स्थिर हैं क्योंकि Rust की कंस्ट्रक्ट्स और नियमों की वजह से।
  - थ्रेडिंग मॉडल साधारण mortals के लिये भी उपयोगी है

- **Cons**
  - ठीक‑ठाक **वेब फ्रेमवर्क्स** अभी भी नई, अपरिक्षित, और अक्सर डॉक्यूमेंटेड नहीं हैं (हालाँकि वे **बहुत प्रभावशाली** होते जा रहे हैं – मार्च 2015 तक)।
  - प्री‑1.0 चरण में कई ब्रेकिंग चेंजेज़ हैं

### Python

- **Pros**
  - अल्गोरिद्म का व्यापक संग्रह पहले से ही Python में उपलब्ध है (देखें: scilearnkit, numpy, matplotlib, pil/pillow, आदि)
  - लिखने में बहुत मज़ा आता है! कॉम्प्रिहेंशन और डिकम्पोज़िशन शानदार फीचर हैं और अन्य भाषाओं को बस फुज़ी बनाते हैं!
  - एरेज़, ‘सीक्वेंसेज़’, ट्यूपल्स आदि अपेक्षाकृत सरल हैं

```python
## dummy code: defines a color + pixel-coord -
def pixel(x, y, r, g, b): return dict(x=x, y=y, r=r, g=g, b=b)
## Create a new pixel object and apply to set of vars
x, y, r, g, b = pixel(10, 20, 255, 255, 255)
## Now we can call pixel
```

- **Cons**
  - कष्टकर रूप से, Python 2.x और 3.x असंगत हैं। महान विभाजन अभी भी जारी है, कई सालों बाद।
  - कुछ आवश्यक लाइब्रेरीज़ (जैसे numpy) सभी डेवलपर्स द्वारा पूरी तरह समझी नहीं जातीं।

### Haskell

- **Pros**

- **Cons**
  - सिंटैक्स और पैटर्न को अपनाने में कठिनाई हो सकती है
