## Notizen zu Programmiersprachen

#### Ich bin mir sicher, dass meine verschiedenen Beobachtungen bereits gemacht wurden, aber hier ist meine Liste der interessantesten Sprachen:

### JavaScript

Meine Eine wahre Liebe, äußerst vielseitig und allgegenwärtig - der Allround-Champion mit unglaublich leistungsstarker Ausstattung!
Es ist die #1 am aktivsten/beliebtesten Sprache auf GitHub.com für Jahre in Folge.

Ich hasse es, das zuzugeben, aber für Jahre dachte ich törichterweise nur mit Verachtung und Spott über das, was jetzt meine Lieblingssprache ist.

**ES6** hat meine ~~~Sucht~~~ Liebe nur noch verstärkt. Während reines ES5 immer einen besonderen Platz in meinem Herzen haben wird, fühle ich jedes Mal, wenn ich **ES6** verwende, diesen radioaktiven Spinnenbiss...

Es gab vier Faktoren, die mich in die **ES6-Gruppe** gezogen haben:

1. Es macht Spaß. Ernsthaft. Es gibt messbare Vorteile in Schönheit, Klarheit und Produktivität.

- Subjektive Behauptungen, sagst du? Lass mich dir ein wenig ES6 zeigen:  
- `let expired = users.filter(u => Date.now() > u.trialDate)`  
- Jetzt musst du nicht mehr vortäuschen, dass du weißt, wie man `Object.create` oder `Object.defineProperty` verwendet  
- Siehe Beispiele unten  

1. Seit Juli 2015 ist ES6 nun eine offiziell finalisierte Norm!  
1. Unterstützung ist effektiv 100 %\*! ... Okay, BabelJS ist notwendig, um deinen Code so zu patchen, dass er ES5-kompatibel ist. Historisch gesehen wurden JavaScript-Transpiler kritisch gesehen. In den letzten Jahren (2014–15) hat sich dies jedoch geändert, da BabelJS zu einem Schlüsselimpulsgeber für Fortschritte in der Sprache geworden ist. Unzählige Unternehmen, darunter Microsoft und Facebook, nutzen es auf einigen der größten Webseiten weltweit.  
1. [Neueste Versionen von Node](https://nodejs.org/en/blog/release/v4.0.0/) enthalten dieselbe V8-JavaScript-Engine wie Chrome v45, es ist v4.5  

#### Beispiele

> Ich zeige Ihnen, was mich endgültig dazu brachte, _anzufangen_, diesen ES6-geschmackvollen Kool-Aid zu trinken.

In meiner jüngsten Erfahrung hilft ES6 Ihnen dabei, Code schneller zu schreiben. Direkt und unumstritten.  
Da der Code kompakter ist, braucht es deutlich weniger geistige Anstrengung, um durchzusehen und zu verstehen, was Ihr alter Code (oder der eines Teamkollegen) tut.

Ich habe regelmäßig KLOC-Einsparungen von etwa 20–50 % beobachtet. Das ist so, als wäre der Code Kate Moss-schlank!

### Rust

##### [Offizielle Website](http://www.rust-lang.org/)

- **Vorteile**

- Stellen Sie sich eine Sprache vor, die so schnell wie C und so mächtig wie Python/C++ ist, ohne die Komplexität/Fallen, die selbst die besten Entwickler fangen.  
  - Tatsächlich würde ich schätzen, dass Rust ungefähr so komplex ist wie die ES6-Spezifikation.  
  - Es enthält eine Menge von Extras:  
    1. Rust übersetzt im Grunde von semi-dynamischer Syntax in **reinen C-Code**!  
    1. Enthält ***alle besten Praktiken*** in C, die Sie wahrscheinlich vermasseln würden, ich ~~eventuell~~ immer tue.  
    - Automatisch erhalten Sie:  
    - Auto Memory Management (kein Bedarf für einen langsamen Garbage Collector!)  
    - Perfekt abgegrenzte Objektbesitz/Sperrung (Mutexting & Kontextwechsel minimiert)  
    - Objekt-Lifetimes (automatisch implementiert\* und automatisch codiert, als hätten Sie jeden Randfall gekannt)  
    - Verhindert praktisch alle Laufzeitfehler (ernsthaft, Ihre Code-Pfade werden explizit: Sie können einfach keinen Code-Pfad übersehen)  
  - Oh ja, es wirft wahrhaftige Sprach-Erweiterbarkeit mit einem sinnvollen 'Macro'-Feature ein.  
    - Need Comprehensions? [Scala style? Done](https://gist.github.com/hanny24/5749688), and [Like Python? Done](https://gist.github.com/JeffBelgum/5e762761cd63c796e803).  
    1.  Zu gut, um wahr zu sein? Nein, es wird besser:  
    - Edge-Indikatoren (github.com-Statistiken) zeigen, dass Rust hochkompetitiv oder sogar Go (Googles heiß-neue Sprache) übertrifft  
      - Etwa 4.000 mehr Sterne als Go (derzeit um 12.200)  
      - Mehr Gesamt-Beiträger (2x! - 1.071 vs. Go's 479)  
      - Mehr Forks (3x! - 2.343 vs. 765)  
      - Anzahl offener Issues, verliert knapp (2.000 vs. 1.730 von Go)  
      - Pull Requests (Rust 70+ vs. Go's 1)  
    - Ich musste die Zahlen dreimal überprüfen.  
  - Andere Bibliotheken sind sehr stabil aufgrund der Konstrukte & Regeln von Rust.  
  - Thread-Modell nutzbar für Sterbliche  

- **Nachteile**  
  - **Web-Frameworks** sind relativ neu, ungetestet und oft dokumentationslos (obwohl sie **immer beeindruckender** werden – Stand März 2015).  
  - Viele vor 1.0-Bruchänderungen  

### Python  

- **Vorteile**  
  - Überwältigend vollständige Auswahl an Algorithmen bereits in Python implementiert (siehe: scilearnkit, numpy, matplotlib, pil/pillow, etc.)  
  - Sehr spaßig zu schreiben! Comprehensions und Dekomposition sind großartige Features und machen andere Sprachen einfach fettig!  
  - Arrays, 'Sequenzen', Tupel usw. sind relativ einfach

```python
## dummy code: defines a color + pixel-coord -
def pixel(x, y, r, g, b): return dict(x=x, y=y, r=r, g=g, b=b)
## Create a new pixel object and apply to set of vars
x, y, r, g, b = pixel(10, 20, 255, 255, 255)
## Now we can call pixel
```

- **Nachteile**  
  - Störend: Python 2.x und 3.x sind inkompatibel. Die Große Spaltung hält, viele Jahre später, weiter an.  
  - Einige essentielle Bibliotheken werden nicht unbedingt von einigen Entwicklern verstanden (numpy)

### Haskell

- **Vorteile**  
  - Sehr belohnend, sobald man genug Syntax gelernt hat, um ausdrucksstarke Muster basierend auf Comprehensions zu erstellen  
  - Man lernt denkintensive Code-Muster – oft teilweise anwendbar auf andere Sprachen  
- **Nachteile**  
  - Syntax & Muster können schwer zu gewöhnen sein
