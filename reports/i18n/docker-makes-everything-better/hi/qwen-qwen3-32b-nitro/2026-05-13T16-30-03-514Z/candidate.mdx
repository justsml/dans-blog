## डॉकर हर चीज़ कर सकता है!\*

> ऐतिहासिक टिप्पणि: यह 2015 के डॉकर उत्साह और स्थानीय-डेव आदतों का एक अंकन है, जो मुख्य रूप से समय के संग्रहालय के रूप में संरक्षित किया गया है। आधुनिकीकरण के लिए शुरुआती बिंदुओं के रूप में इन आदेशों और सुरक्षा नीतियों को लें, उत्पादन मार्गदर्शन नहीं।

अपनी प्रक्रिया में सुधार करें:

1.  अपने पीसी पर आश्रितताओं को खराब करे बिना डेव टूल्स और सर्वरों का परीक्षण करने के लिए लगभग शून्य जोखिम के साथ  
1.  अपने सॉफ्टवेयर का परीक्षण करें  
1.  आपको अधिक आइडेम्पोटेंट और मॉड्यूलर कोड लिखने के लिए मजबूर करता है... (मैं इसे वास्तविक रूप में कैसे प्राप्त करें इसके बारे में एक अनुसरण लेख में लिखूंगा)  

सीखने के लिए नई चीजों की बड़ी मात्रा हो सकती है, **इससे आप निराश न हों** कि शुरुआत करने से बच जाएं।  

### टिप्पणियाँ

- यदि आपको `docker run` कमांड में आप्शन `-d` या `-it` दिखाई देते हैं:  
  _ `-it` या `-i -t` निर्धारित कमांड को इंटरैक्टिव रूप से चलाएगा  
  _ `-d` डॉकर कंटेनर को 'डेमन' के रूप में शुरू करेगा, जिसे पृष्ठभूमि सेवा के रूप में भी जाना जाता है।  

---

### उदाहरण

### nginx

```bash
# नोट: होस्ट-आधारित, साझा फोल्डरों का उपयोग कर रहे हैं  
# (साझा फोल्डरों के साथ VOLUME Dockerfile कमांड के माध्यम से असंभव है)  
sudo docker run --name web01 -d -p 8181:80 \  
  -v $(NGINX_DIR)/etc:/etc/nginx \  
  -v $(NGINX_DIR)/log:/var/log/nginx \  
  -v $(NGINX_DIR)/www:/var/www/html \  
  nginx:latest  

## स्थानीय डेटा, इंस्टेंस के भीतर अलग-अलग  
sudo docker run --name web01 -d -p 8181:80 nginx:latest  

# nodejs  
sudo docker run --name nodejs01 -d -p 3300:3300 -p 4433:4433 nodejs:latest  
```

> क्रेडिट्स: [https://dockerfile.github.io/#/nginx](https://dockerfile.github.io/#/nginx)
