## Docker peut tout faire !\*

> Note historique : il s’agit d’un instantané de 2015 de l’enthousiasme autour de Docker et des habitudes de développement local, conservé principalement comme une capsule temporelle. Considérez les commandes et la posture de sécurité comme des points de départ à moderniser, pas comme des directives de production.

Améliorez votre processus pour :

1.  Tester les outils de développement et les serveurs **avec pratiquement zéro risque** de perturber les dépendances de votre PC  
2.  Tester votre logiciel  
3.  Vous inciter à écrire du code plus idempotent, modulaire… (je détaillerai comment le mettre en pratique dans un article suivant)

Il peut sembler y avoir un volume énorme de nouveautés à apprendre, **ne laissez pas cela vous freiner** dans votre démarrage.

### Notes

- Si vous rencontrez une commande `docker run` avec les options `-d` ou `-it` :
  _ `-it` ou `-i -t` exécute la commande configurée en mode interactif  
  _ `-d` lance le conteneur Docker comme un « daemon », c’est‑à‑dire en arrière‑plan.

### EXEMPLES

### nginx

```bash
# Note: using host-based, shared folders
#(shared folders are not possible with the VOLUME Dockerfile cmd)
sudo docker run --name web01 -d -p 8181:80 \
  -v $(NGINX_DIR)/etc:/etc/nginx \
  -v $(NGINX_DIR)/log:/var/log/nginx \
  -v $(NGINX_DIR)/www:/var/www/html \
  nginx:latest

## Local data, isolated within instance
sudo docker run --name web01 -d -p 8181:80 nginx:latest

# nodejs
sudo docker run --name nodejs01 -d -p 3300:3300 -p 4433:4433 nodejs:latest
```
