## Docker Può Fare Tutto!\*

> Nota storica: questo è uno snapshot del 2015 sull'entusiasmo per Docker e sugli abiti di sviluppo locale, conservato principalmente come capsula del tempo. Tratta i comandi e la postura di sicurezza come punti di partenza da modernizzare, non come guida per ambienti di produzione.

Migliora il tuo processo per:

1.  Testare strumenti e server di sviluppo CON UN RISCHIO QUASI NULLA di rompere le dipendenze sul tuo PC  
1.  Testare il tuo software  
1.  Ti costringe a scrivere codice più idempotente e modulare... (scrivereò come realizzarlo effettivamente in un follow-up)

Potrebbe sembrare un'enorme quantità di nuove cose da imparare, **non permettere che questo ti fermi** dal cominciare.

### Note

- Se vedi un comando `docker run` con le opzioni `-d` o `-it`:  
  _ `-it` o `-i -t` eseguirà il comando configurato in modo interattivo  
  _ `-d` avvierà il contenitore Docker come 'demone' ovvero servizio in background.

### ESEMPI  

### nginx  

```bash
# Nota: utilizzo di cartelle condivise con l'host  
# (le cartelle condivise non sono possibili con il comando VOLUME del Dockerfile)  
sudo docker run --name web01 -d -p 8181:80 \  
  -v $(NGINX_DIR)/etc:/etc/nginx \  
  -v $(NGINX_DIR)/log:/var/log/nginx \  
  -v $(NGINX_DIR)/www:/var/www/html \  
  nginx:latest  

## Dati locali, isolati all'interno dell'istanza  
sudo docker run --name web01 -d -p 8181:80 nginx:latest  

# nodejs  
sudo docker run --name nodejs01 -d -p 3300:3300 -p 4433:4433 nodejs:latest  
```
