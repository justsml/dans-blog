## Impératif vs. Récursif vs. Fonctionnel

> [ En cours de rédaction ]

```javascript
// Impératif : Le plus rapide ( + très simple, pas de nouveaux pointeurs ni d'allocations excessives ) :
function fib(n) {
  var a = 1,
    b = 1,
    c = 0;
  for (var i = 1; i < n - 1; ++i) {
    c = a + b;
    a = b;
    b = c;
  }
  return b;
}

// Récursif : (FIREFOX ou BABELJS uniquement) Définition de fonction ES6 avec
// valeurs par défaut des paramètres utilisées pour définir les valeurs initiales (internes/récursives)
function fib(n, current = 0, a = 1, b = 1, c = 0) {
  current++;
  c = a + b;
  a = b;
  b = c;
  return current >= n ? b : fib(n, current, a, b, c);
}

// Exemple médiocre de manuel - portée de fonction médiocre avec plusieurs valeurs externes mutables
function fib(n) {
  if (!arr) {
    var arr = [1, 1];
    n = n - 2;
  } // Médiocre
  if (n === -1) {
    return [arr[0]];
  }
  if (n === 0) {
    return arr;
  }
  var proc = function() {
    --n;
    arr.push(arr[arr.length - 1] + arr[arr.length - 2]);
    return n === 0 ? arr : proc();
    // Médiocre : fonction récursive interne inutile, indice : les variables utilisées proviennent de la portée de la fonction parente
  };
  var ans = proc();
  return ans[ans.length - 1];
}
```

## Promises : Génial !

```js
// Exemple utilisant les Promises bluebird et ses
var Promise = require("bluebird"),
  fs = Promise.promisifyAll(require("fs")),
  less = Promise.promisifyAll(require("less"));

function writeFileData(data) {
  return fs.writeFileAsync("/tmp/output.css", data);
}
// Bluebird rend quelque chose comme cela peut-être inconfortablement simple et concis :
fs.readFileAsync("./style.less") // Appel de readFile() promisifié
  .then(less.renderAsync) // Transfert à less.render
  .then(writeFileData); // Fonction pour recevoir le contenu CSS (1er paramètre)
```

1.  Bien que les Promises ES6 natives soient excellentes, je préfère la bibliothèque [Bluebird Promise Library](https://github.com/petkaantonov/bluebird/blob/master/API.md), plus robuste.
1.  Bibliothèque ou non, les navigateurs modernes prennent en charge Promise depuis des années.
1.  Les Promises peuvent être utilisées sans schémas complexes - l'usage implicite de `deferred` est préférable.
1.  **$q, c'est nul**, utilisez bluebird, voir ci-dessus.
1.  À noter : les benchmarks de Bluebird sont des tests en cas idéal, gardez-le en tête si vous construisez des chaînes de promises complexes.

## Java vs JavaScript

### Limitation de débit / Debouncing / Throttling

1.  En JavaScript, [David Walsh a implémenté le debounce](http://davidwalsh.name/essential-javascript-functions) en moins de 20 lignes !
1.  En Java, JDebounce, une bibliothèque bien plus complexe, fait ~500+ lignes.
1.  _ Comparaison des deux : _
1.  Le JavaScript est rapide et utilise les fonctions de première classe pour atteindre une simplicité remarquable.
1.  Alors que Java comporte beaucoup plus de pièces mobiles, les annotations sont utilisées pour appliquer le comportement à la compilation, et il y a une tonne de XML, juste pour le fun !
