Sie haben einen KI-Agenten gebaut. Vielleicht ist er sogar gut. Die Prompts sind präzise, das Modell ist schnell und die Antworten fühlen sich natürlich an.  

Doch dann fragt jemand, ob er eine Kundendatei in Salesforce überprüfen soll. Oder die neuesten Jira-Tickets abrufen. Oder die internen Dokumente durchsuchen.  

Und Ihr wunderbarer Agent... kann es einfach nicht.  

Dies ist das Integration-Problem, das jede KI-Plattform irgendwann trifft. Ihr Agent benötigt Hände. Er benötigt Augen in Ihre tatsächlichen Geschäfts-Systeme. Ohne sie laufen Sie nur einen teuren Chatbot.  

Die traditionelle Lösung? Für jeden Dienst, den Sie verbinden möchten, einen benutzerdefinierten API-Wrapper schreiben. Die Dokumentation studieren, die Authentifizierung handhaben, mit den Ratelimits umgehen, beten, dass sie ihre Endpunkte nächsten Monat nicht ändern. Dann wiederholt man das Ganze für den nächsten Dienst. Und den nächsten.  

Der Model Context Protocol ändert diese Gleichung komplett.

## Was MCP tatsächlich löst  

Denken Sie an USB vor USB-C. Sie hatten Mini-USB, Micro-USB, proprietäre Apple-Anschlüsse und eine Schublade voller Kabel, die nur mit bestimmten Geräten funktionierten. USB-C hat nicht nur einen neuen Anschluss hinzugefügt – es hat einen Standard etabliert, der bedeutete, dass jedes Kabel mit jedem Gerät funktionieren konnte.  

MCP tut dasselbe für AI-Tool-Integrationen.  

Statt benutzerdefinierten Code zu schreiben, um Ihren Agenten mit Salesforce, HubSpot, GitHub oder jedem anderen Dienst zu verbinden, implementieren Sie das Protokoll einmal (oder laden Sie einen vorab erstellten Server herunter), und jeder mit MCP-kompatible Agent kann sofort mit ihm kommunizieren.  

Das Protokoll übernimmt die Kommunikationsschicht. Sie definieren einfach, was Ihre Tools tun und welche Daten sie benötigen.

## Einrichten mehrerer Integrationen

Mastra unterstützt MCP natively über seinen [`MCPClient`](https://mastra.ai/docs/mcp/overview). Sie können sowohl lokale Tools (als Kindprozesse) als auch Remote-Dienste (auf eigener Infrastruktur) verbinden.

Hier ist ein realistisches Produktionsbeispiel für die Verbindung zu Google Maps für Routenplanung, einen Wetterdienst und eine lokale Wikipedia-Suche:

```typescript
// src/mastra/mcp/index.ts
import { MCPClient } from '@mastra/mcp';

export const mcpClient = new MCPClient({
  servers: {
    // Lokales Tool (Stdio)
    wikipedia: {
      command: 'npx',
      args: ['-y', 'wikipedia-mcp'],
    },
    // Karten & Navigation (Remote/HTTP)
    googleMaps: {
      url: new URL(process.env.GOOGLE_MAPS_MCP_URL!),
      requestInit: {
        headers: {
          Authorization: `Bearer ${process.env.GOOGLE_MAPS_API_KEY}`,
        },
      },
    },
    // Wetterdienst-Integration
    weather: {
      url: new URL('https://mcp.weatherapi.dev/v1'),
      requestInit: {
        headers: {
          'X-API-Key': process.env.WEATHER_API_KEY!,
        },
      },
    },
  },
});
```

Der Client verwaltet den Verbindungslebenszyklus, behandelt den Prozessstart für lokale Tools und unterhält HTTP-Verbindungen zu Remote-Servern. Sie müssen sich nicht direkt um Sockets oder stdio kümmern.

## Verbinden von Tools mit Agenten

Sobald Ihr MCP-Client konfiguriert ist, ist das Zuweisen dieser Tools zu einem Agenten einfach:

```typescript
// src/mastra/agents/navigation-agent.ts
import { Agent } from '@mastra/core/agent';
import { openai } from '@ai-sdk/openai';
import { mcpClient } from '../mcp';

export const navigationDirectionsAgent = new Agent({
  id: 'navigation-directions-agent',
  name: 'Navigation & Directions Assistant',
  instructions: `You are a helpful navigation assistant that provides route planning and travel advice.
    - Always confirm the start and destination locations
    - Use Google Maps tools to find optimal routes
    - Check weather conditions along the route
    - Provide estimated travel times and suggest alternatives if weather is poor
    - Include relevant details like traffic, road conditions, and points of interest
    - Keep responses clear and actionable`,
  model: openai('gpt-5'),
  tools: await mcpClient.getTools(), // <--- This is the magic line
});
```

Wenn ein Benutzer fragt: *"What's the best route from San Francisco to Lake Tahoe, and should I be worried about weather?"*

Der Agent liest die verfügbaren Tool-Definitionen, erkennt, dass er Zugriff auf Google Maps-Routing- und Wettervorhersage-Tools hat, führt sie mit den richtigen Parametern aus und antwortet mit einer optimalen Route plus aktuellen Wetterbedingungen entlang der Strecke.

Sie haben nicht eine einzige Zeile Google Maps API-Code oder Wetterdienst-Integration geschrieben.

## Pro-Benutzer-Authentifizierung

Ein Sicherheitsfehler, der hier leicht passieren kann: Festcodierung von Anmeldeinformationen.

Wenn Sie einen Google Maps API-Schlüssel in Ihre Umgebungsvariablen eintragen und damit durchstarten, teilen sich alle Benutzer dieselbe Kontingentierung und Ratenbegrenzung. Noch wichtiger: Wenn Sie Dienste nutzen, die Benutzereinstellungen speichern (wie gespeicherte Standorte oder Favoritenrouten), sehen alle Benutzer dieselben Daten. Das funktioniert für Demos gut. In der Produktion ist das ein Risiko.

Mastra löst das Problem, indem Sie dynamisch MCP-Clients mit benutzerspezifischen Anmeldeinformationen erstellen können:

```typescript
async function handleUserRequest(userPrompt: string, userCredentials: UserCreds) {
  // Erstelle einen Client für DIESEN spezifischen Benutzer
  const userMcp = new MCPClient({
    servers: {
      googleMaps: {
        url: new URL(process.env.GOOGLE_MAPS_MCP_URL!),
        requestInit: {
          headers: {
            // Benutzerspezifischer API-Schlüssel oder Token
            Authorization: `Bearer ${userCredentials.mapsApiKey}`,
            'X-User-ID': userCredentials.userId,
          },
        },
      },
    },
  });

  const agent = mastra.getAgent('navigationDirectionsAgent');
  
  // Injiziere Tools zur Laufzeit
  const response = await agent.generate(userPrompt, {
    toolsets: await userMcp.getToolsets(),
  });

  return response;
}
```

Jeder Benutzer erhält seinen eigenen isolierten Toolset mit eigenen API-Kontingenten und Einstellungen. Die gespeicherten Standorte von Benutzer A bleiben privat, der Routenverlauf von Benutzer B ist getrennt. So funktionieren multi-tenant SaaS-Agenten in der Praxis.

## Zusammengesetzte Tools erstellen

Manchmal müssen Sie mehrere MCP-Tools zu einer einzigen Operation kombinieren. Vielleicht möchten Sie eine Route planen, die sowohl Echtzeitverkehr als auch Wetterbedingungen entlang der Strecke berücksichtigt.

Sie können MCP-Tools in benutzerdefinierte Tool-Definitionen einwickeln:

```typescript
export const smartRouteTool = createTool({
  id: 'smart-route-planner',
  description: 'Plans optimal route considering traffic and weather conditions',
  execute: async ({ context, mastra }) => {
    // Get the raw tools
    const tools = await mcpClient.getTools();
    
    // 1. Get base route from Google Maps
    const routeData = await tools.googleMaps_getDirections.execute({ 
      context: { 
        origin: context.origin,
        destination: context.destination 
      } 
    });
    
    // 2. Check weather along the route
    const weatherData = await tools.weather_getForecast.execute({
      context: { coordinates: routeData.waypoints }
    });
    
    // 3. Return enhanced route with weather warnings
    return { 
      ...routeData, 
      weatherAlerts: weatherData.alerts,
      recommendation: weatherData.severe ? 'Consider delaying trip' : 'Safe to travel'
    };
  },
});
```

Dies gibt Ihnen feingranulare Kontrolle darüber, wie Tools interagieren, während Sie den MCP-Protokoll-Standard für die schwere Arbeit nutzen.

## Wo dies hinführt

Das Schreiben von benutzerdefinierten API-Clients für jeden Service, mit dem Ihr AI-Agent kommunizieren muss, war nie nachhaltig. Es skaliert schlecht, bricht häufig und bindet Ihre Plattform an spezifische Implementierungen.

MCP löst nicht jedes Integrationsproblem – Authentifizierung bleibt komplex, Ratenbegrenzung ist weiterhin relevant, und nicht jeder Service verfügt bereits über einen MCP-Server. Aber es legt eine Grundlage, die das Aufbau von Agent-Plattformen erheblich weniger schmerzhaft macht.

Wenn Sie ein AI-System entwerfen, das mit externen Diensten interagieren muss, lohnt es sich wahrscheinlich, MCP zu verstehen.
