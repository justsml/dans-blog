<p class="inset">闭包是每个JavaScript开发者自以为理解的概念之一——直到某天突然发现其实并不理解。这套测验将测试你真正的掌握边界。</p>

闭包出现在事件处理程序、工厂函数、React钩子、模块模式以及几乎所有其他场景。它们并非异类概念，而是JavaScript的编织基底。

    每次调用 `makeCounter()` 都会创建一个**新的执行上下文**，拥有独立的 `count` 变量。`a` 和 `b` 分别闭包捕获了*不同的* `count` 绑定。它们不共享状态。

    结果：`1, 1`

    这就是为什么闭包被用来构建私有状态——每个"实例"都是独立的。

>
  <slot name="question">
  <div className="question">
    100ms后会输出什么？
    ```javascript
        for (var i = 0; i < 5; i++) {
          setTimeout(() => console.log(i), 100);
        }
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    `var`是函数作用域。**只有一个** `i` —— 所有五个回调都闭包在同一个绑定上。当定时器触发时，循环已经完成，此时`i`的值是`5`。

    五个回调。一个`i`。值：`5`。输出五次。

    解决方案是使用`let`（块作用域，每次迭代创建新绑定）或用IIFE包裹来捕获当前值：
    ```javascript
        for (let i = 0; i < 5; i++) {
          setTimeout(() => console.log(i), 100); // 0, 1, 2, 3, 4 ✓
        }
    ```
  </div>
  </slot>
</Challenge>

    ```javascript
        { let i = 0; setTimeout(() => console.log(i), 100) }
        { let i = 1; setTimeout(() => console.log(i), 100) }
    ```

  <slot name="question">
  <div className="question">
    这段代码会输出什么？
    ```javascript
        function wrap() {
          const obj = { x: 1 };
          const getObj = () => obj;
          obj.x = 99;
          return getObj;
        }
        const get = wrap();
        console.log(get(), get());
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    闭包捕获的是 **绑定** `obj`，而不是对象的快照。创建闭包后对象本身被修改了——但 `obj` 仍然指向相同的引用。两次调用返回的是同一个被修改后的对象：`{ x: 99 }`。

    这会让误以为闭包会"冻结"值的开发者掉进陷阱。它们不会。它们捕获的是 **引用**。
  </div>
  </slot>

  client:visible={{rootMargin: "150px"}}
  index={5}
  group="高级"
  title="React中的陈旧闭包"
  options={[
    {text: '每次点击按钮时都会记录当前的计数'},
    {text: '始终记录0', isAnswer: true},
    {text: '记录NaN'},
    {text: '这不会编译'},
  ]}
>
  <slot name="question">
  <div className="question">
    当按钮被多次点击时，这个React组件会表现出什么行为？
    ```jsx
        function Counter() {
          const [count, setCount] = useState(0);

          useEffect(() => {
            const id = setInterval(() => {
              console.log('count is', count);
            }, 1000);
            return () => clearInterval(id);
          }, []); // empty deps

          return <button onClick={() => setCount(c => c + 1)}>
            Click ({count})
          </button>;
        }
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    经典的陈旧闭包问题。`useEffect`只运行一次（空依赖数组），创建的间隔函数在组件挂载时捕获了`count`的值：`0`。无论点击多少次，间隔函数的闭包始终无法看到更新后的值。

    按钮标签会正确更新——这是新的渲染。但`console.log`永远被困在`0`。

    修复方法是将`count`添加到依赖数组中（重新创建间隔）或使用ref：
    ```jsx
        const countRef = useRef(count);
        useEffect(() => { countRef.current = count; }, [count]);
        // then use countRef.current inside the interval
    ```
  </div>
  </slot>
</Challenge>

    等等——仔细读。实际上会发生什么？
    ```javascript
        const bank = (() => {
          let balance = 0;
          return {
            deposit: (n) => (balance += n),
            withdraw: (n) => (balance -= n),
            getBalance: () => balance,
          };
        })();

        bank.deposit(100);
        bank.withdraw(40);
        console.log(bank.getBalance());
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    答案是`60`。

    这是**模块模式**——一个返回闭包对象的立即执行函数表达式（IIFE）。这三个方法共享同一个`balance`绑定。这个变量是真正的私有属性：你无法从外部访问`balance`。
    ```javascript
        console.log(bank.balance); // undefined
    ```
    在class出现之前，这就是JavaScript中*最常用*的封装状态的方式。你仍然会在遗留代码库和工具库中看到这种模式。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={7}
  group="专家级"
  title="闭包内存泄漏"
  options={[
    {text: '它记录data数组的长度'},
    {text: '它创建了内存泄漏', isAnswer: true},
    {text: '它抛出RangeError异常'},
    {text: '没有问题——这很正常'},
  ]}
>
  <slot name="question">
  <div className="question">
    这段代码的问题是什么？
    ```javascript
        function attachHandler(element) {
          const data = new Array(100_000).fill('🔥');
          element.addEventListener('click', () => {
            console.log('clicked!', data.length);
          });
        }
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    事件监听器闭包捕获了`data`，所以只要监听器存在，该数组就保持可访问状态。如果`element`长期存在且监听器从未被移除，这个大数组也会被保留下来。

    `element`在DOM中。监听器持有`data`。GC无法回收它。

    修复方案：除非确实需要，在长期存在的监听器中不要捕获大数据，当所属UI被销毁时移除监听器：
    ```javascript
        function attachHandler(element) {
          const data = new Array(100_000).fill('🔥');
          // do something with data...

          const onClick = () => {
            console.log('clicked!');
          };

          element.addEventListener('click', onClick);
          return () => element.removeEventListener('click', onClick);
        }
    ```
    现代V8引擎在这方面比以前有所改进。但在长期存在的单页应用（SPA）中，如果存在大量事件监听器，这种模式仍然会引发问题。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={8}
  group="专家级"
  title="this 的陷阱"
  options={[
    {text: 'Dan, Dan'},
    {text: 'Dan, undefined'},
    {text: 'undefined, Dan'},
    {text: 'undefined, undefined', isAnswer: true},
  ]}
>
  <slot name="question">
  <div className="question">
    这段代码会输出什么？
    ```javascript
        const user = {
          name: 'Dan',
          greetArrow: () => console.log(this?.name),
          greetRegular() {
            const inner = () => console.log(this?.name);
            inner();
          },
        };

        user.greetArrow();
        const fn = user.greetRegular;
        fn();
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    两个微妙的陷阱：

    **`greetArrow`**：箭头函数没有自己的 `this`。它从封闭的词法作用域捕获 `this`——在这里是模块/全局作用域，而不是 `user`。因此 `this?.name` 是 `undefined`。

    **`fn()`**：`greetRegular` 没有接收者调用（`fn()` 而不是 `user.greetRegular()`）。在严格模式下，`this` 是 `undefined`。在非严格模式下，它是全局对象。无论如何，`this?.name` 都是 `undefined`。

    如果你直接调用 `user.greetRegular()`，你会得到 `'Dan'`——因为此时 `this` 绑定到 `user`。

    结果：`undefined, undefined`。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={9}
  group="附加题"
  title="通过闭包实现记忆化"
  options={[
    {text: '调用 expensive() 3 次'},
    {text: '调用 expensive() 1 次', isAnswer: true},
    {text: '调用 expensive() 2 次'},
    {text: '抛出：超出最大调用栈'},
  ]}
>
  <slot name="question">
  <div className="question">
    `expensive()` 实际被调用多少次？
    ```javascript
        function memoize(fn) {
          const cache = new Map();
          return (arg) => {
            if (cache.has(arg)) return cache.get(arg);
            const result = fn(arg);
            cache.set(arg, result);
            return result;
          };
        }

        let calls = 0;
        const expensive = (n) => { calls++; return n * 2; };
        const fast = memoize(expensive);

        fast(5);
        fast(5);
        fast(5);
        console.log(calls);
    ```
  </div>
  </slot>

  <slot name='explanation'>
  <div className="explanation">
    记忆化函数闭包了 `cache`。第一次调用 (`fast(5)`) 未命中缓存，会调用 `expensive()` 并存储结果。第二、第三次调用命中缓存直接返回。

    `calls` 的值为 `1`。

    这正是闭包在实际中发挥作用的典型案例。`cache` 是私有的（外部无法访问 `memoize` 内部）、持久的（只要 `fast` 存在就会一直存在），并且在所有对 `fast` 的调用之间共享。
  </div>
  </slot>
</Challenge>
