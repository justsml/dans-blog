<p class="inset">闭包是每个 JavaScript 开发者*以为*自己懂的概念——直到真正碰到它们时才发现并非如此。这个测验会帮你找出自己到底掌握到了哪一步。</p>

闭包出现在事件处理函数、工厂函数、React hook、模块模式以及几乎所有其他地方。它们并不神秘，正是 JavaScript 结构的底层纤维。

    ```javascript
        function makeCounter() {

    ```
  </div>

    `tick` 闭包捕获了 `count`。每次调用都会递增并返回*同一个* `count` 变量。三次调用的结果是 `1, 2, 3`。

    关键字：*同一个*。`count` 并没有被复制进闭包，闭包持有对变量绑定的**实时引用**。这就是核心所在。

  <div className="question">
    这段代码会输出什么？
    ```javascript
        function makeCounter() {
          let count = 0;
          return () => ++count;
        }
        const a = makeCounter();
        const b = makeCounter();
        console.log(a(), b());
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    每次调用 `makeCounter()` 都会创建一个 **新的执行上下文**，并拥有各自全新的 `count`。`a` 和 `b` 闭包引用的是*不同的* `count` 绑定，它们之间不共享状态。

    结果：`1, 1`

    这正是闭包用于构建私有状态的原因——每个“实例”都是独立的。

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={2}
  group="经典陷阱"
  title="循环中的 var"
  options={[
    {text: '0, 1, 2, 3, 4'},
    {text: '5, 5, 5, 5, 5', isAnswer: true},
    {text: '1, 2, 3, 4, 5'},
    {text: 'undefined x5'},
  ]}
>
  <slot name="question">
  <div className="question">
    100 毫秒后会输出什么？
    ```javascript
        for (var i = 0; i < 5; i++) {
          setTimeout(() => console.log(i), 100);
        }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    `var` 是函数作用域的。恰好只有 **一个** `i` —— 所有五个回调都闭包到同一个绑定。等定时器触发时，循环已经结束，`i` 为 `5`。

    五个回调。一个 `i`。值：`5`。输出五次。

    解决办法是使用 `let`（块作用域，每次迭代都有新绑定）或用 IIFE 包裹以捕获当前值：
    ```javascript
        for (let i = 0; i < 5; i++) {
          setTimeout(() => console.log(i), 100); // 0, 1, 2, 3, 4 ✓
        }
    ```
  </div>
  </slot>
</Challenge>

>
  <slot name="question">
  <div className="question">
    相同的循环，不同的关键字。会输出什么？
    ```javascript
        for (let i = 0; i < 5; i++) {
          setTimeout(() => console.log(i), 100);
        }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    `let` 在 `for` 循环中很特殊。规范说明：每次迭代都会获得自己的 **新绑定** `i`。好像循环暗中这样做：
    ```javascript
        { let i = 0; setTimeout(() => console.log(i), 100) }
        { let i = 1; setTimeout(() => console.log(i), 100) }
        // ...
    ```
    于是这五个回调各自闭包了五个不同的变量。结果是：`0, 1, 2, 3, 4`。

>
  <slot name="question">
  <div className="question">
    这会输出什么？
    ```javascript
        function wrap() {
          const obj = { x: 1 };
          const getObj = () => obj;
          obj.x = 99;
          return getObj;
        }
        const get = wrap();
        console.log(get(), get());
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    闭包捕获的是 **绑定** `obj`，而不是对象的快照。对象在闭包创建后被修改——但 `obj` 仍指向同一引用。两次调用返回相同的已变更对象：`{ x: 99 }`。

    这会让那些以为闭包会“冻结”值的人吃亏。事实并非如此，闭包捕获的是 **引用**。
  </div>
  </slot>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={5}
  group="高级"
  title="React 中的陈旧闭包"
  options={[
    {text: '在每次点击时记录当前计数'},
    {text: '始终记录 0', isAnswer: true},
    {text: '记录 NaN'},
    {text: '这段代码无法编译'},
  ]}
>
  <slot name="question">
  <div className="question">
    当多次点击按钮时，这个 React 组件会表现出什么行为？
    ```jsx
        function Counter() {
          const [count, setCount] = useState(0);

          useEffect(() => {
            const id = setInterval(() => {
              console.log('count is', count);
            }, 1000);
            return () => clearInterval(id);
          }, []); // empty deps

          return <button onClick={() => setCount(c => c + 1)}>
            Click ({count})
          </button>;
        }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    经典的陈旧闭包。`useEffect` 只运行一次（空依赖数组），并创建一个 interval，捕获当时挂载时的 `count` 值：`0`。无论点击多少次，interval 的闭包都看不到更新后的值。

    按钮标签会正确更新——这是一次全新的渲染。但 `console.log` 永远被卡在 `0`。

    通过将 `count` 加入依赖数组（重新创建 interval）或使用 ref 来修复：
    ```jsx
        const countRef = useRef(count);
        useEffect(() => { countRef.current = count; }, [count]);
        // then use countRef.current inside the interval
    ```
  </div>
  </slot>
</Challenge>

  client:visible={{rootMargin: "150px"}}
  index={6}
  group="高级"
  title="模块模式"
  options={[
    {text: '60', isAnswer: true},
    {text: 'TypeError: balance is not defined'},
    {text: '-40'},
    {text: '100'},
  ]}
>
  <slot name="question">
  <div className="question">
    等等——仔细阅读。实际会发生什么？
    ```javascript
        const bank = (() => {
          let balance = 0;
          return {
            deposit: (n) => (balance += n),
            withdraw: (n) => (balance -= n),
            getBalance: () => balance,
          };
        })();

        bank.deposit(100);
        bank.withdraw(40);
        console.log(bank.getBalance());
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    答案是 `60`。

    这就是 **模块模式** —— 一个返回闭包对象的 IIFE。所有三个方法都闭包了同一个 `balance` 绑定。它是真正私有的：你无法从外部访问 `balance`。
    ```javascript
        console.log(bank.balance); // undefined
    ```
    在 class 出现之前，这就是 *唯一* 在 JavaScript 中封装状态的方式。你仍然会在旧代码库和工具库中看到它。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={7}
  group="专家"
  title="闭包内存泄漏"
  options={[
    {text: '它会记录 data 数组的长度'},
    {text: '它会导致内存泄漏', isAnswer: true},
    {text: '它会抛出 RangeError'},
    {text: '没有异常 — 这没问题'},
  ]}
>
  <slot name="question">
  <div className="question">
    这段代码有什么问题？
    ```javascript
        function attachHandler(element) {
          const data = new Array(100_000).fill('🔥');
          element.addEventListener('click', () => {
            console.log('clicked!', data.length);
          });
        }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    事件监听器闭包了 `data`，因此只要监听器仍然存在，数组就会保持可达。如果 `element` 生命周期很长且监听器从未移除，那个大数组也会被保留。

    `element` 位于 DOM 中。监听器持有 `data`，垃圾回收无法回收它。

    解决方案：除非真的需要，否则不要在长期存在的监听器中捕获大数据，并在所属 UI 被销毁时移除监听器：
    ```javascript
        function attachHandler(element) {
          const data = new Array(100_000).fill('🔥');
          // do something with data...

          const onClick = () => {
            console.log('clicked!');
          };

          element.addEventListener('click', onClick);
          return () => element.removeEventListener('click', onClick);
        }
    ```
    现代 V8 在这方面比以前更好。但在长期运行的 SPA 中拥有大量事件监听器时，这种模式仍会让人头疼。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={8}
  group="专家"
  title="this 陷阱"
  options={[
    {text: 'Dan, Dan'},
    {text: 'Dan, undefined'},
    {text: 'undefined, Dan'},
    {text: 'undefined, undefined', isAnswer: true},
  ]}
>
  <slot name="question">
  <div className="question">
    这会输出什么？
    ```javascript
        const user = {
          name: 'Dan',
          greetArrow: () => console.log(this?.name),
          greetRegular() {
            const inner = () => console.log(this?.name);
            inner();
          },
        };

        user.greetArrow();
        const fn = user.greetRegular;
        fn();
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    两个微妙的陷阱：

    **`greetArrow`**：箭头函数没有自己的 `this`。它们捕获来自外层词法作用域的 `this`——这里是模块/全局作用域，而不是 `user`。所以 `this?.name` 为 `undefined`。

    **`fn()`**：`greetRegular` 在没有接收者的情况下被调用（`fn()` 而不是 `user.greetRegular()`）。在严格模式下，`this` 为 `undefined`。在宽松模式下，它是全局对象。无论哪种情况，`this?.name` 都是 `undefined`。

    如果直接调用 `user.greetRegular()`，会得到 `'Dan'`——因为此时 `this` 绑定到 `user`。

    结果：`undefined, undefined`。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={9}
  group="奖励"
  title="通过闭包实现记忆化"
  options={[
    {text: '调用 `expensive()` 3 次'},
    {text: '调用 `expensive()` 1 次', isAnswer: true},
    {text: '调用 `expensive()` 2 次'},
    {text: '抛出错误：Maximum call stack exceeded'},
  ]}
>
  <slot name="question">
  <div className="question">
    实际调用了多少次 `expensive()`？
    ```javascript
        function memoize(fn) {
          const cache = new Map();
          return (arg) => {
            if (cache.has(arg)) return cache.get(arg);
            const result = fn(arg);
            cache.set(arg, result);
            return result;
          };
        }

        let calls = 0;
        const expensive = (n) => { calls++; return n * 2; };
        const fast = memoize(expensive);

        fast(5);
        fast(5);
        fast(5);
        console.log(calls);
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    记忆化函数闭包了 `cache`。第一次调用 (`fast(5)`) 未命中缓存，调用 `expensive` 并存储结果。第二次和第三次调用命中缓存，直接返回。

    `calls` 为 `1`。

    这正是闭包在实际工作：`cache` 是私有的（无法从 `memoize` 外部访问），持久的（只要 `fast` 存在就一直存在），并且在所有对 `fast` 的调用之间共享。
  </div>
  </slot>
</Challenge>
