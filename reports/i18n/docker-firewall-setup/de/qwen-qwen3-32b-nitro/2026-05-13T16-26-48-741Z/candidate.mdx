## Einrichten der Firewall des Docker-Hosts

1. Es wird ein Debian-/Ubuntu-Server angenommen  
1. Für die Ausführung auf einem Docker-Host-Server konzipiert  

### Installieren der Voraussetzungen  

~~~sh
# Ultimative Firewall erforderlich
apt-get update && apt-get install -y ufw nmap curl
~~~

### Ihre internen und externen IP-Adressen abrufen  

~~~sh
# IP-Adressen abrufen, einfache Ausgabe:
hostname --all-ip-addresses
~~~

# ODER verwenden Sie das ip-Tool, Beispiel:
ip addr
~~~

### Firewall (UFW) Setup - Beispielfälle

~~~sh
ufw logging on # on=low - medium ist besser für Diagnose
ufw logging medium
# Zunächst alles blockieren
ufw default deny incoming

# ERFORDERLICH: WÄHLEN SIE EINE DER FOLGENDEN STANDARD-REGELN FÜR AUSGANGSVERKEHR:
ufw default deny outgoing
ufw default allow outgoing

# Erlauben und protokollieren Sie alle neuen SSH-Verbindungen,
ufw allow log proto tcp from any to any port 22
## Erlauben Sie HTTP-Verkehr (ohne explizites Logging)
ufw allow out on docker0 53/udp to 172.17.0.1/16
ufw allow out on eth0 to any port 53
ufw allow out on eth0 from 0.0.0.0/0 to any port 80 proto tcp
ufw allow out on eth0 from 0.0.0.0/0 to any port 443 proto tcp

# Verbose: ufw allow proto tcp from any to any port 80
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow log 22/tcp
ufw limit ssh # Grundlegende Rate-Limiting für 4 SSH-Brute-Force-Mitigation
~~~

# Setzen Sie Ihre externe IP
export EXTERNAL_IP=123.123.123.123
# Aktualisieren Sie die Docker-IP bei Bedarf
export DOCKER_IP=172.17.42.1
# Weiterleiten von TCP-8080-Traffik zu Docker-App
ufw allow proto tcp from $EXTERNAL_IP port 8080 to $DOCKER_IP port 3000
~~~

## Firewall aktivieren / starten

> Achtung: SSH-Port nicht sperren (Standard ist 22)

~~~sh
ufw --force enable

ufw reset
~~~

### Testen Sie Ihre Firewall

> Wichtig: VERWENDEN SIE EINE EXTERNE IP-ADRESSE/LOCATION

~~~sh
# Abhängigkeiten prüfen
apt-get update && apt-get install -y nmap

# Ziel-Host festlegen
export TARGET_HOST=123.123.123.123

# Beispiel-Scan-Befehle:
# Schnelle Überprüfung offener Ports
nmap -p 1-10240,27017 -T5 $TARGET_HOST
# Ausführlicher Scan
nmap -p 1-10240,27017 --open -v -APN $TARGET_HOST
# Dienst-Inspektion
nmap -p 1-10240,27017 -O --osscan-guess $TARGET_HOST
~~~

> FERTIG! Jetzt sollten Sie NUR die Ports sehen, die Sie konfiguriert haben!
