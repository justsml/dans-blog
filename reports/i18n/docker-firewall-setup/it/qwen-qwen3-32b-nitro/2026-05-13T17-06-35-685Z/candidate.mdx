## Configurazione del firewall del host Docker

1. Si presuppone l'uso di Debian/Ubuntu Server  
1. Progettato per essere eseguito su un server host Docker  

### Installazione dei requisiti  

~~~sh
# È necessario il firewall definitivo
apt-get update && apt-get install -y ufw nmap curl
~~~  

### Ottieni gli indirizzi IP interni ed esterni  

~~~sh
# Ottieni gli indirizzi IP, output semplice:
hostname --all-ip-addresses
~~~

# O usa lo strumento ip, esempio:  
ip addr  
~~~  

### Firewall (UFW) Setup - Esempi di comandi  

~~~sh  
ufw logging on # on=low - medium potrebbe essere migliore per la diagnostica  
ufw logging medium  
# Prima, blocca tutto  
ufw default deny incoming  

# OBIETTIVO: SCEGLI *UNO* TRA I SEGUENTI COMANDI PER LE REGOLE DI USCITA PREDEFINITE:  
ufw default deny outgoing  
ufw default allow outgoing  

# Permette e registra le nuove connessioni SSH,  
ufw allow log proto tcp from any to any port 22  
## Permette il traffico HTTP (senza registrazione esplicita)  
ufw allow out on docker0 53/udp to 172.17.0.1/16  
ufw allow out on eth0 to any port 53  
ufw allow out on eth0 from 0.0.0.0/0 to any port 80 proto tcp  
ufw allow out on eth0 from 0.0.0.0/0 to any port 443 proto tcp  

# Dettagliato: ufw allow proto tcp from any to any port 80  
ufw allow 80/tcp  
ufw allow 443/tcp  
ufw allow log 22/tcp  
ufw limit ssh # Limita la frequenza per mitigare attacchi SSH con forza bruta  
~~~

# Imposta la tua IP esterna  
export EXTERNAL_IP=123.123.123.123  
# Aggiorna l'IP di Docker se necessario  
export DOCKER_IP=172.17.42.1  
# Inoltra il traffico TCP 8080 verso l'app Dockerizzata  
ufw allow proto tcp from $EXTERNAL_IP port 8080 to $DOCKER_IP port 3000  
~~~  

## Abilita / Avvia il Firewall  

> Fai attenzione, non bloccare la tua porta SSH (sshd di default è 22)  

~~~sh  
ufw --force enable  

ufw reset  
~~~

### Testa il tuo Firewall

> Importante: USA UN INDIRIZZO/IP REMOTO

~~~sh
# Verifica le dipendenze
apt-get update && apt-get install -y nmap

# Imposta l'obiettivo dello scan
export TARGET_HOST=123.123.123.123

# Comandi di scansione di esempio:
# Verifica rapida delle porte aperte
nmap -p 1-10240,27017 -T5 $TARGET_HOST
# Scansione completa
nmap -p 1-10240,27017 --open -v -APN $TARGET_HOST
# Ispezione servizi
nmap -p 1-10240,27017 -O --osscan-guess $TARGET_HOST
~~~

> Finito! Ora dovresti vedere SOLO le porte che hai configurato!
