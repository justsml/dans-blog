## Настройка брандмауэра хоста Docker

1. Предполагается использование сервера Debian/Ubuntu  
1. Предназначено для запуска на сервере хоста Docker  

### Установка необходимых пакетов  

~~~sh
# Требуется брандмауэр
apt-get update && apt-get install -y ufw nmap curl
~~~  

### Получите свои внутренние и внешние IP-адреса  

~~~sh
# Получите свои IP-адреса, простой вывод:
hostname --all-ip-addresses
~~~

# ИЛИ используйте ip инструмент, пример:  
ip addr  
~~~  

### Настройка брандмауэра (UFW) - Пример команд  

~~~sh  
ufw logging on # вкл=низкий - средний может быть лучше для диагностики  
ufw logging medium  
# Сначала, блокируем всё  
ufw default deny incoming  

# ВАЖНО: ВЫБЕРИТЕ *ОДИН* ИЗ СЛЕДУЮЩИХ ПРАВИЛ ПО УМОЛЧАНИЮ ДЛЯ ВЫХОДЯЩЕГО ТРАФИКА:  
ufw default deny outgoing  
ufw default allow outgoing  

# Разрешить и логировать все новые ssh соединения,  
ufw allow log proto tcp from any to any port 22  
## Разрешить http трафик (без явного логирования)  
ufw allow out on docker0 53/udp to 172.17.0.1/16  
ufw allow out on eth0 to any port 53  
ufw allow out on eth0 from 0.0.0.0/0 to any port 80 proto tcp  
ufw allow out on eth0 from 0.0.0.0/0 to any port 443 proto tcp  

# Подробно: ufw allow proto tcp from any to any port 80  
ufw allow 80/tcp  
ufw allow 443/tcp  
ufw allow log 22/tcp  
ufw limit ssh # Основной лимит частоты для предотвращения брутфорса SSH  
~~~

# Установите ваш внешний IP  
export EXTERNAL_IP=123.123.123.123  
# Обновите IP Docker, если необходимо  
export DOCKER_IP=172.17.42.1  
# Перенаправьте трафик TCP 8080 на приложение в Docker  
ufw allow proto tcp from $EXTERNAL_IP port 8080 to $DOCKER_IP port 3000  
~~~  

## Включение / Запуск брандмауэра  

> Будьте осторожны, не заблокируйте порт SSH (sshd по умолчанию 22)  

~~~sh  
ufw --force enable  

ufw reset  
~~~

### Тестирование брандмауэра  

> Важно: ИСПОЛЬЗУЙТЕ УДАЛЕННЫЙ IP-АДРЕС/ЛОКАЦИЮ  

~~~sh  
# Проверка зависимости  
apt-get update && apt-get install -y nmap  

# Установка цели сканирования  
export TARGET_HOST=123.123.123.123  

# Примеры команд сканирования:  
# Быстрая проверка открытых портов  
nmap -p 1-10240,27017 -T5 $TARGET_HOST  
# Полное сканирование  
nmap -p 1-10240,27017 --open -v -APN $TARGET_HOST  
# Анализ сервисов  
nmap -p 1-10240,27017 -O --osscan-guess $TARGET_HOST  
~~~  

> Готово! Теперь вы должны видеть ТОЛЬКО те порты, которые настроили!
