    Qu'est-ce qui se passe ici ?
    ```bash
        str="Bark bark"
        echo ${str/bark/meow}
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La syntaxe `${var/pattern/replacement}` remplace la première occurrence de `pattern` par `replacement`. Ici, la sortie est :
    ```plaintext
        Bark meow
    ```
    Elle est sensible à la casse. Pour gérer à la fois `bark` et `Bark`, utilisez un motif tel que `${var/[Bb]ark/Bark}` ou normalisez la chaîne avant le remplacement.

    Pour remplacer toutes les occurrences, utilisez `${var//pattern/replacement}`.

    Pour remplacer depuis le début de la chaîne, utilisez `${var/#pattern/replacement}`.

    Pour remplacer depuis la fin de la chaîne, utilisez `${var/%pattern/replacement}`.
  </div>
  </slot>

  <div className="explanation">
    Avez-vous repéré l'espace manquant avant l'accolade fermante ?

    Bash est très exigeant sur ce point : les espaces sont obligatoires à l'intérieur des expressions entre crochets.

    L'espace manquant fait que la commande `[` ne voit pas de `]` de fermeture, Bash affiche un diagnostic, traite le test comme échoué, et passe à la branche `else`.

    La syntaxe correcte est :
    ```bash
        if [ -e example.txt ]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
    Note : Les accolades doubles `[[ ]]` sont **recommandées** pour les expressions conditionnelles. [Voir BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="Pièges"
  title="Substitution de commande"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    Quelle option exécutera la commande `ls -l` et renverra la sortie ?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La syntaxe `$(ls -l)` exécute la commande entre **parenthèses** et substitue la sortie. Par exemple :
    ```bash
        echo "Today is $(date +%F)"
        # => Today is 2029-12-31
    ```
    L'option 1 utilise des apostrophes simples `'`, **pas des backticks.** Cela empêche l'expansion, donc `'$(date +%F)'` afficherait simplement la chaîne littérale `$(date +%F)`.

    Bien que l'utilisation des backticks (`` `ls -l` ``) pour l'exécution de commandes soit encore supportée, c'est désormais un anti-modèle (dans certains contextes). La plupart recommandent d'utiliser `$(command)` pour une meilleure lisibilité et compatibilité avec différentes versions de shells.

    Les accolades `${}` sont utilisées pour l'expansion de variables, pas pour la substitution de commandes.

    Le caractère `%` n'est pas utilisé pour la substitution de commandes.

  </slot>
