    Note : `$name` est utilisé pour **référencer** ou lire la valeur d’une variable.

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={4}
  group="Remplacement de sous-chaînes"
  title="Remplacer une sous-chaîne"
  options={[
    {text: 'meow meow'},
    {text: 'Meow meow'},
    {text: 'Bark meow', isAnswer: true},
    {text: 'Bark bark'},
    {text: 'Error'},
  ]}
>
  <slot name="question">
  <div className="question">
    Que se passe-t-il ici ?
    ```bash
        str="Bark bark"
        echo ${str/bark/meow}
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La syntaxe `${var/pattern/replacement}` remplace la première occurrence de `pattern` par `replacement`. Ici, la sortie est :
    ```plaintext
        Bark meow
    ```
    Il est sensible à la casse. Pour gérer à la fois `bark` et `Bark`, utilisez un motif tel que `${var/[Bb]ark/Bark}` ou normalisez la chaîne avant le remplacement.

    Pour remplacer toutes les occurrences, utilisez `${var//pattern/replacement}`.

    Pour remplacer depuis le début de la chaîne, utilisez `${var/#pattern/replacement}`.

    Pour remplacer depuis la fin de la chaîne, utilisez `${var/%pattern/replacement}`.
  </div>
  </slot>
</Challenge>

    Que produit ce script si le fichier `cats.txt` EXISTE ?
    ```bash
        if [ -e cats.txt]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Avez‑vous remarqué l’espace manquant avant la parenthèse fermante ?

    Bash est assez pointilleux ici : des espaces sont requis à l’intérieur des expressions entre crochets.

    Parce que l’espace manquant fait que la commande `[` ne voit pas de `]` de fermeture, Bash affiche un diagnostic, considère le test comme échoué et passe à la branche `else`.

    La syntaxe correcte est :
    ```bash
        if [ -e example.txt ]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
    Remarque : les doubles crochets `[[ ]]` sont **recommandés** pour les expressions conditionnelles. [Voir BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)
  </div>
  </slot>

    La syntaxe `(( ))` effectue des calculs entiers dans Bash.

    ```bash
        if (( 2 > 1 )); then
          echo "2 is greater than 1"
        fi
    ```
    Pour l'arithmétique à virgule flottante, envisagez d'utiliser [`bc`](https://www.gnu.org/software/bc/manual/html_mono/bc.html) ou [`awk`](https://www.gnu.org/software/gawk/manual/gawk.html).
