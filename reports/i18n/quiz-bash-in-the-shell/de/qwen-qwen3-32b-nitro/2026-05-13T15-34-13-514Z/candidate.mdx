    ```plaintext

    ```
    Dies ist fallunterscheidend. Um sowohl `bark` als auch `Bark` zu behandeln, verwenden Sie ein Muster wie `${var/[Bb]ark/Bark}` oder normalisieren Sie die Zeichenkette vor der Ersetzung.

    Um am Anfang der Zeichenkette zu ersetzen, verwenden Sie `${var/#pattern/replacement}`.

    Um am Ende der Zeichenkette zu ersetzen, verwenden Sie `${var/%pattern/replacement}`.

<Challenge
  client:visible={{rootMargin: "150px"}}

  group="Bedingte Anweisungen"
  title="Grundlegende If-Else"
  options={[

  ]}
>
  <slot name="question">
  <div className="question">
    Was gibt dieses Skript aus, wenn die Datei `cats.txt` VORHANDEN ist?
    ```bash
        if [ -e cats.txt]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Haben Sie den fehlenden Leerzeichen vor der schließenden Klammer bemerkt?

    Bash ist hier sehr empfindlich: Leerzeichen sind innerhalb der Ausdrücke erforderlich.

    Der fehlende Leerzeichen führt dazu, dass der [ Befehl keine schließende ] sieht, Bash eine Diagnose ausgibt, den Test als fehlgeschlagen betrachtet und zum else-Zweig übergeht.

    Die korrekte Syntax ist:
    ```bash
        if [ -e example.txt ]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
    Hinweis: Doppelte Klammern [[ ]] werden **empfohlen**, um Bedingungsausdrücke zu verwenden. [Siehe BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="Fallen"
  title="Befehlsersetzung"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    Welche Option führt den Befehl `ls -l` aus und gibt die Ausgabe zurück?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Die Syntax `$(ls -l)` führt den Befehl in den **Klammern** aus und ersetzt ihn durch die Ausgabe. Beispiel:
    ```bash
        echo "Today is $(date +%F)"
        # => Today is 2029-12-31
    ```
    Die erste Option verwendet einfache Anführungszeichen `'`, **nicht Backticks.** Dies verhindert die Ersetzung, also würde `'$(date +%F)'` einfach den Literalstring `$(date +%F)` ausgeben.

    Obwohl das Verwenden von Backticks (`` `ls -l` ``) zur Befehlsausführung immer noch unterstützt wird, gilt es in einigen Kontexten mittlerweile als Anti-Muster. Die meisten empfehlen `$(Befehl)` wegen besserer Lesbarkeit und Konsistenz mit verschiedenen Shell-Versionen.

    Geschweifte Klammern `${}` werden für Variablenauswertung verwendet, nicht für Befehlsersetzung.

    Das `%`-Zeichen wird nicht für Befehlsersetzung verwendet.
  </div>
  </slot>
</Challenge>
