## «Будьте осторожны» — не граница

«Будьте осторожны» — слабый совет. Он возлагает ответственность за границу на человека.

Человек не является границей. Даже внимательный пользователь может выполнить неправильную команду, открыть неверный проект, одобрить плохое расширение или довериться вредному файлу.

Если запускается вредоносный процесс, важны следующие вопросы:

1. Что этот процесс **может читать**?
2. Какие учётные данные он **может использовать**?
3. Куда он **может отправлять данные**?

Стандарт не «никогда не нажимать ничего странного». Это совет для плаката, а не для системы.

Стандарт — «один странный клик не должен иметь большого радиуса поражения».

## 1. Поместите рискованную работу в «коробку»

[DevContainers](https://github.com/devcontainers/spec) — это самое эффективное изменение, которое большинство локальных сред разработки всё ещё упускают. Они запускают работу над проектом внутри изолированного Docker‑контейнера. Установка пакетов, скрипты `postinstall`, команды AI‑shell, языковые серверы и инструменты проекта выполняются в месте, которое не требует доступа к вашему полному домашнему каталогу.

Монтируйте репозиторий. Не монтируйте `$HOME`, `~/.ssh`, `~/.aws`, `~/Downloads` и ваш менеджер паролей из соображений удобства. Если проект нуждается в секрете, предоставьте ему один узко‑направленный секрет намеренно.

Попросите вашего код‑агента настроить Dev Containers. Затем проверьте монтирования. Этот обзор имеет значение.

```jsonc
// .devcontainer/devcontainer.json
{
  "name": "app",
  "image": "mcr.microsoft.com/devcontainers/typescript-node:1-22",
  "mounts": [
    "source=${localWorkspaceFolder},target=/workspaces/app,type=bind,consistency=cached"
  ]
}
```

Инструкция, внедрённая через подсказку, может достичь только того, к чему процесс имеет доступ. Сделайте это скучным.

## 2. Посадите канарейки там, где ищут злоумышленники

[Canarytokens](https://canarytokens.org) — бесплатные цифровые ловушки. Разместите поддельный, но правдоподобный секрет в месте, которое будет проверять атакующий. Когда он будет затронут, вы получите оповещение, зачастую в течение нескольких секунд.

Разбросайте их рядом с реальными секретами: файлы `.aws/credentials`, `.env`, переменные CI/CD, менеджеры паролей, дампы баз данных и контекст AI‑кода. Канарейка не предотвращает кражу. Она превращает тихое разведывательное действие в сигнал тревоги.

<p class="inset">Злоумышленники делают инвентаризацию перед кражей. Этот разведывательный проход — ваше окно.</p>

```text
~/.aws/credentials            # поддельный профиль [prod-billing-admin]
~/backups/customer-export.sql # URL‑канарейка внутри старого дампа
.env.local                    # поддельный API‑ключ рядом с реальной локальной конфигурацией
```

Если канарейка сработала, предполагаем, что машина всё ещё враждебна:

- Изолируйте её от сети, если подозреваете активный вредонос.
- Смените ключи с чистого устройства.
- Проверьте наличие постоянных точек: новые OAuth‑приложения, deploy‑ключи, IAM‑пользователи, токены доступа, CI‑секреты.
- Завершите активные сеансы браузера для важных сервисов.
- Сообщите человеку, обладающему достаточным контекстом, чтобы помочь.

Не позволяйте первым двадцати минутам реагирования зависеть от памяти. Храните короткую общую runbook‑инструкцию со ссылками на важные системы и порядок их обновления.

## 3. Замедлите свежие пакеты

Вы не в состоянии лично проверять каждого мейнтейнера, транзитивную зависимость, реестр пакетов, workflow и расширение перед установкой. Злоумышленнику нужен лишь один слабый звено. Вам нужны контрольные меры, предполагающие, что одна из уязвимостей в конечном итоге проскочит.

Инциденты с цепочкой поставок и инфостилерами постоянно подтверждают скучную истину: учётные данные живут слишком долго и находятся слишком близко к инструментам, которые исполняют код. [Исследование Snowflake от Mandiant](https://cloud.google.com/blog/topics/threat-intelligence/unc5537-snowflake-data-theft-extortion) проследило множество компромиссов до старых инфостилер‑учётных данных. Кампании [Shai‑Hulud](https://www.ox.security/blog/shai-hulud-here-we-go-again-170-packages-hit-across-npm-pypi/) и [Mini Shai‑Hulud/TanStack](https://www.bleepingcomputer.com/news/security/openai-confirms-security-breach-in-tanstack-supply-chain-attack/) нацеливались на учётные данные разработчиков и облаков через пакеты и CI.

Используйте инструменты безопасности пакетов, где это возможно. [Socket.dev](https://socket.dev), [Snyk](https://snyk.io) и [Wiz](https://wiz.io) могут помочь отловить сигналы, которые вы не заметите вручную.

Для JavaScript‑проектов, где можно применить текущий pnpm, добавьте [minimum release age](https://pnpm.io/settings#minimumreleaseage). Ново‑опубликованные пакеты представляют самый рискованный промежуток: вредоносная версия может быть обнаружена и удалена до вашего следующего установки.

```yaml
minimumReleaseAge: 1440
minimumReleaseAgeStrict: true
minimumReleaseAgeIgnoreMissingTime: false
minimumReleaseAgeExclude:
  - 'typescript'
```

Эта настройка ждёт один день перед принятием новых версий пакетов. Используйте `minimumReleaseAgeExclude` экономно, только для пакетов, где немедленные обновления важнее задержки.

## 4. Сделайте учётные данные скучными

Долгосрочные, широкие учётные данные превращают локальную ошибку в проблему инфраструктуры.

Используйте токены, ограниченные проектом. Предпочитайте краткоживущие облачные учётные данные. Удаляйте старые ключи развертывания. Требуйте пароли‑ключи или аппаратные ключи безопасности для важных учётных записей. Храните дампы баз данных вне обычных папок. Делайте отзыв браузерных сессий частью вашего чек‑листа по инцидентам.

Это не блестящая безопасность. Хорошо. Блестящая безопасность обычно означает, что кто‑то собирается продать вам панель управления.

Выигрыш — меньший радиус поражения: плохая зависимость не должна иметь доступа ко всем облачным учётным записям на вашем ноутбуке. Документ, внедрённый через запрос, не должен выводить ваш домашний каталог. Инфостилер не должен находить старые резервные копии и долгоживущие токены, не сработав сигнализации.

Контейнеры ограничивают охват. Канарейки делают кражу шумнее. Задержки пакетов снижают риск устаревания. Краткоживущие учётные данные уменьшают ущерб.

Это большая часть стратегии: меньше секретов рядом, меньше способов их использовать и быстрее замечать, когда к ним прикоснулись.
