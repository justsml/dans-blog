## 你就是凭证仓库

你的笔记本不再是笔记本。它是一间带键盘的凭证仓库——浏览器会话、SSH 密钥、`.env` 文件、GitHub 令牌、云 CLI、具有 shell 访问权限的 AI 编码工具、那些被你遗忘的数据库导出文件。

旧模式是：生产环境危险，本地环境便捷。这个模式已经结束了。

<p class="inset">
问题不在于你能否避免每一次误点。问题在于一次误点是否能在你察觉之前读取所有内容、使用所有内容并离开。
</p>

开发者会遇到看起来足够正常的东西：来自承包商的 PDF、要求粘贴内容到终端的虚假 CAPTCHA、带 `postinstall` 脚本的包、深入文件系统超出任务需求的 AI 编码会话。有些路径会安装恶意软件。有些会窃取凭证。有些甚至不需要本地漏洞利用——用户自己运行了攻击者的命令。

这就是现代攻击面。有时你就是那个漏洞。

## 供应链问题大得不可思议

有趣的部分来了。要想完全安全，你只需要对你依赖的每个包进行深度跨平台安全评估——包括维护者、历史记录和传递依赖——覆盖所有包注册表。然后每次依赖树变更或更新时重复这个评估，因为供应链攻击正是这样运作的：它们利用信任链。

简单。

哦对了，攻击者只需要成功一次。你却必须每次都维持完美的防御。

Lumma Stealer——一款广泛使用的静默收集密码、浏览器 Cookie、API 密钥和云凭证的信息窃取器——通过虚假 CAPTCHA、投毒搜索广告和木马化应用到达受害者。Mandiant 的 Snowflake 调查追溯了一系列企业入侵事件，源头是信息窃取器窃取的凭证，有些甚至可追溯到 2020 年。攻击中使用的账户中至少有 79.7% 存在已知的先前泄露记录。那些锁从未被更换过。

攻击者没有闯入仓库。他们在抽屉里找到了旧钥匙。

对开发者来说，那个抽屉是这样的：

| 本地产物 | 攻击者关注的原因 |
| --- | --- |
| 浏览器 Cookie | 可以绕过登录，有时甚至能跳过 MFA。 |
| `.env` 文件 | API 密钥、数据库 URL、JWT 密钥。 |
| 云 CLI 配置 | 把笔记本被攻陷变成完整的基础设施访问权限。 |
| SSH 密钥 | 无处不在，权限巨大，还会在机器间复制。 |
| 包管理器令牌 | 你的 npm 或 PyPI 发布令牌就是供应链的入口。 |
| 数据库转储 | 比生产环境保护更少，往往更完整。 |
| AI 编码上下文 | 助手可能被塞给了敏感文件作为"上下文"。 |

然后还有备份——某人把生产导出文件丢在 `~/Downloads` 里就忘了。备份不会因为静态而更安全。它只是没有报警系统的生产数据。

## "小心点"这个不靠谱的方案

"小心点"是软弱的建议。它让人类成为边界。

人类不是边界。人类是流量。

边界是无聊的：文件系统隔离、静态加密的秘密、短生命周期凭证、硬件支持的认证，以及假秘密被触碰时立即触发的警报。

如果恶意进程运行，决定你是过个糟糕的下午还是全公司级别事件的，是这几个问题：
1. 这个进程能**读取**什么？
2. 它能**使用**什么凭证？
3. 它能**发送数据**到哪里？

## 现在杠杆效应最高的动作

### Dev Container——默认开启

[开发容器](https://github.com/devcontainers/spec)是大多数团队没有在做的、杠杆效应最高的变化。Dev Container 在隔离的 Docker 容器内运行项目工作。`npm install`、`pip install`、`postinstall` 脚本、AI shell 命令、VS Code 扩展——所有这些都发生在"工作区"或容器中，无法看到你机器的其他部分。

<p class="inset">让 Claude Code 在任何项目中设置 DevContainers。</p>

挂载仓库。只包含该项目所需的秘密。不要图方便挂载 `~/.ssh`、`~/.aws` 或你的主目录。被注入指令的提示词只能到达代理能到达的范围——让那个范围变得无聊。

```jsonc
// .devcontainer/devcontainer.json
{
  "name": "app",
  "image": "mcr.microsoft.com/devcontainers/typescript-node:1-22",
  "mounts": [
    "source=${localWorkspaceFolder},target=/workspaces/app,type=bind,consistency=cached"
  ]
}
```

攻击者在动手前会清点资产。那次侦察就是你的窗口。
