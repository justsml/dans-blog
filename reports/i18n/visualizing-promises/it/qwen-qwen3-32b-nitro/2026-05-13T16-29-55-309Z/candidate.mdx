Per visualizzare l'esecuzione delle Promises, definiamo un nuovo metodo `delay(millisecs)`.

```js
function delay(millisecs) {
  return new Promise(resolve => {
    setTimeout(() => resolve(millisecs), millisecs);
  });
}
```

Questo è un metodo utilità che risolve una Promise dopo il timeout.

Il ritardo in millisecondi verrà passato al callback di `.then`.

Analizziamo 4 esempi (con tabelle di cronologia animate).

## Esempio #1/4

Questo mostra come l'esecuzione di `console.log()` sarà ritardata da `delay(msec)`.

```js
delay(1000).then(() => console.log("done"));
```

## Esempio #2/4

_This mostra un errore comune._

Il `console.log` viene eseguito non appena inizia `delay(1000)`. Non dopo il ritardo come probabilmente intendevi.

Poiché `console.log` restituisce `undefined`, il nostro `.then()` viene ignorato in modo silenzioso.
