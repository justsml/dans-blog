Per visualizzare come vengono eseguiti i Promise, definiamo un nuovo metodo `delay(millisecs)`.

```js
function delay(millisecs) {
  return new Promise(resolve => {
    setTimeout(() => resolve(millisecs), millisecs);
  });
}
```

Questo è un metodo di utilità che si risolve una volta scaduto il timeout.

Il ritardo in millisecondi verrà passato al callback di `.then`.

Esaminiamo 4 esempi (con timeline animate).

## Example #1/4

Mostra come l'esecuzione di `console.log()` venga posticipata da `delay(msec)`.

```js
delay(1000).then(() => console.log("done"));
```

## Example #2/4

_Questo mostra un errore comune._

Il `console.log` viene eseguito subito quando il `delay(1000)` **inizia**. Non **dopo** il ritardo, come probabilmente ti aspettavi.

Poiché `console.log` restituisce `undefined`, il nostro `.then()` viene silenziosamente ignorato.

Nota ladifferenza tra `typeof console.log === 'function'` e `typeof console.log() === undefined`.

In generale, l'uso corretto di `console.log` è mostrato nell'Esempio #1. Assicurati di passare funzioni a `.then` e `.catch`.

```js
delay(1000).then(console.log("done"));
```

![Timeline che mostra console.log eseguito immediatamente prima che il ritardo termini](../N_1000ms_!log.webp)

## Example #3/4

3 Promise vengono eseguite simultaneamente.

```js

delay(3000).then(console.log);
```

## Esempio #4/4

`Promise.all` con 3 promesse `delay`. Verranno eseguite simultaneamente.

```js
Promise.all([delay(1000), delay(2000), delay(3000)]).then(console.log);
```
