Um zu veranschaulichen, wie Promises ausgeführt werden, definieren wir eine neue Methode `delay(millisecs)`.

```js
function delay(millisecs) {
  return new Promise(resolve => {
    setTimeout(() => resolve(millisecs), millisecs);
  });
}
```

Dies ist eine Hilfsmethode, die sich auflöst, sobald die Timeoutdauer abgelaufen ist.

Die Verzögerung in Millisekunden wird an den Callback von `.then` übergeben.

Schauen wir uns 4 Beispiele (mit animierten Zeitlinien) an.

## Beispiel #1/4

Dies zeigt, wie die Ausführung von `console.log()` durch `delay(msec)` verzögert wird.

```js
delay(1000).then(() => console.log("done"));
```

## Beispiel #2/4

_Dies zeigt einen häufigen Fehler._

`console.log` wird ausgelöst, sobald `delay(1000)` **beginnt**. Nicht **nach** der Verzögerung, wie Sie es wahrscheinlich gewollt haben.  

Da `console.log` `undefined` zurückgibt, wird unser `.then()` stillschweigend ignoriert.  

Beachten Sie den Unterschied zwischen `typeof console.log === 'function'` und `typeof console.log() === undefined`.  

Im Allgemeinen ist die gewünschte Verwendung von `console.log` in Beispiel #1 dargestellt. Stellen Sie sicher, dass Sie Funktionen an `.then` und `.catch` übergeben.  

```js
delay(1000).then(console.log("done"));
```

![Zeitlinie, die zeigt, wie console.log unmittelbar vor dem Abschluss der Verzögerung ausgeführt wird](../N_1000ms_!log.webp)

## Beispiel #3/4

3 Promises werden gleichzeitig ausgeführt.  

```js
delay(1000).then(console.log);
delay(2000).then(console.log);
delay(3000).then(console.log);
```

