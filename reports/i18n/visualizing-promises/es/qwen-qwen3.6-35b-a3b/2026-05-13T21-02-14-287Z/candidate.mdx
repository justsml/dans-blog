Para visualizar cómo se ejecutan las Promises, definamos un nuevo método `delay(millisecs)`.

```js
function delay(millisecs) {
  return new Promise(resolve => {
    setTimeout(() => resolve(millisecs), millisecs);
  });
}
```

Se trata de un método de utilidad que se resolverá una vez transcurrido el tiempo de espera.

El retraso en milisegundos se pasará a la función de devolución de llamada de `.then`.

Analizaremos 4 ejemplos (con líneas de tiempo animadas).

## Ejemplo #1/4

Esto muestra cómo la ejecución de `console.log()` se retrasará debido a `delay(msec)`.

```js
delay(1000).then(() => console.log("done"));
```

## Ejemplo #2/4

_Esto muestra un error común._

El `console.log` se ejecuta justo cuando `delay(1000)` **comienza**. No **después** del retraso, como probablemente esperabas.

Dado que `console.log` devuelve `undefined`, nuestro `.then()` se ignora en silencio.

Fíjate en la diferencia entre `typeof console.log === 'function'` y `typeof console.log() === undefined`.

El uso adecuado de `console.log` se muestra en el Ejemplo #1. Asegúrate de pasar funciones a `.then` y `.catch`.

```js
delay(1000).then(console.log("done"));
```

![Línea de tiempo que muestra console.log ejecutándose inmediatamente antes de que termine el retraso](../N_1000ms_!log.webp)

<!-- ```

## Ejemplo #3/4

3 Promises se ejecutan simultáneamente.

```js
delay(1000).then(console.log);
delay(2000).then(console.log);
delay(3000).then(console.log);
```

`Promise.all` con 3 Promises `delay`. Se ejecutarán simultáneamente.

