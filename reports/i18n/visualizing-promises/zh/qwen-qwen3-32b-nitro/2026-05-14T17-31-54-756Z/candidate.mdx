为了可视化Promise的执行过程，我们定义一个新的方法`delay(millisecs)`。

```js
function delay(millisecs) {
  return new Promise(resolve => {
    setTimeout(() => resolve(millisecs), millisecs);
  });
}
```

这是一个工具方法，超时时间到达后会执行resolve。

以毫秒为单位的延迟时间将传递给`.then`的回调函数。

让我们通过4个示例（配合动画时间线）来观察。

## 示例 #1/4

这展示了`console.log()`的执行将被`delay(msec)`延迟。

```js
delay(1000).then(() => console.log("done"));
```

![时间线展示延迟1000ms后console log在1秒后运行](../N_1000ms_log.webp)

## 示例 #2/4

_这展示了常见的错误用法。_

`console.log`会在`delay(1000)`**开始时**立即执行，而不是**延迟后**执行。

因为`console.log`返回`undefined`，我们的`.then()`会被静默忽略。

注意`typeof console.log === 'function'`与`typeof console.log() === undefined`的区别。

通常推荐的`console.log`用法如示例#1所示。确保将函数传递给`.then`和`.catch`。

```js
delay(1000).then(console.log("done"));
```

![时间线展示console log在延迟完成前立即运行](../N_1000ms_!log.webp)

3个Promise同时执行。

```js

delay(2000).then(console.log);
delay(3000).then(console.log);
```
