<blockquote class="inset">
**अपडेट:** इस लेख ने `llm://` URI स्कीम के लिए एक [इंटरनेट-ड्राफ्ट](https://datatracker.ietf.org/doc/draft-levy-llm-uri-scheme/) को प्रेरित किया।
</blockquote>

पुराने दिनों को याद करें जब डेटाबेस से कनेक्ट होना मतलब अनगिनत पर्यावरण वेरिएबल्स का झंझट था?

यह एक नाज़ुक कॉन्फ़िगरेशन टॉवर था। `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`... या रुकिए, यह `DB_USERNAME` था? यह `DB_PASS` है या `DB_PWD`? क्या इस बार मुझे `PG_*` प्रीफ़िक्स चाहिए? और टाइमआउट सेटिंग कहाँ रखनी है?

यह एक नाज़ुक कार्डों का घर था, जो आपके प्रोडक्शन बिल्ड को तबाह कर सकता था बस इसलिए कि आपने `HOST` को कैपिटलाइज़ करना भूल गए।

फिर किसी ने शानदार विचार किया: बस एक URL¹ इस्तेमाल करें:

```bash
postgres://user:pass@host:5432/dbname
```

एक स्ट्रिंग। सब कुछ जो चाहिए। सार्वभौमिक रूप से पार्सेबल। पोर्टेबल। कहूँ तो... सुंदर!

तो हम LLM को 1999 जैसा क्यों ट्रीट कर रहे हैं?

## एनव वेरिएबल विस्फोट

अभी मेरा `.env` फ़ाइल abandoned API keys की एक कब्रिस्तान जैसा दिखता है। `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `MISTRAL_API_KEY`, `GROQ_API_KEY`। और Azure की बात तो छोड़ो—एक endpoint, एक deployment name, एक API version, और एक key चाहिए बस “हैलो” कहने के लिए।

यह सिर्फ बदसूरत नहीं है; यह घर्षण पैदा करता है। हर बार जब मैं मॉडल बदलना चाहता हूँ या नया प्रोवाइडर टेस्ट करना चाहता हूँ, मुझे initialization कोड फिर से लिखना पड़ता है, विशिष्ट पैरामीटर नामों के लिए डॉक्यूमेंटेशन ढूँढना पड़ता है, और अपने environment config में तीन और लाइनें जोड़नी पड़ती हैं।

क्या होगा अगर हम बस… ~~चुराएँ~~ DB URL का विचार अपनाएँ?

## LLM कनेक्शन स्ट्रिंग्स का परिचय

कल्पना करें कि आप अपनी पूरी मॉडल इंटरफ़ेस को एक ही लाइन में कॉन्फ़िगर कर रहे हैं:

```bash
llm://api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7&max_tokens=1500
llm://api.z.ai/glm-4.7?top_p=0.9&cache=true
```

### LLM कनेक्शन स्ट्रिंग की संरचना

![the parts of a LLM connection string](../inline-url-diagram-dark.svg)

स्कीम `llm://` है। होस्ट प्रोवाइडर का API बेस URL है। पाथ मॉडल का नाम दर्शाता है। और क्वेरी पैरामीटर उन सभी रन‑टाइम विकल्पों को संभालते हैं जो आमतौर पर आपके कोड को गड़बड़ कर देते हैं।

## ऑथकी ज़रूरत? बढ़िया, जोड़ दें।

बिल्कुल `postgres://` की तरह, हम ऑथेंटिकेशन को सीधे URL में एम्बेड कर सकते हैं:

```bash
llm://app-name:sk-proj-123456@api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7
```

*नोट: हाँ, URL में क्रेडेंशियल्स डालना सुरक्षा जोखिम हो सकता है अगर आप उन्हें सार्वजनिक लॉग में पेस्ट करते हैं। लेकिन आधुनिक लॉगिंग सर्विसेज़ इन पैटर्न को साफ़ करने में काफी सक्षम हैं, और ईमानदारी से कहें तो, क्या आप अपनी `.env` फ़ाइल को उससे बेहतर संभाल रहे हैं? सत्यापित करें, सैनिटाइज़ करें, और सावधानी से उपयोग करें।*

## रेज़िलिएंसी? क्यों नहीं।

कई डेटाबेस लाइब्रेरीज़ कई होस्ट निर्दिष्ट करके राउंड‑रॉबिन फेलओवर को सपोर्ट करती हैं। हमारे AI एजेंट्स को वही भरोसेमंदता क्यों नहीं मिलनी चाहिए?

```bash
llms://primary.gpt,backup.gpt/gpt-6?temp=0.9
```

`llms://` में वाला `s` टाइपो नहीं है। यह बहुवचन दर्शाता है। अगर `primary.gpt` फँस जाता है, तो क्लाइंट स्वचालित रूप से `backup.gpt` को रीट्राइ करता है। कोई जटिल राउटर लॉजिक की जरूरत नहीं।

<blockquote class="inset">एक स्ट्रिंग जिसमें **ऑथ** से लेकर **एंडपॉइंट** और **हाइपरपैरामीटर** तक सब कुछ शामिल हो।</blockquote>

## वैकल्पिक फ़ॉर्मेट

मैं `llm://` से बंधा नहीं हूँ। विशिष्ट स्कीम की महत्ता मानक से कम है।

मैं ऐसी स्थिति की कल्पना कर सकता हूँ जहाँ हम संक्षिप्तता के लिए प्रोवाइडर‑विशिष्ट स्कीम इस्तेमाल करें, लेकिन मानक संरचना को बरकरार रखें:

```bash
ollama://localhost:11434/llama3
vercel://anthropic/sonnet-4.5?temp=0.8&web_search={"maxUses":3}
bedrock://us-west-2.aws/anthropic/sonnet-4.5?temp=0.8&cacheControl=ephemeral
```

सटीक सिंटैक्स चाहे जो भी हो, मुख्य लाभ निर्विवाद हैं:

1.  **पोर्टेबिलिटी:** स्थानीय स्क्रिप्ट से क्लाउड वर्कर तक अपनी पूरी कॉन्फ़िग को कॉपी‑पेस्ट करें।
2.  **CLI‑फ्रेंडली:** अपने स्क्रिप्ट को एक ही आर्गुमेंट पास करें। `my-agent --model "llm://..."` बेहतर है `my-agent --model gpt-4 --temp 0.7 --key $KEY --host ...` से।
3.  **भाषा‑अज्ञेय:** हर प्रोग्रामिंग भाषा में एक मजबूत URL पार्सर होता है। हमें वैलिडेशन, पार्सिंग और सैनिटाइज़ेशन मुफ्त में मिल जाता है।

<blockquote class="ai-response inset">डेटाबेस दुनिया को इसे समझने में दशकों लगे।<br /><b>अच्छी खबर, AI टाइमलाइन में यह सिर्फ आधा “वाइब‑इयर” पहले हुआ था।</b></blockquote>

## निष्कर्ष

हमें किसी और जटिल कॉन्फ़िगरेशन मानक या नई YAML‑आधारित मैनिफेस्ट फ़ाइल की जरूरत नहीं है। हमें बस वही टूल इस्तेमाल करना है जो पिछले 30 सालों से इंटरनेट के बाकी हिस्सों में काम कर रहा है।

पहिया फिर से बनाने से बचें और अपने LLM कनेक्शन को उसी सम्मान से देखें जैसा हम डेटाबेस को देते हैं। आपका `.env` फ़ाइल (और आपका दिमाग) आपका धन्यवाद करेगा।
