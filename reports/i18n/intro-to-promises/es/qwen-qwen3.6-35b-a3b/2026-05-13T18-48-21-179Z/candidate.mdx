## Promises... ¿qué tienen de especial?

Cada vez que ejecutas código, hay dos resultados posibles: **éxito** o **fallo**.

Si ese código es asíncrono, depender de su resultado de forma fiable se vuelve más complejo.

**`Promises`** ofrecen una forma práctica de manejarlo.

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> Nota al margen: Aunque los Promises deberían resolver o rechazar, pueden fallar en hacer ambas cosas. Esto provoca que las aplicaciones se queden colgadas y resulta muy difícil de depurar.

### ¿De dónde provienen los Promises?

En muchas ocasiones no necesitarás crear tú mismo un Promise. Las APIs nativas como `fetch` y bibliotecas populares como `axios` ya devuelven Promises.

Sin embargo, si debes crear un Promise, existen 2 formas de hacerlo:

### Creación de Promises #1/2:

La forma más sencilla de crear un Promise es mediante el método auxiliar: `Promise.resolve()`.

Puedes envolver (o "convertir") cualquier valor en un Promise utilizando `Promise.resolve(value)`.

```js
// Without Promises:
function add10(num) {
  return num + 10
}

// With Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Creación de Promises #2/2:

Otro método más flexible: utiliza el constructor `Promise`.

`new Promise(callback)` acepta una función `callback` con la siguiente interfaz:

```js
new Promise(function(resolve, reject) {
  // The arguments `resolve` and `reject` are both functions.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` must get executed when the promise is fulfilled
  // `reject(Error)` must get executed if the promise is rejected
})
```

### API de Promises

La API de Promises en realidad expone un número reducido de métodos.

2 métodos de instancia y 4 funciones estáticas/de utilidad.

#### Métodos de instancia de Promise

Acceder a los valores de una promesa no funcionará con los trucos habituales (p. ej., `console.log(promise)`).

Todas las Promises devuelven un resultado de éxito (vía `.then(fn)`) o de fallo (vía `.catch(fn)`).

#### Métodos de utilidad de Promise

* `Promise.resolve(value)` - Convierte cualquier valor en una Promise
* `Promise.reject(Error)` - Crea un valor de Promise de fallo, activando el `.catch()` subsiguiente
* `Promise.all([...promises])` - Espera a que TODAS las Promises del array se completen
* `Promise.race([...promises])` - Se resuelve en cuanto la primera Promise lo hace
