## Promises... Was ist ihr Zweck?

Wann immer du Computercode ausführst, gibt es zwei mögliche Ergebnisse: **Erfolg** oder **Fehlschlag**.

Wenn dieser Code asynchroner Natur ist, kann es schwieriger sein, sich zuverlässig auf dieses Ergebnis zu verlassen.

**`Promises`** bieten eine praktische Möglichkeit, damit umzugehen.

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> Randnotiz: Obwohl Promises eigentlich resolved oder rejected werden sollten, können sie beides nicht tun. Das führt dazu, dass Apps hängen bleiben, und kann sehr schwer zu debuggen sein.

### Woher kommen Promises?

Oft musst du kein Promise selbst erstellen. Native APIs wie `fetch` und verbreitete Bibliotheken wie `axios` geben bereits Promises zurück.

Falls du dennoch ein Promise erstellen musst, gibt es zwei Möglichkeiten:

### Promises erstellen #1/2:

Der einfachste Weg, ein Promise zu erstellen, ist mit der Hilfsmethode: `Promise.resolve()`.

Du kannst jeden Wert mit `Promise.resolve(value)` in ein Promise verpacken (oder „umwandeln“).

```js
// Ohne Promises:
function add10(num) {
  return num + 10
}

// Mit Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Promises erstellen #2/2:

Eine weitere, flexiblere Methode: der `Promise`-Konstruktor.

`new Promise(callback)` akzeptiert eine `callback`-Funktion mit folgendem Interface:

```js
new Promise(function(resolve, reject) {
  // Die Argumente `resolve` und `reject` sind beides Funktionen.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` muss ausgeführt werden, wenn das Promise erfüllt ist
  // `reject(Error)` muss ausgeführt werden, wenn das Promise abgelehnt wird
})
```

### Die Promise-API

Die Promise-API besteht tatsächlich aus einer kleinen Anzahl von Methoden.

2 Instanzmethoden und 4 statische/Hilfsfunktionen.

#### Instanzmethoden von Promises

Der Zugriff auf Werte eines Promises funktioniert nicht mit den üblichen Tricks (z.B. `console.log(promise)`).

Alle Promises geben entweder Erfolg (via `.then(fn)`) oder Fehlschlag (via `.catch(fn)`) zurück.

#### Promise-Hilfsmethoden

* `Promise.resolve(value)` – Wandelt jeden Wert in ein Promise um
* `Promise.reject(Error)` – Erzeugt einen Fehler-Promise-Wert, löst nachfolgendes `.catch()` aus
* `Promise.all([...promises])` – Wartet, bis ALLE Promises eines Arrays abgeschlossen sind
* `Promise.race([...promises])` – Wird aufgelöst, sobald das erste Promise aufgelöst wird
