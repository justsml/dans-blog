## Promises... comment ça marche ?

Chaque exécution de code informatique aboutit à l'une des deux issues suivantes : **succès** ou **échec**.

Si le code est asynchrone par nature, il devient plus complexe de s'appuyer de manière fiable sur son résultat.

Les **`Promises`** apportent une réponse pratique à cette contrainte.

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> Note : Bien que les Promises soient conçues pour se résoudre ou être rejetées, elles peuvent ne faire ni l'un ni l'autre. Cela provoque des blocages d'application et rend le débogage particulièrement ardu.

### D'où viennent les Promises ?

Vous n'aurez souvent pas besoin de créer vous-même une promesse. Les API natives comme `fetch` et les bibliothèques populaires comme `axios` retournent déjà des Promises.

Toutefois, si vous devez en créer une, deux méthodes s'offrent à vous :

### Créer des Promises #1/2 :

La méthode la plus simple pour créer une Promise est d'utiliser la fonction utilitaire : `Promise.resolve()`.

Vous pouvez envelopper (ou « convertir ») n'importe quelle valeur en une Promise via `Promise.resolve(value)`.

```js
// Without Promises:
function add10(num) {
  return num + 10
}

// With Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Créer des Promises #2/2 :

Une autre méthode, plus flexible : utiliser le constructeur `Promise`.

`new Promise(callback)` accepte une fonction `callback` avec l'interface suivante :

```js
new Promise(function(resolve, reject) {
  // The arguments `resolve` and `reject` are both functions.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` must get executed when the promise is fulfilled
  // `reject(Error)` must get executed if the promise is rejected
})
```

### API des Promises

L'API des Promises se compose en réalité d'un nombre restreint de méthodes.

2 méthodes d'instance, et 4 méthodes statiques/utilitaires.

#### Méthodes d'instance de Promise

Accéder aux valeurs d'une promesse ne fonctionnera pas avec les astuces habituelles (ex. `console.log(promise)`).

Toutes les Promises retournent soit un succès (via `.then(fn)`) soit un échec (via `.catch(fn)`).

#### Méthodes utilitaires de Promise

* `Promise.resolve(value)` - Convertit n'importe quelle valeur en Promise
* `Promise.reject(Error)` - Crée une Promise en état de rejet, déclenche le `.catch()` suivant
* `Promise.all([...promises])` - Attend que TOUS les éléments d'un tableau de Promises soient résolus
* `Promise.race([...promises])` - Se résout dès que la première Promise se résout
