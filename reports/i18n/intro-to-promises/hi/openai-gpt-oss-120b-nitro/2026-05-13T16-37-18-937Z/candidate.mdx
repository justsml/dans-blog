## Promises... उनका मकसद क्या है?

जब भी आप कोई कंप्यूटर कोड चलाते हैं, दो संभावित परिणाम होते हैं: **सफलता** या **विफलता**।

यदि वह कोड असिंक्रोनस है, तो उस परिणाम पर भरोसा करना मुश्किल हो सकता है।

**`Promises`** इस समस्या से निपटने का एक सुविधाजनक तरीका प्रदान करते हैं।

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> नोट: जबकि Promises को resolve या reject होना चाहिए, वे कभी‑कभी दोनों में से कोई भी नहीं कर पाते। इससे एप्लिकेशन हँग हो जाता है, और डिबग करना काफी कठिन हो जाता है।

### Promises कहाँ से आते हैं?

बहुत बार आपकोस्वयं एक Promise बनाने की ज़रूरत नहीं पड़ेगी। नेटिव API जैसे `fetch` और लोकप्रिय लाइब्रेरी जैसे `axios` पहले से ही Promise लौटाते हैं।

हालाँकि यदि आपको ज़रूर एक Promise बनाना पड़े, तो इसके दो तरीके हैं:

### Creating Promises #1/2:

Promise बनाने का सबसे आसान तरीका है हेल्पर मेथड `Promise.resolve()` का उपयोग करना।

आप किसी भी मान को `Promise.resolve(value)` से एक Promise में लपेट (या “कनवर्ट”) सकते हैं।

```js
// Without Promises:
function add10(num) {
  return num + 10
}

// With Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Creating Promises #2/2:

एक अधिक लचीला तरीका: `Promise` कंस्ट्रक्टर का उपयोग करना।

`new Promise(callback)` एक `callback` फ़ंक्शन लेता है जिसका इंटरफ़ेस इस प्रकार है:

```js
new Promise(function(resolve, reject) {
  // आर्ग्यूमेंट `resolve` और `reject` दोनों फ़ंक्शन हैं।
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` को तब चलाया जाना चाहिए जब प्रॉमिस पूर्ण हो
  // `reject(Error)` को तब चलाया जाना चाहिए जब प्रॉमिस अस्वीकृत हो
})
```

### Promises API

Promises API वास्तव में कुछ ही मेथड्स का समूह है।

2 इंस्टेंस फ़ंक्शन, और 4 स्टैटिक/यूटिलिटी फ़ंक्शन।

#### Promise इंस्टेंस मेथड्स

एक प्रॉमिस से मान निकालना सामान्य ट्रिक्स (जैसे `console.log(promise)`) से काम नहीं करेगा।

सभी प्रॉमिस या तो सफलता (`.then(fn)` द्वारा) या विफलता (`.catch(fn)` द्वारा) लौटाते हैं।

#### प्रॉमिस यूटिलिटी मेथड्स

* `Promise.resolve(value)` - किसी भी मान को प्रॉमिस में बदलता है  
* `Promise.reject(Error)` - एक विफलता प्रॉमिस बनाता है, जिससे आगे का `.catch()` ट्रिगर होता है  
* `Promise.all([...promises])` - प्रॉमिसों की एरे के सभी तत्वों के पूरा होने की प्रतीक्षा करता है  
* `Promise.race([...promises])` - पहला प्रॉमिस रिजॉल्व होते ही परिणाम देता है
