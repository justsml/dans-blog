## Promises... Qual è il loro problema?

Ogni volta che esegui qualsiasi codice informatico, ci sono 2 possibili esiti: **successo** o **fallimento**.

Se quel codice è asincrono, può essere più difficile contare in modo affidabile su quel risultato.

Le **`Promises`** forniscono un modo utile per gestire questa situazione.

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> Nota laterale: Anche se le Promises dovrebbero risolversi o rifiutarsi, potrebbero non fare né l'una né l'altra. Questo fa sì che le applicazioni si blocchino e può essere molto difficile da debuggare.

### Da dove vengono le Promises?

Spesso non avrai bisogno di creare una Promise da solo. API native come `fetch` e librerie popolari come `axios` restituiscono già Promises.

Tuttavia, se devi creare una Promise, esistono due modi per farlo:

### Creazione di Promises #1/2:

Il modo più semplice per creare una Promise è utilizzare il metodo helper: `Promise.resolve()`.

Puoi avvolgere (o "convertire") qualsiasi valore in una Promise utilizzando `Promise.resolve(value)`.

```js
// Senza Promises:
function add10(num) {
  return num + 10
}

// Con Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Creazione di Promises #2/2:

Un altro metodo più flessibile: utilizzare il costruttore `Promise`.

`new Promise(callback)` accetta una funzione `callback` con l'interfaccia seguente:

```js
new Promise(function(resolve, reject) {
  // Gli argomenti `resolve` e `reject` sono entrambi funzioni.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` deve essere eseguito quando la promise è completata
  // `reject(Error)` deve essere eseguito se la promise è rifiutata
})
```

### API delle Promise

L'API delle Promise è in realtà costituita da un numero ridotto di metodi.

2 metodi di istanza e 4 funzioni statiche/utility.

#### Metodi di istanza delle Promise

Accedere ai valori di una Promise utilizzando i trucchi usuali (es. `console.log(promise)`) non funzionerà.  

Tutte le Promises restituiscono successo (tramite `.then(fn)`) o fallimento (tramite `.catch(fn)`).  

#### Metodi utilità delle Promise  

* `Promise.resolve(value)` - Converti qualsiasi valore in una Promise  
* `Promise.reject(Error)` - Crea un valore Promise di fallimento, attiva il successivo `.catch()`  
* `Promise.all([...promises])` - Aspetta che un array di Promises venga COMPLETAMENTE completato  
* `Promise.race([...promises])` - Risolve non appena la prima Promise viene risolta
