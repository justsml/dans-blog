## Promises... a che servono?

Ogni esecuzione di codice comporta due possibili esiti: **successo** o **fallimento**.

Se il codice è asincrono, diventa più complesso garantire un affidamento deterministico sul risultato.

**`Promises`** forniscono un meccanismo pratico per gestire questa dinamica.

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> Nota: Sebbene un Promise debba risolversi o essere rifiutato, potrebbe non fare né l'uno né l'altro. Ciò causa il blocco dell'applicazione e rende il debug estremamente complesso.

### Da dove provengono i Promises?

Spesso non sarà necessario creare un Promise manualmente. API native come `fetch` e librerie popolari come `axios` restituiscono già Promise.

Tuttavia, se devi crearne uno, esistono 2 modi per farlo:

### Creare un Promise #1/2:

Il modo più semplice per istanziare un Promise è utilizzare il metodo helper `Promise.resolve()`.

Puoi incapsulare (o "convertire") qualsiasi valore in un Promise utilizzando `Promise.resolve(value)`.

```js
// Without Promises:
function add10(num) {
  return num + 10
}

// With Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Creare un Promise #2/2:

Un altro approccio più flessibile: usa il costruttore `Promise`.

`new Promise(callback)` accetta una funzione `callback` con la seguente interfaccia:

```js
new Promise(function(resolve, reject) {
  // The arguments `resolve` and `reject` are both functions.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` must get executed when the promise is fulfilled
  // `reject(Error)` must get executed if the promise is rejected
})
```

### API dei Promise

L'API dei Promise è in realtà composta da un numero limitato di metodi.

2 metodi di istanza e 4 metodi statici/utilità.

#### Metodi di istanza di Promise

Recuperare i valori da una Promise non funziona con approcci diretti (es. `console.log(promise)`).

Ogni Promise restituisce un esito di successo (tramite `.then(fn)`) o di fallimento (tramite `.catch(fn)`).

#### Metodi utility di Promise

* `Promise.resolve(value)` - Converte qualsiasi valore in una Promise
* `Promise.reject(Error)` - Crea un valore Promise di fallimento, attiva il successivo `.catch()`
* `Promise.all([...promises])` - Attende che TUTTE le Promise dell'array siano completate
* `Promise.race([...promises])` - Si risolve non appena la prima Promise viene risolta
