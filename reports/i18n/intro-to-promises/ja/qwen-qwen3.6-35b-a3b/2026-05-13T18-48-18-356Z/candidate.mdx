## Promises… 結局何が便利なのか？

コンピュータコードを実行する際、結果は常に**成功**か**失敗**のいずれかになる。

コードが非同期である場合、その結果を確実に信頼して利用するのは難しくなる。

**`Promise`** はこれを処理するための実用的な手段を提供する。

```
                        +--Promise---+
                        |            |
                        | <-either-> |
                        |            |
                <-------+            +-------->
                |Failure?             Success?|
                |                             |
                v                             v
                (Rejected)           (Resolved)
```

> 補足: Promise は本来 resolve または reject されるべきだが、どちらにも至らない場合がある。これによりアプリケーションがハングアップし、デバッグが極めて困難になる。

### Promise はどこから来るのか？

多くの場合、自分で Promise を生成する必要はない。`fetch` などのネイティブ API や `axios` などの人気ライブラリは、すでに Promise を返す。

ただし、あえて Promise を生成する必要がある場合は、2つの方法がある。

### Promise の生成方法 1/2:
Promise を生成する最も簡単な方法は、ヘルパーメソッド `Promise.resolve()` を使うことだ。

`Promise.resolve(value)` を通じて、任意の値を Promise でラップ（または「変換」）できる。

```js
// Without Promises:
function add10(num) {
  return num + 10
}

// With Promises:
function add10Promised(num) {
  return Promise.resolve(num + 10)
}

console.log(add10(10)) //=> 20

add10Promised(10)
  .then(x => console.log(x)) //=> 20
```

### Promise の生成方法 2/2:

より柔軟な別の手段：`Promise` コンストラクタを使用する。

`new Promise(callback)` は、以下のインターフェースを持つ `callback` 関数を受け取る。

```js
new Promise(function(resolve, reject) {
  // The arguments `resolve` and `reject` are both functions.
  // typeof resolve === 'function'
  // typeof reject === 'function'

  // `resolve(result)` must get executed when the promise is fulfilled
  // `reject(Error)` must get executed if the promise is rejected
})
```

### Promise API

Promise API が提供するメソッドは、実はごく少ない。

インスタンスメソッドが 2 つ、静的（ユーティリティ）メソッドが 4 つ。

#### Promise インスタンスメソッド

Promise から内部の値を取り出す場合、JavaScript の通常の同期ログ出力手法（例：`console.log(promise)`）では期待通りに動作しない。これは `console.log` が実行時点で Promise オブジェクトの参照を出力するためであり、非同期タスクが完了した後の実際の値（fulfilled/rejected の中身）は表示されない。値を取得するには、必ず `.then()` や `.catch()` 経由で処理を記述する必要がある。

すべての Promise インスタンスは、処理結果として成功（`.then(fn)` 経由）か失敗（`.catch(fn)` 経由）のいずれかの実行パスを返す。これにより、非同期処理の制御フローを同期的なコードのように連鎖させられる。

#### Promise ユーティリティメソッド

* `Promise.resolve(value)` - 任意の値（Promise 以外も含む）を Promise インスタンスに変換する
* `Promise.reject(Error)` - 失敗状態の Promise を生成し、チェーン内の後続 `.catch()` ハンドラをトリガーする
* `Promise.all([...promises])` - 配列に格納されたすべての Promise が完了（解決または拒否）するまで待機し、結果を配列で返す。いずれか一つでも拒否されると、即座に拒否状態になる
* `Promise.race([...promises])` - 配列内の最初の Promise が解決または拒否された時点で、その結果で即時解決する
