## Gestire in modo sicuro segreti e token API

### Articolo correlato: [Proteggi i tuoi token](../protect-your-tokens/)

Ricapitoliamo velocemente la differenza tra `secret` e `non-secret`.

* 🔒 Le `Secret keys` DEVONO usare un server personalizzato (es. Node/Express/Heroku) per nascondere (proxy) le richieste ai servizi API di terze parti.
* 🌍 Le `Non-secret keys` descrivono chiavi che possono essere inviate al browser.

<br />

---------------------------------------------

> In questo articolo ci concentreremo sulla gestione di 🔒 `Secret keys` usando **Variabili d'ambiente**.

[Esempi di codice sono inclusi più sotto.](#️-code-example)

#### Panoramica

Per **accedere in modo sicuro ai segreti nel tuo codice NodeJS:**

1. Sostituisci le chiavi hard-coded con variabili d'ambiente. es. `process.env.API_SECRET`
1. Usa una libreria come [`dotenv`](https://github.com/motdotla/dotenv) insieme a un file `.env`. Aggiungi i segreti precedentemente hard-coded al file `.env`.
1. Verifica la riga `.env` nel tuo file `.gitignore`!

> **NON** creare un file `.env` sui server in produzione. Usa lo strumento di gestione delle variabili d'ambiente fornito dal tuo servizio di hosting (es. [Heroku](https://devcenter.heroku.com/articles/config-vars), Netlify, AWS EC2): ad esempio **dashboard o riga di comando.**

### Esempio di codice

Definiremo alcuni file.

1. `.env`
1. `./db/connection.js`
1. `./api/users.js`

Per prima cosa, installa il pacchetto [`dotenv`](https://www.npmjs.com/package/dotenv).

```bash
npm install dotenv
```

Successivamente, crea un file `.env` nella radice del tuo progetto.

```
# .env
PGDATABASE="postgres"
PGHOST="localhost"
PGPORT=5234
PGUSER="postgres"
PGPASSWORD="password"
```

❌ **NON** eseguire mai il commit del file `.env`.

❌ Evita di creare `.env` sui server.

Consulta la documentazione del tuo provider di hosting per configurare le _variabili d'ambiente_.

Per assicurarti facilmente che il tuo `.gitignore` contenga una riga per `.env`.

```bash
# Aggiorna automaticamente .gitignore
# Esegui nel terminale:
[ "$(grep '^.env' .gitignore)" == "" ] && echo '.env' >> .gitignore
# nota: non verrà stampato alcun output
```

Il file `./db/connection.js` fornisce un'istanza condivisa di `pg.Pool`. Verrà utilizzata per interrogare il database.

```js
// ./db/connection.js
require('dotenv').config(); // ✅ Carica il file .env
const pg = require('pg');
const {PGUSER, PGHOST, PGPORT} = process.env;

if (process.env.NODE_ENV === 'development')
  console.log(`Connessione a ${PGUSER} @ ${PGHOST}:${PGHOST}`);
// ^^ solo per mostrare le variabili di debug della connessione

// pg utilizza automaticamente le variabili d'ambiente PG*
module.exports = new pg.Pool();
```

La cartella `./api` contiene le interfacce per le tue tabelle/viste.

Ecco un esempio di `./api/users.js` per la tabella `users`.

```js
// ./api/users.js
const db = require('../db/connection.js');

module.exports = {
  findUsername: function(username) {
    return db.query('SELECT * FROM users WHERE username=$1', username);
  }
};
```

-  Non eseguire mai il commit dei tuoi segreti `.env` su git!
-  Non condividere file `.env` in un team. *

\* Ogni nuovo laptop o desktop di sviluppo dovrebbe **generare nuove chiavi di accesso e token.**
Se non è possibile, usa molta cautela nel condividere il tuo `.env` (nei casi in cui un servizio potrebbe invalidare tutte le chiavi precedenti, o hai un token di accesso limitato per un'API a pagamento.)

#### ⚠️ Importante: se necessario, utilizza sempre un servizio di messaggistica sicuro (preferibilmente con supporto per messaggi a scadenza.)
