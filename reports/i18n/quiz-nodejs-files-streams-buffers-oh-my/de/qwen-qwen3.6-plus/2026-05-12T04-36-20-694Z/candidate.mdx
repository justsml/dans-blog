`fs.watch()` garantiert nicht, dass es genau einmal pro logischer Dateiänderung ausgelöst wird. Es wird oft mehrfach ausgelöst, da viele Texteditoren:
    1. In eine temporäre Datei speichern
    2. Diese in die Zieldatei umbenennen

    - Das `chokidar`-Paket
    - Debouncing des Callbacks
    - Die Verwendung von `fs.watchFile()` (obwohl dies weniger effizient ist)

  options={[
    {text: 'Verhindert Speicherüberlauf', isAnswer: true},
    {text: 'Erhöht die Lesegeschwindigkeit'},
    {text: 'Komprimiert Daten'},
    {text: 'Verschlüsselt Daten'},
  ]}
>
  <slot name="question">
  <div className="question">
Was ist der Hauptzweck von Stream-Backpressure?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
Backpressure ist ein Mechanismus, der Speicherüberlauf verhindert, indem das Lesen pausiert wird, wenn das schreibende Ende nicht hinterherkommt.

Beispiel für manuelles Backpressure:
```js
    readable.on('data', (chunk) => {
      const canContinue = writable.write(chunk);
      if (!canContinue) {
        readable.pause();
        writable.once('drain', () => readable.resume());
      }
    });
```
`pipe()` regelt das automatisch!

[Mehr über Backpressure erfahren](https://nodejs.org/en/learn/modules/backpressuring-in-streams#backpressuring-in-streams)
  </div>
  </slot>
