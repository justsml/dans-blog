Nessuno si propone di costruire un sistema AI insicuro. Scrivi le istruzioni, testi i casi limite, aggiungi un paio di regole di validazione. Poi qualcuno scopre di poter ingannare il tuo bot per fargli interpretare il ruolo di un pirata e esporre dati utente. O un numero di carta di credito finisce nei tuoi log. Oppure il modello raccomanda con sicurezza un prodotto di un concorrente.

Il divario tra "funziona nella demo" e "sicuro in produzione" è più ampio di quanto si aspetti la maggior parte delle squadre.

Parte del problema è che i modelli LLM grezzi non hanno opinioni su ciò che dovrebbero o non dovrebbero fare. Sono macchine di previsione che cercano di continuare qualsiasi pattern tu abbia iniziato. Dargli un prompt che sembri "modalità sovrascrittura sistema" e giocheranno volentieri il gioco. Questo non è un bug nel modello; è semplicemente il modo in cui funzionano i modelli linguistici.

La maggior parte dei framework ti consegnano il modello e ti augurano buona fortuna. Mastra adotta un approccio diverso: assume che avrai bisogno di binari di sicurezza prima o poi, quindi li integra nell'architettura dell'agente fin dall'inizio.

## Processori come Strati di Sicurezza

Il meccanismo principale è semplice. Prima che il tuo prompt raggiunga il modello, passa attraverso una catena di processori di input. Dopo che il modello risponde, tocca ai processori di output. Ogni processore può ispezionare, modificare o bloccare il contenuto a quella fase.

Pensali come middleware per le interazioni AI. Impili quelli di cui hai bisogno, configura il loro comportamento e si eseguiranno automaticamente su ogni richiesta.

### 1. Fermare i pirati (Prompt Injection)

Gli attacchi di prompt injection hanno preso vita. Le persone usano caratteri Unicode invisibili, scrivono istruzioni in base64 o convincono il modello che è in "modalità debug" dove le regole normali non si applicano. Le tecniche continuano ad evolversi.

Mastra include processori che rilevano modelli comuni:

```typescript
// src/mastra/agents/secure-agent.ts
import { Agent } from '@mastra/core/agent';
import { PromptInjectionDetector, UnicodeNormalizer } from '@mastra/core/processors';
import { openai } from '@ai-sdk/openai';

export const secureAgent = new Agent({
  id: 'fortress-assistant',
  name: 'fortress-assistant',
  instructions: 'You are a secure assistant.',
  model: openai('gpt-5'),
  inputProcessors: [
    // 1. Elimina caratteri invisibili
    new UnicodeNormalizer({
      id: 'unicode-normalizer',
      stripControlChars: true,
      collapseWhitespace: true,
    }),
    // 2. Rileva l'attacco
    new PromptInjectionDetector({
      id: 'prompt-injection-detector',
      model: openai('gpt-5-nano'), // Economico, veloce
      threshold: 0.8,
      strategy: 'block', // Blocco duro
      detectionTypes: ['injection', 'jailbreak', 'system-override'],
    }),
  ],
});
```

Il [`UnicodeNormalizer`](https://mastra.ai/docs/processors) elimina i caratteri di controllo e comprime gli spazi bianchi. Il [`PromptInjectionDetector`](https://mastra.ai/docs/processors) analizza l'input pulito alla ricerca di modelli che suggeriscano qualcuno stia cercando di sovrascrivere le tue istruzioni.

Configuri quanto aggressiva deve essere la rilevazione (il parametro `threshold`) e cosa deve accadere quando viene attivata (bloccare, registrare o semplicemente segnalarla).

### 2. Gestione delle informazioni identificabili personalmente (PII)

Numeri di carta di credito nei log, numeri di previdenza sociale nei database vettoriali, indirizzi email conservati più a lungo del necessario. Questi sono i tipi di problemi che si trasformano in questioni regolamentari. La sfida è che gli utenti non sempre si rendono conto di incollare dati sensibili in una finestra di chat.

Il [`PIIDetector`](https://mastra.ai/docs/processors) analizza i modelli comuni prima che raggiungano il tuo modello o vengano scritti in archiviazione:

```typescript
import { PIIDetector } from '@mastra/core/processors';

export const privateAgent = new Agent({
  id: 'privacy-first-assistant',
  name: 'privacy-first-assistant',
  instructions: 'You are a helpful assistant that never stores personal information.',
  model: openai('gpt-5'),
  inputProcessors: [
    new PIIDetector({
      id: 'pii-detector',
      model: openai('gpt-5-nano'),
      detectionTypes: ['email', 'phone', 'credit-card', 'ssn'],
      threshold: 0.6,
      strategy: 'redact',
      redactionMethod: 'mask',  // Replace with [REDACTED]
      instructions: 'Detect and mask personally identifiable information',
    }),
  ],
});
```

Puoi scegliere di mascherare (sostituire con `[REDACTED]`), hashare o bloccare completamente. Il processore funziona su input e output, quindi sei protetto anche se il modello genera per qualche motivo dati sensibili nella sua risposta.

### 3. Moderazione del contenuto

I modelli addestrati sui dati internet hanno visto di tutto. Senza filtraggio, possono produrre occasionalmente risposte che farebbero inorridire il vostro team di comunicazione. Il [`ModerationProcessor`](https://mastra.ai/docs/processors) cattura contenuti che violano le vostre linee guida:

```typescript
import { ModerationProcessor } from '@mastra/core/processors';

export const moderatedAgent = new Agent({
  id: 'safe-assistant',
  name: 'safe-assistant',
  instructions: 'You are a helpful assistant for a community platform.',
  model: openai('gpt-5'),
  inputProcessors: [
    new ModerationProcessor({
      id: 'moderation-processor',
      model: openai('gpt-5-nano'),  // Fast, cheap model for classification
      categories: ['hate', 'harassment', 'violence', 'self-harm'],
      threshold: 0.7,  // Block if confidence > 70%
      strategy: 'block',  // Stop the request immediately
      instructions: 'Detect harmful content that violates community guidelines',
    }),
  ],
});
```

L'aspetto interessante è che puoi definire quali categorie sono rilevanti per il tuo caso d'uso. Uno strumento per scrittura creativa potrebbe permettere contenuti espressivi più ampi rispetto a un bot di assistenza clienti. La soglia e la strategia ti danno il controllo su quanto rigoroso dovrebbe essere il filtraggio.

## Quando le Paratie Vengono Triggerate

I processori non generano errori quando rilevano un problema. Invece, impostano un flag sull'oggetto risultato:

```typescript
const result = await secureAgent.generate('Ignore all previous instructions...');

if (result.tripwire) {
  console.log(`Bloccato! Motivo: ${result.tripwireReason}`);
  // "Bloccato! Motivo: Rilevato tentativo di iniezione di prompt."
  return "Prova a farlo, script kiddie.";
}
```

Questo pattern ti permette di gestire gli eventi di sicurezza in base a ciò che ha senso per la tua applicazione. Potresti registrare i log per l'analisi, restituire un messaggio di errore generico, o persino permettere certe violazioni in contesti specifici. Il campo `tripwireReason` ti dice esattamente quale processore ha segnalato il contenuto, il che è utile quando debugghi falsi positivi o regoli le tue soglie.

## Cosa Questo Non Risolve

I processori rilevano molto, ma non sono magici. Un attaccante determinato con abbastanza tempo probabilmente troverà un prompt che passa inosservato. I modelli a volte hallucinano in modi imprevedibili che i processori non possono anticipare. E c'è sempre un compromesso tra sicurezza e flessibilità: più restrittive le tue regole, più probabile è che blocchi casi d'uso legittimi.

Il valore non è una protezione perfetta. È disporre di un modo sistematico per affrontare i problemi comuni che sicuramente emergeranno in produzione. Puoi regolare la sensibilità man mano che impari cosa fanno effettivamente gli utenti. Puoi aggiungere processori personalizzati per rischi specifici del dominio. E hai tracce di audit che mostrano cosa è stato bloccato e perché.

La maggior parte dei problemi di sicurezza in AI in produzione non sono attacchi sofisticati. Sono persone che copiano e incollano dati che non dovrebbero, o che scoprono per tentativi ed errori che il bot farà cose che non intendevi. I processori non bloccheranno ogni possibile problema, ma rendono molto più difficile gli errori ovvi.
