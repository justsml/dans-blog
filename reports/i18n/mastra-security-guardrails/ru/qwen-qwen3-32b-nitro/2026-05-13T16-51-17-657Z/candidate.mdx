Ни у кого нет намерения создавать небезопасную систему ИИ. Вы пишете инструкции, тестируете граничные случаи, добавляете несколько правил проверки. Потом кто-то обнаруживает, что может обмануть вашего бота, чтобы тот играл роль пирата и раскрывал данные пользователя. Или номер кредитной карты оказывается в ваших логах. Или модель уверенно рекомендует продукт конкурента.

Разрыв между "работает в демонстрации" и "безопасно в продакшене" шире, чем ожидают большинство команд.

Одна из проблем в том, что сырые языковые модели не имеют мнений о том, что они должны или не должны делать. Это машины для предсказаний, которые пытаются продолжить любую начатую вами последовательность. Дайте им промпт вроде "режим системного перезапуска", и они с радостью сыграют по этой схеме. Это не баг модели — это просто способ, как работают языковые модели.

Большинство фреймворков дают вам модель и пожелают удачи. Mastra идёт другим путём: она предполагает, что вам понадобятся ограждения, и встраивает их в архитектуру агента с самого начала.

## Процессоры как слои безопасности

Ядро механизма простое. Перед тем, как ваш промпт достигнет модели, он проходит через цепочку процессоров ввода. После ответа модели, процессоры вывода получают свой шанс. Каждый процессор может проверять, изменять или блокировать содержимое на этом этапе.

Мыслите о них как о промежуточном ПО для взаимодействия с ИИ. Вы стекаете те, которые вам нужны, настраиваете их поведение, и они автоматически выполняются для каждого запроса.

### 1. Остановка пиратов (Инъекция промптов)

Атаки инъекции промптов становятся изощренными. Люди используют невидимые символы Unicode, пишут инструкции в Base64 или убеждают модель, что она находится в "режиме отладки", где обычные правила не действуют. Техники продолжают эволюционировать.

Mastra включает процессоры, которые обнаруживают распространенные паттерны:

```typescript
// src/mastra/agents/secure-agent.ts
import { Agent } from '@mastra/core/agent';
import { PromptInjectionDetector, UnicodeNormalizer } from '@mastra/core/processors';
import { openai } from '@ai-sdk/openai';

export const secureAgent = new Agent({
  id: 'fortress-assistant',
  name: 'fortress-assistant',
  instructions: 'You are a secure assistant.',
  model: openai('gpt-5'),
  inputProcessors: [
    // 1. Удаление невидимых символов
    new UnicodeNormalizer({
      id: 'unicode-normalizer',
      stripControlChars: true,
      collapseWhitespace: true,
    }),
    // 2. Обнаружение атаки
    new PromptInjectionDetector({
      id: 'prompt-injection-detector',
      model: openai('gpt-5-nano'), // Дешево и быстро
      threshold: 0.8,
      strategy: 'block', // Жесткая остановка
      detectionTypes: ['injection', 'jailbreak', 'system-override'],
    }),
  ],
});
```

[`UnicodeNormalizer`](https://mastra.ai/docs/processors) удаляет управляющие символы и объединяет пробелы. [`PromptInjectionDetector`](https://mastra.ai/docs/processors) анализирует очищенный ввод на наличие паттернов, указывающих на попытку обойти ваши инструкции.

Вы настраиваете, насколько агрессивно должна быть настроена детекция (параметр `threshold`) и что должно происходить при срабатывании (блокировать, записывать в журнал или просто помечать).

### 2. Обработка ПДн

Номера кредитных карт в логах, номера социального страхования в векторных базах данных, адреса электронной почты, хранящиеся дольше, чем необходимо. Это те проблемы, которые превращаются в регуляторные. Вызов заключается в том, что пользователи не всегда осознают, что вставляют конфиденциальные данные в окно чата.

[`PIIDetector`](https://mastra.ai/docs/processors) сканирует распространённые паттерны перед тем, как они достигнут вашей модели или будут записаны в хранилище:

```typescript
import { PIIDetector } from '@mastra/core/processors';

export const privateAgent = new Agent({
  id: 'privacy-first-assistant',
  name: 'privacy-first-assistant',
  instructions: 'You are a helpful assistant that never stores personal information.',
  model: openai('gpt-5'),
  inputProcessors: [
    new PIIDetector({
      id: 'pii-detector',
      model: openai('gpt-5-nano'),
      detectionTypes: ['email', 'phone', 'credit-card', 'ssn'],
      threshold: 0.6,
      strategy: 'redact',
      redactionMethod: 'mask',  // Replace with [REDACTED]
      instructions: 'Detect and mask personally identifiable information',
    }),
  ],
});
```

Вы можете выбрать зачёркивание (заменить на `[REDACTED]`), хеширование или полную блокировку. Процессор работает как с входными, так и с выходными данными, поэтому вы защищены даже в том случае, если модель каким-то образом сгенерирует конфиденциальные данные в своём ответе.

### 3. Модерация контента

Модели, обученные на интернет-данных, видели разные вещи. Без фильтрации они могут время от времени генерировать ответы, которые вызовут беспокойство у вашей PR-команды. [`ModerationProcessor`](https://mastra.ai/docs/processors) обнаруживает контент, нарушающий ваши рекомендации:

```typescript
import { ModerationProcessor } from '@mastra/core/processors';

export const moderatedAgent = new Agent({
  id: 'safe-assistant',
  name: 'safe-assistant',
  instructions: 'You are a helpful assistant for a community platform.',
  model: openai('gpt-5'),
  inputProcessors: [
    new ModerationProcessor({
      id: 'moderation-processor',
      model: openai('gpt-5-nano'),  // Быстрая и дешевая модель для классификации
      categories: ['hate', 'harassment', 'violence', 'self-harm'],
      threshold: 0.7,  // Блокировать, если уверенность > 70%
      strategy: 'block',  // Немедленно остановить запрос
      instructions: 'Detect harmful content that violates community guidelines',
    }),
  ],
});
```

Интересный момент заключается в том, что вы сами определяете, какие категории важны для вашего сценария. Инструмент для творческого письма может позволить более выразительный контент, чем чат-бот для службы поддержки. Порог и стратегия дают вам контроль над тем, насколько строгим должно быть фильтрование.

## Когда срабатывают защиты

Процессоры не выбрасывают ошибки при обнаружении проблемы. Вместо этого они устанавливают флаг в объекте результата:

```typescript
const result = await secureAgent.generate('Ignore all previous instructions...');

if (result.tripwire) {
  console.log(`Заблокировано! Причина: ${result.tripwireReason}`);
  // "Заблокировано! Причина: Обнаружена инъекция подсказки."
  return "Молодец, попробуй еще.";
}
```

Этот паттерн позволяет обрабатывать события безопасности так, как это имеет смысл для вашей системы. Вы можете логировать их для анализа, возвращать обобщённое сообщение об ошибке или даже разрешать определённые нарушения в конкретных контекстах. Поле `tripwireReason` точно указывает, какой процессор выявил контент, что помогает при отладке ложных срабатываний или настройке пороговых значений.

## Что это не решает

Процессоры ловят многое, но они не волшебство. Упрямый атакующий, имеющий достаточно времени, вероятно, найдет запрос, который проскользнет. Модели время от времени галлюцинируют так, что процессоры не могут предсказать. И всегда есть компромисс между безопасностью и гибкостью: чем жестче ваши правила, тем выше вероятность блокировки законных случаев использования.

Ценность не в идеальной защите. Это наличие систематического способа обрабатывать распространенные проблемы, которые точно возникнут в продакшене. Вы можете настроить чувствительность по мере изучения того, что реально делают пользователи. Вы можете добавить пользовательские процессоры для рисков, специфичных для вашей области. И у вас есть аудитные трассировки, показывающие, что было заблокировано и почему.

Большинство проблем безопасности в продакшен-АИ — это не сложные атаки. Это люди, копирующие и вставляющие данные, которых не должны, или обнаруживающие через опыт и ошибку, что бот будет делать вещи, которые вы не предусматривали. Процессоры не остановят каждую возможную проблему, но делают очевидные проблемы намного сложнее.
