Никто не ставит перед собой задачу создать небезопасную ИИ‑систему. Вы пишете инструкции, проверяете граничные случаи, добавляете несколько правил валидации. Затем кто‑то обнаруживает, что можно заставить вашего бота сыграть роль пирата и раскрыть данные пользователей. Или номер кредитной карты оказывается в ваших журналах. Или модель уверенно рекомендует продукт конкурента.

Разрыв между «работает в демо» и «безопасно в продакшене» гораздо больше, чем ожидает большинство команд.

Частично проблема в том, что сырые LLM не имеют собственного мнения о том, что им следует или не следует делать. Они — предсказательные машины, которые пытаются продолжить любой заданный вами шаблон. Дайте им запрос, похожий на «режим системного переопределения», и они с радостью подыграют. Это не баг модели; это просто способ работы языковых моделей.

Большинство фреймворков просто передают вам модель и желают удачи. Mastra идёт иначе: она предполагает, что вам в конечном итоге понадобятся ограждения, поэтому сразу встраивает их в архитектуру агента.

## Процессоры как уровни безопасности

Основной механизм прост. Прежде чем ваш запрос попадёт в модель, он проходит через цепочку входных процессоров. После того как модель выдаёт ответ, к работе подключаются процессоры вывода. Каждый процессор может инспектировать, изменять или блокировать содержимое на своём этапе.

Считайте их посредниками для взаимодействий с ИИ. Вы накладываете нужные, настраиваете их поведение и они автоматически запускаются для каждого запроса.

### 1. Остановка пиратов (внедрение подсказок)

Атаки внедрения подсказок становятся всё изобретательнее. Люди используют невидимые Unicode‑символы, пишут инструкции в base64 или убеждают модель, что она находится в «режиме отладки», где обычные правила не действуют. Техники постоянно эволюционируют.

Mastra включает процессоры, которые ловят типичные шаблоны:

```typescript
// src/mastra/agents/secure-agent.ts
import { Agent } from '@mastra/core/agent';
import { PromptInjectionDetector, UnicodeNormalizer } from '@mastra/core/processors';
import { openai } from '@ai-sdk/openai';

export const secureAgent = new Agent({
  id: 'fortress-assistant',
  name: 'fortress-assistant',
  instructions: 'You are a secure assistant.',
  model: openai('gpt-5'),
  inputProcessors: [
    // 1. Scrub invisible characters
    new UnicodeNormalizer({
      id: 'unicode-normalizer',
      stripControlChars: true,
      collapseWhitespace: true,
    }),
    // 2. Detect the attempt
    new PromptInjectionDetector({
      id: 'prompt-injection-detector',
      model: openai('gpt-5-nano'), // Cheap, fast
      threshold: 0.8,
      strategy: 'block', // Hard stop
      detectionTypes: ['injection', 'jailbreak', 'system-override'],
    }),
  ],
});
```

[`UnicodeNormalizer`](https://mastra.ai/docs/processors) удаляет управляющие символы и сводит пробелы к единому виду. [`PromptInjectionDetector`](https://mastra.ai/docs/processors) анализирует очищенный ввод на наличие шаблонов, указывающих на попытку переопределить ваши инструкции.

Вызадаёте, насколько агрессивным будет обнаружение (параметр `threshold`) и что должно происходить, когда срабатывает правило (блокировать, логировать или просто помечать).

### 2. Обработка персональных данных (PII)

Номера кредитных карт в логах, номера социального страхования в векторных базах, адреса электронной почты, хранящиеся дольше, чем необходимо. Такие случаи быстро превращаются в регуляторные проблемы. Сложность в том, что пользователи не всегда понимают, что вставляют конфиденциальные данные в окно чата.

[`PIIDetector`](https://mastra.ai/docs/processors) сканирует на наличие типичных шаблонов до того, как данные попадут в вашу модель или будут записаны в хранилище:

```typescript
import { PIIDetector } from '@mastra/core/processors';

export const privateAgent = new Agent({
  id: 'privacy-first-assistant',
  name: 'privacy-first-assistant',
  instructions: 'You are a helpful assistant that never stores personal information.',
  model: openai('gpt-5'),
  inputProcessors: [
    new PIIDetector({
      id: 'pii-detector',
      model: openai('gpt-5-nano'),
      detectionTypes: ['email', 'phone', 'credit-card', 'ssn'],
      threshold: 0.6,
      strategy: 'redact',
      redactionMethod: 'mask',  // Replace with [REDACTED]
      instructions: 'Detect and mask personally identifiable information',
    }),
  ],
});
```

Можно выбрать редактирование (замена на `[REDACTED]`), хеширование или полное блокирование. Процессор работает как на входе, так и на выходе, поэтому вы защищены даже если модель каким‑то образом сгенерирует конфиденциальные данные в ответе.

### 3. Модерация контента

Модели, обученные на данных из интернета, видели многое. Без фильтрации они иногда могут генерировать ответы, которые заставят ваш отдел PR нервничать. [`ModerationProcessor`](https://mastra.ai/docs/processors) отлавливает контент, нарушающий ваши правила:

```typescript
import { ModerationProcessor } from '@mastra/core/processors';

export const moderatedAgent = new Agent({
  id: 'safe-assistant',
  name: 'safe-assistant',
  instructions: 'You are a helpful assistant for a community platform.',
  model: openai('gpt-5'),
  inputProcessors: [
    new ModerationProcessor({
      id: 'moderation-processor',
      model: openai('gpt-5-nano'),  // Быстрая, дешевая модель для классификации
      categories: ['hate', 'harassment', 'violence', 'self-harm'],
      threshold: 0.7,  // Блокировать, если уверенность > 70%
      strategy: 'block',  // Немедленно остановить запрос
      instructions: 'Detect harmful content that violates community guidelines',
    }),
  ],
});
```

Интересно то, что вы сами определяете, какие категории важны для вашего случая. Инструмент для творческого письма может позволять более выразительный контент, чем бот службы поддержки. Порог и стратегия дают вам контроль над тем, насколько строгой будет фильтрация.

## Когда происходит срабатывание

Процессоры не бросают исключения при обнаружении проблемы. Вместо этого они ставят флаг в объекте результата:

```typescript
const result = await secureAgent.generate('Ignore all previous instructions...');

if (result.tripwire) {
  console.log(`Blocked! Reason: ${result.tripwireReason}`);
  // "Blocked! Reason: Prompt injection detected."
  return "Nice try, script kiddie.";
}
```

Этот шаблон позволяет обрабатывать события безопасности так, как это имеет смысл для вашего приложения. Вы можете логировать их для анализа, возвращать общее сообщение об ошибке или даже разрешать некоторые нарушения в определённых контекстах. Поле `tripwireReason` точно указывает, какой процессор пометил контент, что облегчает отладку ложных срабатываний и настройку порогов.

## Что Это Не Решает

Процессоры ловят многое, но они не волшебные. Приготовленный атакующий с достаточным временем, скорее всего, найдёт запрос, который проскочит мимо. Модели иногда галлюцинируют так, как процессоры не могут предсказать. И всегда существует компромисс между безопасностью и гибкостью: чем строже правила, тем выше шанс заблокировать легитимные сценарии использования.

Ценность здесь не в идеальной защите. Это наличие системного подхода к обработке типичных проблем, которые неизбежно появятся в продакшене. Вы можете настраивать чувствительность по мере того, как узнаёте, что делают ваши пользователи. Можно добавить собственные процессоры для доменно‑специфических рисков. И у вас будет журнал аудита, показывающий, что было заблокировано и почему.

Большинство проблем безопасности в продакшн‑ИИ — это не изощрённые атаки. Это люди, копирующие и вставляющие данные, которые им не следует использовать, или обнаруживающие методом проб и ошибок, что бот делает то, чего вы не планировали. Процессоры не остановят каждую возможную проблему, но они делают очевидные векторы атак значительно труднее.
