## 🚀 مقدمة

قد يبدو استخراج عناوين URL من النص الخام أحيانًا وكأنه لعبة مملة من لعبة "اضرب الخلد". علامات الترقيم، والأقواس، والتنسيق الغامض تتآمر جميعًا لإحباط جهودك. سواء كنت تبني ماسح ويب، أو محلل بيانات، أو تطبيق دردشة، فإن استخراج عناوين URL بدقة أمر ضروري.

في هذه المقالة، سنتعامل مع المشكلة مباشرةً باستخدام نهج مرن من خطوتين. هدفنا هو **التقاط جميع السلاسل _المحتملة_ الشبيهة بعناوين URL أولاً** ثم التعامل مع التحقق في عملية لاحقة.

> 💡 **ملاحظة:** هذا النمط ليس للتحقق من صحة عناوين URL! إنه متسامح عمدًا مع علامات الترقيم والإملاء السيئ.

## 🔍 الهدف: استخراج عناوين URL من النص

عند استخراج عناوين URL من النص الخام، يكون النهج المكون من خطوتين فعالاً:

1. **التقاط كل ما يشبه عنوان URL**: ألقِ شبكة واسعة لالتقاط جميع السلاسل التي *قد* تكون عناوين URL. هنا يبرز "تعبيرنا النمطي الذي يزيد عن 120 بايت".
2. **التحقق**: بمجرد التقاط هذه المرشحات، استخدم فحوصات ثانوية (مثل تحليل DNS، المقارنة مع نطاقات معروفة) لاستبعاد الإدخالات غير الصالحة.

### تصور التحدي

غالبًا ما تُستخدم مصطلحات مثل `استخراج` و`تحليل` بالتبادل، إلا أنها تشير إلى عمليات متميزة. استخراج عناوين URL يتضمن تحديد والتقاط عناوين URL المحتملة من نص أكبر. من ناحية أخرى، تحليل عناوين URL يتضمن تفكيك هذه العناوين إلى أجزائها المكونة.

## التعبير النمطي الذي يزيد عن 120 بايت

فيما يلي تعبير نمطي موجز مصمم لاستخراج وتحليل عناوين URL في ضربة واحدة. يدعم بروتوكولات مختلفة، ونطاقات، ومسارات، وأقسام استعلام/جزء اختيارية.

لا تقلق — سنقوم بتفصيله خطوة بخطوة!

```js title="تعبير نمطي لعنوان URL يزيد عن 120 بايت" frame="code"
const urlRegex = /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;
// Compatibility: ES5+
```

// نفس النمط، مفصول بأسطر جديدة لسهولة القراءة:

([^-\/\.[\](|)\s?][^`\/\s\]?]+)
([-_a-z0-9!@$%^&*()=+;/~\.]*)
[?]?([^#\s`?]*)
[#]?([^#\s'"`\.,!]*)

## 🧩 تفصيل المكونات خطوة بخطوة

دعنا نحلل التعبير النمطي إلى مكوناته لفهم كيفية عمله:

<h3>1. البروتوكول (المجموعة 1): <code>{`([-.a-z0-9]+:\/{1,3})`}</code></h3>

<ul>
  <li>**الغرض:** يطابق جزء البروتوكول من عنوان URL (مثل `http://`، `ftp://`، `custom-scheme://`).</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[-.a-z0-9]+</code>: يطابق حرفًا أو أكثر من الأحرف الصغيرة أو الأرقام أو الواصلات أو النقاط (شائعة في مخططات البروتوكول).</li>
      <li><code>{`:\/{1,3}`}</code>: يطابق نقطتين متبوعتين بشرطة مائلة إلى ثلاث شرط مائلة (<code>:/</code>، <code>://</code>، أو <code>:///</code>).</li>
    </ul>
  </li>
</ul>

<h3>2. النطاق (المجموعة 2): <code>{`([^-\/\.[\](|)\s?][^\`\/\s\]?]+)`}</code></h3>

<ul>
  <li>**الغرض:** يلتقط جزء النطاق أو المضيف من عنوان URL.</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[^-\/\.[\](|)\s?]</code>: يطابق أي حرف باستثناء الأحرف الخاصة المحددة والمسافات البيضاء.</li>
      <li><code>[^`\/\s\]?]+</code>: يطابق حرفًا أو أكثر باستثناء علامات التنصيص الخلفية والشرط المائلة والمسافات البيضاء أو الأقواس المربعة المغلقة.</li>
    </ul>
  </li>
</ul>

<h3>3. المسار (المجموعة 3): <code>{`([-_a-z0-9!@$%^&*()=+;/~\\.]*)`}</code></h3>

<ul>
  <li>**الغرض:** يطابق مكون المسار من عنوان URL.</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[-_a-z0-9!@$%^&*()=+;/~\.]*</code>: يطابق صفرًا أو أكثر من الأحرف الآمنة في عناوين URL والموجودة عادةً في المسارات.</li>
    </ul>
  </li>
</ul>

<h3>4. الاستعلام (المجموعة 4): <code>[?]?([^#\s`?]*)</code></h3>

<ul>
  <li>**الغرض:** يطابق اختياريًا سلسلة استعلام، تبدأ بأي حرف <code>?</code>.</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[?]?</code>: يطابق اختياريًا <code>?</code>. (الأقواس المربعة ليست ضرورية تمامًا، لكنها أوضح قليلاً من الصيغة المختصرة المزدوجة <code>??</code>. كما أنها توفر توازيًا بصريًا مع مجموعة المطابقة التالية المشابهة <code>[#]?</code>.)</li>
      <li><code>([^#\s`?]*)</code>: يطابق صفرًا أو أكثر من الأحرف التي ليست علامة مربعة أو مسافة بيضاء أو علامة تنصيص خلفية أو علامة استفهام.</li>
    </ul>
  </li>
</ul>

<h3>5. المقطع (المجموعة 5): <code>[#]?([^#\s'"`\.,!]*)</code></h3>

<ul>
  <li>**الغرض:** يطابق اختياريًا معرف المقطع الذي يبدأ بـ <code>#</code>.</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[#]?</code>: يطابق اختياريًا <code>#</code>.</li>
      <li><code>([^#\s'"`\.,!]*)</code>: يطابق صفرًا أو أكثر من الأحرف التي ليست علامات ترقيم محظورة أو مسافات بيضاء.</li>
    </ul>
  </li>
</ul>

## 🛠️ مثال على التحليل

إليك كيفية استخدام هذا التعبير النمطي الضخم مع القليل من JavaScript:

<CodeTabs client:only
 tabs={[
    "الكود: استخراج عناوين URL",
    "النتائج: عناوين URL المستخرجة",
    "النتائج: أجزاء URL",
  ]} >
```js title="extract-urls.js" frame="code"
const text = `
Check this out: https://example.com/path?query=123#section
And also (ftp://files.server.org/index).
Plus a weird one: custom-scheme://host/param;weird^stuff
`;

const urlRegex =
  /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;

const matches = [
  ...text.matchAll(urlRegex),
].map((match) => match[0]);
console.log("Extracted URLs:", matches);

const parts = [
  ...text.matchAll(urlRegex),
].map((match) => match.slice(1));
console.log("Extracted Parts:", parts);
```

  "ftp://files.server.org/index",
  "custom-scheme://host/param;weird^stuff"
]
```

    "section"      // Fragment
  ],
  [
    "ftp://",           // Protocol
    "files.server.org", // Domain
    "/index",           // Path
    "",                 // Query
    ""                  // Fragment
  ],
  [
    "custom-scheme://",   // Protocol
    "host",               // Domain
    "/param;weird^stuff", // Path
    "",                   // Query
    ""                    // Fragment
  ]
]
```

## ☑️ الخطوات التالية

اعتمادًا على حالة الاستخدام الخاصة بك، قد تحتاج إلى تحسين هذا التعبير النمطي أو إضافة المزيد من خطوات التحقق والمعالجة اللاحقة.

### مشاريع مختلفة واحتياجات متباينة

للمشاريع متطلبات ومخاوف أمنية متنوعة:

١. **استخراج البيانات من الويب**: التحقق من صحة عناوين URL لضمان إمكانية الوصول إليها وموثوقيتها.
٢. **معالجة البيانات**: استخراج عناوين URL من المحتوى الذي ينشئه المستخدم مع ضمان الأمان.
٣. **تحليل البيانات**: تصفية الروابط المكررة أو غير ذات الصلة لأغراض البحث أو التسويق.
٤. **التطبيقات الموجهة للمستخدم**: ربط عناوين URL تلقائيًا في تطبيقات الدردشة أو المنتديات.

### المعالجة اللاحقة والتحقق

بعد جمع عناوين URL المحتملة، قم بتطبيق فحوصات إضافية:

- **بحث DNS**: التحقق من أن النطاقات قابلة للحل.
- **فحوصات الأمان**: استخدام خدمات للتحقق من المواقع الضارة أو التصيد.
- **قواعد مخصصة**: تطبيق مرشحات خاصة بالمشروع (مثل نطاقات المستوى الأعلى المسموح بها، الحد الأقصى لطول عنوان URL).

## 📝 ملخص

قد يكون استخراج بيانات السلاسل شبه المنظمة أكثر الأجزاء إرضاءً في إتقان التعبيرات النمطية.

فيما يلي ملخص للنقاط الرئيسية:

- **استخدم أداة مرئية لكتابة واختبار** وفهم [أنماط التعبيرات النمطية.](https://regex101.com/r/jO8bC4/69)
- **قسّم التحدي إلى أجزاء** وحل كل جزء على حدة. بمعنى ما، توفر لنا مجموعات الالتقاط 'علامات مسار' مجازية لتعبيرنا النمطي.
- **استخدم تعبيرات مطابقة 'فضفاضة'، وتجنب الالتزام الصارم بالمواصفات** عند إدخال البيانات.
- **تطبيق خطوات التحقق** بعد الاستخراج الأولي أمر أساسي—ضع دائمًا في اعتبارك أمان مشروعك واحتياجاته الخاصة.

باتباع هذه الخطوات، يمكنك استخراج أي بيانات نصية شبه منظمة بفعالية، مما يضع الأساس لمزيد من المعالجة والتحقق.
