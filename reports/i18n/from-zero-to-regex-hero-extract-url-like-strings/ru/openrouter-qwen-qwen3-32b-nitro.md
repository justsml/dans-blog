````mdx
## Регулярное выражение на 120+ байт

Ниже приведено компактное регулярное выражение, предназначенное для извлечения и разбора URL за один шаг. Оно поддерживает различные протоколы, домены, пути и необязательные разделы запроса/фрагмента.

Не волнуйтесь — мы разберем его пошагово!

```js title="Регулярное выражение на 120+ байт" frame="code"
const urlRegex = /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;
// Совместимость: ES5+

// Тот же паттерн, разбитый на строки для наглядности:
([-.a-z0-9]+:\/{1,3})
([^-\/\.[\](|)\s?][^`\/\s\]?]+)
([-_a-z0-9!@$%^&*()=+;/~\.]*)
[?]?([^#\s`?]*)
[#]?([^#\s'"`\.,!]*)

```

## 🧩 Пошаговое разборка

Давайте разберем регулярное выражение по компонентам, чтобы понять, как оно работает:

<h3>1. Протокол (Группа 1): <code>{`([-.a-z0-9]+:\/{1,3})`}</code></h3>

<ul>
  <li>**Цель:** Сопоставляет протокольную часть URL (например, `http://`, `ftp://`, `custom-scheme://`).</li>
  <li>
    **Объяснение:**
    <ul>
      <li><code>[-.a-z0-9]+</code>: Сопоставляет одну или более строчных букв, цифр, дефисов или точек (часто встречающихся в схемах протоколов).</li>
      <li><code>{`:\/{1,3}`}</code>: Сопоставляет двоеточие, за которым следуют один-три слэша (<code>:/</code>, <code>://</code> или <code>:///</code>).</li>
    </ul>
  </li>
</ul>

<h3>2. Домен (Группа 2): <code>{`([^-\/\.[\](|)\s?][^\`\/\s\]?]+)`}</code></h3>

<ul>
  <li>**Цель:** Захватывает доменную или хостовую часть URL.</li>
  <li>
    **Объяснение:**
    <ul>
      <li><code>[^-\/\.[\](|)\s?]</code>: Сопоставляет любой символ, кроме указанных специальных символов и пробелов.</li>
      <li><code>[^`\/\s\]?]+</code>: Сопоставляет один или более символов, кроме обратных галочек, слешей, пробелов или закрывающих квадратных скобок.</li>
    </ul>
  </li>
</ul>

<h3>3. Путь (Группа 3): <code>{`([-_a-z0-9!@$%^&*()=+;/~\\.]*)`}</code></h3>

<ul>
  <li>**Цель:** Сопоставляет компонент пути URL.</li>
  <li>
    **Пояснение:**
    <ul>
      <li><code>[-_a-z0-9!@$%^&*()=+;/~\.]*</code>: Сопоставляет ноль или более URL-совместимых символов, часто встречающихся в путях.</li>
    </ul>
  </li>
</ul>

<h3>4. Запрос (Группа 4): <code>[?]?([^#\s`?]*)</code></h3>

<ul>
  <li>**Цель:** Необязательно сопоставляет строку запроса, начинающуюся с символа <code>?</code>.</li>
  <li>
    **Пояснение:**
    <ul>
      <li><code>[?]?</code>: Необязательно сопоставляет <code>?</code>. (Квадратные скобки не обязательны, но они немного яснее, чем сокращённая форма <code>??</code>. Также это создаёт визуальное параллелизм с аналогичной следующей группой <code>[#]?</code>.)</li>
      <li><code>([^#\s`?]*)</code>: Сопоставляет ноль или более символов, не являющихся решёткой, пробелами, обратными апострофами или вопросительными знаками.</li>
    </ul>
  </li>
</ul>

<h3>5. Фрагмент (Группа 5): <code>[#]?([^#\s'"`\.,!]*)</code></h3>

<ul>
  <li>**Цель:** Необязательно сопоставляет идентификатор фрагмента, начинающийся с <code>#</code>.</li>
  <li>
    **Пояснение:**
    <ul>
      <li><code>[#]?</code>: Необязательно сопоставляет <code>#</code>.</li>
      <li><code>([^#\s'"`\.,!]*)</code>: Сопоставляет ноль или более символов, не являющихся запрещённой пунктуацией или пробелами.</li>
    </ul>
  </li>
</ul>

const text = `

const urlRegex =
  /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;

const matches = [
  ...text.matchAll(urlRegex),
].map((match) => match[0]);
console.log("Извлечённые URL:", matches);

const parts = [
  ...text.matchAll(urlRegex),
].map((match) => match.slice(1));
console.log("Извлечённые Части:", parts);
```

### Разные проекты, разные требования

Проекты имеют разные требования и проблемы с безопасностью:

1. **Веб-скрапинг**: Проверять URL на доступность и надежность.  
2. **Обработка данных**: Извлекать URL из пользовательского контента с учетом безопасности.  
3. **Анализ данных**: Удалять дубликаты или нерелевантные ссылки для исследований или маркетинга.  
4. **Пользовательские приложения**: Автоматически превращать URL в гиперссылки в чатах или форумах.  

### 🧹 Постобработка и проверка  

После сбора потенциальных URL примените дополнительные проверки:  

- **Проверка DNS**: Убедиться, что домены разрешаются.  
- **Проверки безопасности**: Использовать сервисы для проверки на вредоносные или фишинговые сайты.
````
