## Docker-Server-Setup

> Historische Anmerkung: Dies ist ein 2015er-Referenz für lokale Entwicklungsumgebungen. Sie ist nützlich, um den alten Workflow zu verstehen, aber behandeln Sie diese Code-Snippets nicht als aktuelle Produktivumgebungsempfehlung. Moderne Docker-Compose-Dateien, benutzerdefinierte Netzwerke, fixierte Image-Versionen, Authentifizierung, Geheimnisse und überprüfte Update-Automatisierung sind heute sicherere Standards.

### Für wen ist diese Anleitung?

- Haben Sie jemals gewünscht, eine App mit einer 'wegwerf'-Datenbank zu testen?
- Haben Sie ein verdächtiges Code-Repository geerbt? Teilen Sie lieber nicht den Zugriff auf Ihre bestehende Datenbank?
- Arbeiten Sie mit sicherheitskritischen Kunden? Riskieren Sie nicht Kreuzkontamination! Nutzen Sie Container und steuern Sie Datenpersistenz!
- Können Sie Ihr Entwicklungs-Setup nicht auf die neueste Datenbankversion aktualisieren, weil Ihre Legacy-Apps von einer 12 Jahre alten MySQL-Version abhängen?

> Lassen Sie diese Gründe nie wieder Ihre Arbeit behindern!

### PostgreSQL-Server  

> Starten Sie einen Container mit dem Namen `pg-localhost`  

```sh
# Speichern Sie DB-Dateien an einem lokalen Pfad außerhalb des Containers  
mkdir -p $HOME/.postgres-data  

docker run \  
  --name pg-localhost \  
  -p 127.0.0.1:5432:5432 \  
  -e POSTGRES_PASSWORD=password \  
  --restart unless-stopped \  
  -it \  
  --shm-size=256mb \  
  postgres:16-alpine \  
  postgres -c 'listen_addresses=*' \  
    -c 'password_encryption=scram-sha-256' \  
    -c 'shared_memory_type=sysv' \  
    -c 'shared_buffers=256MB' \  
    -c 'max_connections=200'  
```  

> Passen Sie die Befehlszeilenoptionen nach Bedarf an. (Die PostgreSQL-Daemon-Argumente beginnen nach dem Docker-Imagename `postgres:16-alpine`. Siehe `postgres -c 'listen_addresses=*'...`)  

> Zugriff auf den `psql`-Prompt als Benutzer postgres  

```sh
docker exec --user postgres -it pg-localhost psql  
```  

> Zugriff auf die Shell des Containers als Root  

```sh
docker exec -it pg-localhost bash  
```  

**Hinweis:** Der obige Befehl verwendet die offiziellen Alpine-Linux-Base-Images. _Es handelt sich nicht um ein typisches Debian-Umgebung._  

> Um Debian/Ubuntu-Base-Images zu verwenden, ändern Sie `postgres:12-alpine` in `postgres:12`.  

### MongoDB-Server  

```sh
mkdir -p $HOME/.mongodb/data  

docker run -d \  
  --name mongodb \  
  --restart on-failure:5 \  
  -p 127.0.0.1:27017:27017 \  
  -v $HOME/.mongodb:/data \  
  mongo:7 \  
  bash -c 'mongod --bind_ip 0.0.0.0'  
```  

Nachdem Ihr Server eingerichtet ist, überprüfen Sie Ihre Daten an `$HOME/.mongodb` mit:  

```sh
ls -lach $HOME/.mongodb  
```  

Stellen Sie eine Verbindung zum Server mit dem `mongosh`-CLI-Tool her. (Falls Sie es nicht installiert haben, siehe unten.)

```sh
#> Verwenden der Standardargumente:
mongosh
```

### MySQL-Server

> **WARNUNG:** ÄNDERN SIE DAS PASSWORT IN `MYSQL_ROOT_PASSWORD` UNTER.

```sh
mkdir -p $HOME/.mysql

docker run -d \
  -v $HOME/.mysql:/var/lib/mysql \
  -p 127.0.0.1:3306:3306 \
  --name mysql-$USER \
  -e MYSQL_DATABASE=$USER \
  -e MYSQL_ROOT_HOST='172.*.*.*' \
  -e MYSQL_ROOT_PASSWORD='p@ssw0rd' \
  mysql/mysql-server:8
```

### ElasticSearch-Server

```sh
mkdir -p $HOME/.elastic

docker run -d \
  --name elastic \
  -p 127.0.0.1:9200:9200 \
  -p 127.0.0.1:9300:9300 \
  -v $HOME/.elastic:/data \
  docker.elastic.co/elasticsearch/elasticsearch:8.15.5 bash -c 'elasticsearch --cluster.name elastic_cluster --node.name elastic01 --path.data /data/elastic-data --path.logs /data/elastic-logs '
```

##### Sicherheitshinweise

> **HINWEIS:** Die `-p 127.0.0.1:27017:27017`-Art der Portoption verhindert den Zugriff auf Ihre Instanz außer vom localhost-Netzwerk des Docker-Servers.  
> Um die veröffentlichten Ports freizugeben, entfernen Sie das lokale IP-Präfix, um externen Zugriff zu erlauben: `-p 27017:27017`. **Stellen Sie sicher, dass Sie notwendige Sicherheitsvorkehrungen getroffen haben.**

**Empfohlen:** Verwenden Sie immer ein Port-Scanning-Tool (z. B. nmap/masscan), um Ihre Netzwerkkonfiguration zu überprüfen (von einem separaten System in einem anderen Netzwerk).

## Ein NodeJS-Web-App verpacken  

1.  Erstellen Sie eine leere Datei namens `Dockerfile` im Wurzelverzeichnis Ihres Projekts.  
1.  _(Optional, empfohlen)_ Fügen Sie eine `.dockerignore`-Datei hinzu, die `.gitignore`-Regeln verwendet, um große nicht-essentielle Pfade auszuschließen. Standardmäßig werden alle Projektdateien eingeschlossen.  

#### Erstellen Sie eine `Dockerfile` in der Wurzel Ihres Projekts  

```dockerfile
# Beispiel für NodeJS
FROM node:22
EXPOSE 3000
COPY . /app/
WORKDIR /app
RUN apt-get update && apt-get dist-upgrade -yqq
RUN ["npm", "install"]

# Übertragbarer Befehl
CMD ["npm", "start"]
```

Es ist einfacher, zu zeigen, wie man die Dockerfile verwendet und die Ergebnisse über die Konsole demonstriert (siehe Befehle unten).  

Im Terminal wechseln Sie mit `cd` in Ihr Projektverzeichnis und führen den folgenden `build`-Befehl **jedes Mal** aus, wenn Sie Änderungen bereitstellen – oder wenn Sie das Betriebssystem oder die Umgebungs-Konfiguration ändern/upgraden möchten.  

```sh
docker build -t app-name-here .
```

### Wichtige Docker-Befehle im Überblick  

#### Docker-Image erstellen  

```sh
docker build -t app-name-here .
```

#### Web-App erstellen/ausführen mit Links zu DB-Servern  

```sh
docker network create app-local
docker network connect app-local mongodb
docker network connect app-local elastic
docker run -d --name webapp01 --network app-local -p 3000:3000 app-name-here
```

#### Interaktiv ausführen (kein Daemon, im Terminal)

```sh
docker run -it --name webapp01 --network app-local -p 3000:3000 app-name-here bash
```

#### Container-Instanz oder Image löschen

> Wichtig: Alle Daten, die nicht auf einem eingehängten Volume-Pfad gespeichert sind, gehen verloren!!

```sh
# Image löschen
docker rmi -f app-name-here
docker rm -f webapp01
# jetzt erneut deinen `docker run...` aus ^^^ ausführen
# Beispiel: Beende deine oben genannten DB-Instanzen, führe aus: ( beginne z.B. mit `docker stop {mongo,elastic}` )
docker rm -f mongo elastic
```
