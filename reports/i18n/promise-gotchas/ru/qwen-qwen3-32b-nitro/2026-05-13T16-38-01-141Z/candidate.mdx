### Промисы не работают как другие значения

Вы не можете напечатать их значение как большинство значений:

```js
// Это не имеет смысла с промисами:
console.log(Promise.resolve(42));

// Мы должны использовать интерфейс `.then`:
Promise.resolve(42).then(value => console.log(value));
```

### Промисы не предупреждают вас при совершении ошибки

Ну, вероятная ошибка.

По разным причинам было решено в TC39, что `.then` и `.catch` могут принимать null. Например, `.then(null, null)` является допустимым, и требуемое поведение — пропустить этот 'шаг' в цепочке.

Печальный результат этого в том, что очень легко всё испортить.

##### На примере

Давайте рассмотрим небольшой пример: какой из следующих вариантов выведет в консоль 42?

```js
// Вариант #1:
Promise.resolve(42).then(console.log());

// Вариант #2:
Promise.resolve(42).then(console.log);

// Вариант #3:
Promise.resolve(42).then(value => console.log(value));

// Вариант #4:
Promise.resolve(42)
  .then(console.log())
  .then(console.log);
```

##### Ответ

Ответ: #2, #3 и #4.

Почему? Давайте посмотрим на **типы** аргументов, переданных в `.then()`:

```js
var arg1 = console.log();
var arg2 = console.log;
var arg3 = value => console.log(value);

typeof arg1 === "undefined";
typeof arg2 === "function";
typeof arg3 === "function";
```

Все еще думаете, как работает 4-й вариант?

Это эквивалентно выполнению следующего:

```js
// Вариант #4 - фактически
Promise.resolve(42)
  .then(undefined) // это не повлияет на значение, оно будет передано следующему `.then(fn)`
  .then(console.log);
```
