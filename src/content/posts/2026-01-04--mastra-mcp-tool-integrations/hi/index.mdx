आपने एक AI एजेंट बनाया है। शायद यह एक अच्छा भी है। प्रॉम्प्ट ठीक हैं, मॉडल तेज़ है, और जवाब प्राकृतिक लगते हैं।

लेकिन फिर कोई इससे Salesforce से एक ग्राहक रिकॉर्ड चेक करने को कहता है। या नवीनतम Jira टिकट पुल करने को। या आपकी आंतरिक दस्तावेज़ीकरण खोजने को।

और आपका खूबसूरत एजेंट बस... कर नहीं पाता।

यह वह एकीकरण समस्या है जिससे हर AI प्लेटफ़ॉर्म अंततः टकराता है। आपके एजेंट को हाथों की जरूरत है। उसे आपके असली व्यापार प्रणालियों में आंखें चाहिए। बिना उनके, आप बस एक महंगा चैटबॉट चला रहे हैं।

पारंपरिक समाधान? हर सेवा के लिए एक कस्टम API रैपर लिखें जिससे आप जुड़ना चाहते हैं। उनके डॉक्स पढ़ें, उनका auth संभालें, उनकी rate limits से निपटें, प्रार्थना करें कि अगले महीने उनके endpoints न बदलें। फिर अगले सेवा के लिए यही करें। और अगले।

मॉडल कॉन्टेक्स्ट प्रोटोकॉल इस गणित को पूरी तरह बदल देता है।

---

## MCP वास्तव में क्या हल करता है

USB-C से पहले के USB के बारे में सोचें। आपके पास Mini-USB, Micro-USB, मालिकाना Apple कनेक्टर थे, और एक दराज भरा केबल जो केवल विशिष्ट उपकरणों के साथ काम करते थे। USB-C ने बस एक नया कनेक्टर नहीं जोड़ा—इसने एक मानक स्थापित किया जिसने मतलब दिया कि कोई भी केबल किसी भी उपकरण के साथ काम कर सकता है।

MCP AI टूल एकीकरण के लिए यही कर रहा है।

इसके बजाय कि आप अपने एजेंट को Salesforce, HubSpot, GitHub, या किसी अन्य सेवा से जोड़ने के लिए कस्टम कोड लिखें, आप एक बार प्रोटोकॉल लागू करते हैं (या एक पहले से बना सर्वर डाउनलोड करते हैं), और कोई भी MCP-संगत एजेंट तुरंत उससे बात कर सकता है।

प्रोटोकॉल संचार परत को संभालता है। आप बस यह परिभाषित करते हैं कि आपके टूल क्या करते हैं और उन्हें क्या डेटा चाहिए।

---

## कई एकीकरण सेट अप करना

Mastra में अपने [`MCPClient`](https://mastra.ai/docs/mcp/overview) के माध्यम से MCP के लिए मूल समर्थन है। आप लोकल टूल (child processes के रूप में चलने वाले) और रिमोट सेवाओं (अपने स्वयं के इन्फ्रास्ट्रक्चर पर चलने वाली) दोनों को जोड़ सकते हैं।

यहाँ Google Maps रूटिंग, एक मौसम सेवा, और स्थानीय Wikipedia खोज को जोड़ने का एक यथार्थवादी प्रोडक्शन सेटअप है:

```typescript
// src/mastra/mcp/index.ts
import { MCPClient } from '@mastra/mcp';

export const mcpClient = new MCPClient({
  servers: {
    // लोकल टूल (Stdio)
    wikipedia: {
      command: 'npx',
      args: ['-y', 'wikipedia-mcp'],
    },
    // Maps & Navigation (Remote/HTTP)
    googleMaps: {
      url: new URL(process.env.GOOGLE_MAPS_MCP_URL!),
      requestInit: {
        headers: {
          Authorization: `Bearer ${process.env.GOOGLE_MAPS_API_KEY}`,
        },
      },
    },
    // मौसम सेवा एकीकरण
    weather: {
      url: new URL('https://mcp.weatherapi.dev/v1'),
      requestInit: {
        headers: {
          'X-API-Key': process.env.WEATHER_API_KEY!,
        },
      },
    },
  },
});
```

क्लाइंट कनेक्शन जीवनचक्र प्रबंधित करता है, लोकल टूल के लिए प्रोसेस स्पॉनिंग संभालता है, और रिमोट सर्वर के लिए HTTP कनेक्शन बनाए रखता है। आप सॉकेट या stdio को सीधे नहीं छूते।

## एजेंट से टूल जोड़ना

एक बार जब आपके पास अपना MCP क्लाइंट कॉन्फ़िगर हो जाए, तो उन टूल को एक एजेंट को देना सीधा है:

```typescript
// src/mastra/agents/navigation-agent.ts
import { Agent } from '@mastra/core/agent';
import { openai } from '@ai-sdk/openai';
import { mcpClient } from '../mcp';

export const navigationDirectionsAgent = new Agent({
  id: 'navigation-directions-agent',
  name: 'Navigation & Directions Assistant',
  instructions: `You are a helpful navigation assistant that provides route planning and travel advice.
    - Always confirm the start and destination locations
    - Use Google Maps tools to find optimal routes
    - Check weather conditions along the route
    - Provide estimated travel times and suggest alternatives if weather is poor
    - Include relevant details like traffic, road conditions, and points of interest
    - Keep responses clear and actionable`,
  model: openai('gpt-5'),
  tools: await mcpClient.getTools(), // <--- यही जादू की पंक्ति है
});
```

जब कोई उपयोगकर्ता पूछता है: *"सैन फ्रांसिस्को से लेक टाहो के लिए सबसे अच्छा रूट क्या है, और क्या मुझे मौसम की चिंता होनी चाहिए?"*

एजेंट उपलब्ध टूल परिभाषाओं को पढ़ता है, को समझता है कि उसके पास Google Maps रूटिंग और मौसम पूर्वानुमान टूल हैं, उन्हें सही पैरामीटर के साथ निष्पादित करता है, और एक विकल्प रूट के साथ साथ रास्ते के मौसम की स्थिति के बारे में जवाब देता है।

आपने एक भी लाइन Google Maps API कोड या मौसम सेवा एकीकरण नहीं लिखा।

## प्रति-उपयोगकर्ता प्रमाणीकरण

यहाँ एक सुरक्षा गलती है जो करना आसान है: क्रेडेंशियल्स हार्डकोड करना।

यदि आप अपने एनवायरनमेंट वेरिएबल्स में एक Google Maps API key डालते हैं और उसे दिन मान लेते हैं, तो हर उपयोगकर्ता एक ही quota और rate limits साझा करता है। अधिक महत्वपूर्ण बात, यदि आप उन सेवाओं का उपयोग कर रहे हैं जो उपयोगकर्ता प्राथमिकताओं को स्टोर करती हैं (जैसे सहेजे गए स्थान या पसंदीदा रूट), तो हर कोई एक ही डेटा देखेगा। यह डेमो के लिए ठीक है। प्रोडक्शन में यह एक दायित्व है।

Mastra इसे आपको उपयोगकर्ता-विशिष्ट क्रेडेंशियल्स के साथ गतिशील रूप से MCP क्लाइंट बनाने देकर संभालता है:

```typescript
async function handleUserRequest(userPrompt: string, userCredentials: UserCreds) {
  // THIS specific user के लिए एक क्लाइंट बनाएं
  const userMcp = new MCPClient({
    servers: {
      googleMaps: {
        url: new URL(process.env.GOOGLE_MAPS_MCP_URL!),
        requestInit: {
          headers: {
            // उपयोगकर्ता का विशिष्ट API key या token
            Authorization: `Bearer ${userCredentials.mapsApiKey}`,
            'X-User-ID': userCredentials.userId,
          },
        },
      },
    },
  });

  const agent = mastra.getAgent('navigationDirectionsAgent');
  
  // रनटाइम पर टूल इंजेक्ट करें
  const response = await agent.generate(userPrompt, {
    toolsets: await userMcp.getToolsets(),
  });

  return response;
}
```

प्रत्येक उपयोगकर्ता को अपना अलग पृथक टूलसेट मिलता है जिसमें उनके अपने API quotas और प्राथमिकताएं हैं। उपयोगकर्ता A के सहेजे गए स्थान निजी रहते हैं, उपयोगकर्ता B का रूट इतिहास अलग रहता है। यही तरीका है जिससे मल्टी-टेनेंट SaaS एजेंट व्यवहार में काम करते हैं।

## कम्पोजिट टूल बनाना

कभी-कभी आपको एक ही ऑपरेशन में कई MCP टूल को जोड़ने की जरूरत होती है। शायद आप एक ऐसा रूट प्लान करना चाहते हैं जो रास्ते में वास्तविक समय ट्रैफ़िक और मौसम की स्थिति दोनों को ध्यान में रखे।

आप कस्टम टूल परिभाषाओं में MCP टूल रैप कर सकते हैं:

```typescript
export const smartRouteTool = createTool({
  id: 'smart-route-planner',
  description: 'ट्रैफ़िक और मौसम की स्थिति को ध्यान में रखते हुए विकल्प रूट की योजना बनाता है',
  execute: async ({ context, mastra }) => {
    // कच्चे टूल प्राप्त करें
    const tools = await mcpClient.getTools();
    
    // 1. Google Maps से बेस रूट प्राप्त करें
    const routeData = await tools.googleMaps_getDirections.execute({ 
      context: { 
        origin: context.origin,
        destination: context.destination 
      } 
    });
    
    // 2. रास्ते में मौसम जांचें
    const weatherData = await tools.weather_getForecast.execute({
      context: { coordinates: routeData.waypoints }
    });
    
    // 3. मौसम चेतावनियों के साथ बेहतर रूट लौटाएं
    return { 
      ...routeData, 
      weatherAlerts: weatherData.alerts,
      recommendation: weatherData.severe ? 'यात्रा विलंब करने पर विचार करें' : 'यात्रा के लिए सुरक्षित'
    };
  },
});
```

यह आपको बारीक नियंत्रण देता है कि टूल कैसे इंटरैक्ट करते हैं जबकि भारी काम के लिए अभी भी MCP प्रोटोकॉल का लाभ उठाते हैं।

## यह कहाँ जाता है

हर सेवा के लिए जिससे आपके AI एजेंट को बात करने की जरूरत है, कस्टम API क्लाइंट लिखना कभी टिकाऊ नहीं था। यह बुरी तरह स्केल होता है, अक्सर टूटता है, और आपके प्लेटफ़ॉर्म को विशिष्ट कार्यान्वयन से बांधता है।

MCP हर एकीकरण चुनौती को हल नहीं करता—auth अभी भी जटिल है, rate limiting अभी भी मायने रखती है, और हर सेवा के पास अभी तक MCP सर्वर नहीं है। लेकिन यह एक नींव स्थापित करता है जो एजेंट प्लेटफ़ॉर्म बनाना काफी कम दर्दनाक बनाता है।

यदि आप एक AI सिस्टम का आर्किटेक्ट कर रहे हैं जिसे बाहरी सेवाओं के साथ इंटरैक्ट करने की जरूरत है, तो MCP को समझना शायद आपके समय के लायक है।
