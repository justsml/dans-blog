## 🚀 المقدمة

استخراج الروابط من النص الخام قد يشعر أحيانًا كأنك تلعب لعبة شاقة من “ضرب الفئران”. علامات الترقيم، الأقواس، والتنسيق الغامض كلها تتآمر لتعقّب جهودك. سواءً كنت تبني أداة جمع بيانات ويب، محلل بيانات، أو تطبيق دردشة، فإن استخراج الروابط بدقة أمر أساسي.

في هذه المقالة، سنواجه المشكلة مباشرةً باستخدام نهج مرن من خطوتين. هدفنا هو **التقاط جميع السلاسل التي قد تشبه الروابط أولًا** ثم التعامل مع التحقق في عملية لاحقة.

> 💡 **ملاحظة:** هذا النمط ليس لـ **_التحقق من صحة_** الروابط! إنه متساهل عمدًا مع علامات الترقيم والأخطاء الإملائية.

## 🔍 الهدف: استخراج الروابط من النص

عند استخراج الروابط من النص الخام، يكون النهج ذو الخطوتين فعالًا:

1. **التقاط كل ما يشبه الرابط**: صِدّ شبكة واسعة لالتقاط جميع السلاسل التي *قد* تكون روابط. هنا يبرز «تعبير regex بحجم 120+ بايت».
2. **التحقق**: بعد أن تلتقط هذه المرشحات، استخدم فحوصًا ثانوية (مثل حل DNS، أو المقارنة مع نطاقات معروفة) لتصفية الإدخالات غير الصالحة.

### تصور التحدي

تُستَخدم مصطلحات `extract` و `parse` غالبًا بصورة متبادلة، لكنهما تشير إلى عمليتين مميزتين. استخراج الروابط يعني تحديد والتقاط الروابط المحتملة من نص أكبر. أما التحليل (parsing) فيعني تفكيك هذه الروابط إلى أجزائها المكوّنة.

## تعبير regex بحجم 120+ بايت

فيما يلي تعبير نمطي مختصر صُمم لاستخراج وتحليل الروابط في خطوة واحدة. يدعم بروتوكولات مختلفة، ونطاقات، ومسارات، وأقسام استعلام/جزء اختيارية.

لا تقلق — سنفصّله خطوة بخطوة!

```js title="120+ Byte URL Regex" frame="code"
const urlRegex = /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;
// Compatibility: ES5+
// نفس النمط، مقسّم على أسطر لسهولة القراءة:
([-.a-z0-9]+:\/{1,3})
([^-\/\.[\](|)\s?][^`\/\s\]?]+)
([-_a-z0-9!@$%^&*()=+;/~\.]*)
[?]?([^#\s`?]*)
[#]?([^#\s'"`\.,!]*)

```

## 🧩 تفصيله خطوة بخطوة

دعونا نفكك التعبير النمطي إلى مكوّناته لفهم طريقة عمله:

<h3>1. البروتوكول (المجموعة 1): <code>{`([-.a-z0-9]+:\/{1,3})`}</code></h3>

<ul>
  <li>**الغرض:** يطابق جزء البروتوكول من الرابط (مثل `http://`، `ftp://`، `custom-scheme://`).</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[-.a-z0-9]+</code>: يطابق حرفًا أو أكثر من الأحرف الصغيرة، الأرقام، الشرطات أو النقاط (شائع في مخططات البروتوكول).</li>
      <li><code>{`:\/{1,3}`}</code>: يطابق نقطتين متبوعتين بواحد إلى ثلاثة شرطات مائلة (<code>:/</code>، <code>://</code>، أو <code>:///</code>).</li>
    </ul>
  </li>
</ul>

<h3>2. النطاق (المجموعة 2): <code>{`([^-\/\.[\](|)\s?][^\`\/\s\]?]+)`}</code></h3>

<ul>
  <li>**الغرض:** يلتقط جزء النطاق أو المضيف من الرابط.</li>
  <li>
    **الشرح:**
    <ul>
      <li><code>[^-\/\.[\](|)\s?]</code>: يطابق أي حرف ما عدا الأحرف الخاصة المحددة والمسافات.</li>
      <li><code>[^`\/\s\]?]+</code>: يطابق حرفًا أو أكثر ما عدا العلامات العكسية، الشرطات المائلة، المسافات، أو القوس المربع المغلق.</li>
    </ul>
  </li>
</ul>

<h3>3. المسار (المجموعة 3): <code>{`([-_a-z0-9!@$%^&*()=+;/~\\.]*)`}</code></h3>

<ul>
  <li>**الغرض:** يطابق مكوّن المسار في عنوان URL.</li>
  <li>
    **التفسير:**
    <ul>
      <li><code>[-_a-z0-9!@$%^&*()=+;/~\.]*</code>: يطابق صفر أو أكثر من الأحرف الآمنة في URL والتي تُستخدم عادةً في المسارات.</li>
    </ul>
  </li>
</ul>

<h3>4. الاستعلام (المجموعة 4): <code>[?]?([^#\s`?]*)</code></h3>

<ul>
  <li>**الغرض:** يطابق اختياريًا سلسلة الاستعلام، بدءًا بأي حرف <code>?</code>.</li>
  <li>
    **التفسير:**
    <ul>
      <li><code>[?]?</code>: يطابق اختياريًا <code>?</code>. (الأقواس المربعة ليست ضرورية تمامًا، لكنها أكثر وضوحًا قليلًا من <code>??</code> المتقربة، وتوفر تماثلًا بصريًا مع المجموعة التالية <code>[#]?</code>.)</li>
      <li><code>([^#\s`?]*)</code>: يطابق صفر أو أكثر من الأحرف التي ليست علامة شباك، أو مسافة بيضاء، أو علامة اقتباس عكسية، أو علامة سؤال.</li>
    </ul>
  </li>
</ul>

<h3>5. الجزء المقطّع (المجموعة 5): <code>[#]?([^#\s'"`\.,!]*)</code></h3>

<ul>
  <li>**الغرض:** يطابق اختياريًا معرف الجزء المقطّع بدءًا بـ <code>#</code>.</li>
  <li>
    **التفسير:**
    <ul>
      <li><code>[#]?</code>: يطابق اختياريًا <code>#</code>.</li>
      <li><code>([^#\s'"`\.,!]*)</code>: يطابق صفر أو أكثر من الأحرف التي ليست علامات ترقيم محظورة أو مسافات بيضاء.</li>
    </ul>
  </li>
</ul>

## 🛠️ مثال على التحليل

إليك كيفية تشغيل هذا التعبير النمطي الضخم باستخدام قليل من JavaScript:

<CodeTabs client:only

```js title="extract-urls.js" frame="code"
const text = `
Check this out: https://example.com/path?query=123#section
And also (ftp://files.server.org/index).
Plus a weird one: custom-scheme://host/param;weird^stuff
`;

const urlRegex =
  /([-.a-z0-9]+:\/{1,3})([^-\/\.[\](|)\s?][^`\/\s\]?]+)([-_a-z0-9!@$%^&*()=+;/~\.]*)[?]?([^#\s`?]*)[#]?([^#\s'"`\.,!]*)/gi;

const matches = [
  ...text.matchAll(urlRegex),
].map((match) => match[0]);
console.log("Extracted URLs:", matches);

const parts = [
  ...text.matchAll(urlRegex),
].map((match) => match.slice(1));
console.log("Extracted Parts:", parts);
```

  "ftp://files.server.org/index",

]
```

    "section"      // Fragment
  ],
  [
    "ftp://",           // Protocol
    "files.server.org", // Domain
    "/index",           // Path
    "",                 // Query
    ""                  // Fragment
  ],
  [
    "custom-scheme://",   // Protocol
    "host",               // Domain
    "/param;weird^stuff", // Path
    "",                   // Query
    ""                    // Fragment
  ]
]
```

## ☑️ الخطوات التالية

اعتمادًا على حالتك، قد تحتاج إلى صقل هذا التعبير النمطي أو إضافة المزيد من خطوات التحقق والمعالجة اللاحقة.

### مشاريع مختلفة، احتياجات مختلفة

المشاريع لها متطلبات ومخاوف أمنية متفاوتة:

1. **استخلاص الويب**: التحقق من صحة عناوين URL لضمان إمكانية الوصول إليها وثقتها.
2. **معالجة البيانات**: استخراج عناوين URL من المحتوى الذي يولده المستخدم مع الحفاظ على الأمان.
3. **تحليل البيانات**: تصفية الروابط المكررة أو غير ذات الصلة لأغراض البحث أو التسويق.
4. **التطبيقات الموجهة للمستخدم**: تحويل عناوين URL تلقائيًا إلى روابط في تطبيقات الدردشة أو المنتديات.

### ما بعد المعالجة والتحقق

بعد جمع عناوين URL المحتملة، طبّق فحوصات إضافية:

- **استعلام DNS**: التأكد من أن النطاقات تُحل.
- **فحوصات الأمان**: استخدام خدمات للتحقق من المواقع الخبيثة أو الاحتيالية.
- **قواعد مخصصة**: تطبيق مرشحات خاصة بالمشروع (مثل السماح بـ TLDs معينة، الحد الأقصى لطول URL).

## 📝 ملخص

استخلاص البيانات النصية شبه المهيكلة قد يكون أكثر جوانب إتقان الـ regex إرضاءً.

إليك ملخص للنقاط الرئيسية:

- **استخدم أداة بصرية لكتابة واختبار** وفهم أنماط [Regex الخاصة بك.](https://regex101.com/r/jO8bC4/69)
- **قسّم التحدي إلى أجزاء** وحل كل جزء على حدة. بمعنى ما، توفر مجموعات الالتقاط علامات “دليلية” رمزية للـ regex.
- **استخدم تعبيرات مطابقة “فضفاضة”، وتجنب الالتزام الصارم بالمواصفات** عند استهلاك البيانات.
- **تطبيق خطوات التحقق** بعد الاستخلاص الأولي أمر أساسي—دائمًا ضع في اعتبارك أمان مشروعك واحتياجاته الخاصة.

باتباع هذه الخطوات، يمكنك استخراج أي بيانات نصية شبه مُهيكلة بفعالية، مما يضع الأساس للمعالجة اللاحقة والتحقق.
