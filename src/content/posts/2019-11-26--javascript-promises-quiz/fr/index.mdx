  <slot name="question">
  <div className="question">
    Quel sera le résultat du code suivant ?
    ```js
        var p = new Promise((resolve, reject) => {
          return Promise.reject(Error('The Fails!'))
        })
        p.catch(error => console.log(error.message))
        p.catch(error => console.log(error.message))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lors de l'utilisation du constructeur Promise, vous devez invoquer les callbacks `resolve()` ou `reject()`. Le constructeur Promise ignore la valeur de retour de l'exécuteur, donc la promesse supplémentaire créée avec `Promise.reject()` n'est pas chaînée à `p`. Les deux gestionnaires sont attachés à `p`, qui reste en attente, tandis que la promesse rejetée retournée est signalée comme non gérée par l'environnement hôte.
  </div>
  </slot>
</Challenge>

  options={[

    {text: 'UnhandledPromiseRejectionWarning'},

>
  <slot name="question">
  <div className="question">
    Quelle sera la sortie du code suivant ?
    ```js
      var p = new Promise((resolve, reject) => {
          reject(Error('The Fails!'))
        })
        .catch(error => console.log(error))
        .then(error => console.log(error))
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Lorsque vous enchaînez des `.then` et des `.catch`, il est utile de les considérer comme une série d'étapes. Chaque `.then` reçoit la valeur retournée par le `.then` précédent (comme argument). Cependant, si votre « étape » rencontre une erreur, toutes les « étapes » `.then` suivantes seront ignorées jusqu'à ce qu'un `.catch` soit rencontré. Si vous voulez outrepasser une erreur, il vous suffit de retourner une valeur non-erreur. Elle peut être accédée via n'importe quel `.then` suivant.
  </div>
  </slot>
</Challenge>
