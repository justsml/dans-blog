## Docker は何でもできる！\*

> 歴史的注記：これは2015年当時のDockerへの熱狂とローカル開発の習慣をスナップショットとして記録したもので、主にタイムカプセルとして保存されています。

コマンドとセキュリティ設定は、最新化のための出発点として扱い、本番環境の指針としては使用しないでください。

以下のプロセスを改善してください：

1.

開発ツールとサーバーのテストを、PC上の依存関係を実質的にリスクゼロで壊さずに行う
1.  ソフトウェアのテスト
1.  より冪等でモジュール化されたコードを書くようになる...

（これの具体的な実現方法は後続記事で書きます）

学ぶべき新しいことが山のようにあるように思えるかもしれませんが、**それで始めるのをためらわないでください**。

### 注記

- `docker run` コマンドに `-d` または `-it` オプションが付いている場合:
  _ `-it` または `-i -t` は、設定されたコマンドを対話的に実行します
  _ `-d` は、Docker コンテナを「デーモン」、つまりバックグラウンドサービスとして起動します

---

### 例

### nginx

```bash
# Note: using host-based, shared folders
#(shared folders are not possible with the VOLUME Dockerfile cmd)
sudo docker run --name web01 -d -p 8181:80 \
  -v $(NGINX_DIR)/etc:/etc/nginx \
  -v $(NGINX_DIR)/log:/var/log/nginx \
  -v $(NGINX_DIR)/www:/var/www/html \
  nginx:latest

## Local data, isolated within instance
sudo docker run --name web01 -d -p 8181:80 nginx:latest

# nodejs
sudo docker run --name nodejs01 -d -p 3300:3300 -p 4433:4433 nodejs:latest
```

> クレジット: [https://dockerfile.github.io/#/nginx](https://dockerfile.github.io/#/nginx)
