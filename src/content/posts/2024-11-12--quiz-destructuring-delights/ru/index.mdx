    ```plaintext
      In modern Chrome: `TypeError: Cannot read properties of undefined (reading 'place')`

      In Node, this is also a `TypeError` because JavaScript tries to destructure `place` from `undefined`.

      Exact wording varies between browsers and runtimes.
    ```

    {text: 'Ошибка: отсутствует свойство \'last\''},
    {text: 'Ошибка: отсутствуют свойства \'birth\' и \'age\'', isAnswer: true},
    {text: 'Ошибка: отсутствует свойство \'place\''},
    {text: 'Ошибка: \'string\' не имеет свойств в {...}'},
  ]}
>
  <slot name="question">
  <div className="question">
    Найдите ошибку в типах:
    ```ts
    function greet({
      name: {first = "N/A", last = "N/A"},
      birth: {place = "N/A"} = {},
      age = -1,
    }: {
      name: {first?: string, last?: string};
      birth: {place?: string};
      age: number;
    }) {
      console.log(`Hi ${first} ${last} from ${place}`);
    }
    greet({ name: {first: 'Dan'} });
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Ошибка находится в сигнатуре функции `greet`. Свойства `age` и `birth` отсутствуют в переданном объекте, поэтому они должны быть необязательными в определении типа.

    Даже если свойство `birth` деструктурируется со значением по умолчанию, определение типа требует его наличия. Чтобы пометить свойство как необязательное в TypeScript, используйте оператор `?`.

    Обратите внимание: `birth?: { place?: string }` — это не то же самое, что `birth: { place?: string } | undefined`.
  </div>
  </slot>
</Challenge>
