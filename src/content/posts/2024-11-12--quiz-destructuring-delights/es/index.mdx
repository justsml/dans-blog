¿Qué tal un poco de desestructuración anidada?
```js
    ```js
    'use strict';
    const person = {
      name: { first: 'Dan' },
      address: { city: 'Denver' },
    };
    const {
      name: { first },

```
  </div>

  <slot name='explanation'>
  <div className="explanation">
La propiedad `birth: { place }` no existe en `person`, así que saltará un error.
  Una solución es asignar valores por defecto a las propiedades anidadas.

  Ojo al acceder a propiedades anidadas: los errores se camuflan fácil. Y como cada navegador o plataforma escupe mensajes distintos, depurarlo se vuelve un poco más tedioso.

  En Chrome moderno: `TypeError: Cannot read properties of undefined (reading 'place')`

  En Node también es un `TypeError`, porque JavaScript intenta desestructurar `place` desde `undefined` antes de siquiera leer `place`.

  El texto exacto del error cambia según el navegador o el runtime.

Detecta el error de tipo:
```ts
    ```ts
    function greet({
      name: {first = "N/A", last = "N/A"},
      birth: {place = "N/A"} = {},
      age = -1,
    }: {
      name: {first?: string, last?: string};
      birth: {place?: string};
      age: number;
    }) {

    }
    greet({ name: {first: 'Dan'} });
    ```
```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
El error está en la firma de la función `greet`. Las propiedades `age` y `birth` faltan en el objeto pasado, por lo que deberían ser opcionales en la definición del tipo.

  Aunque la propiedad `birth` se desestructura con un valor predeterminado, la definición del tipo exige que esté presente. Para marcar una propiedad como opcional en TypeScript, debes usar el operador `?`.
  
  Ten en cuenta que `birth?: { place?: string }` no es lo mismo que `birth: { place?: string } | undefined`.
  </div>
