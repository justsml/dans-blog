<blockquote class="inset">
**अपडेट:** इस लेख ने [`llm://` URI स्कीम के लिए एक इंटरनेट-ड्राफ्ट](https://datatracker.ietf.org/doc/draft-levy-llm-uri-scheme/) को जन्म दिया।
</blockquote>


याद कीजिए उन पुराने बुरे दिनों को जब डेटाबेस से कनेक्ट करने का मतलब था पर्यावरण चरों (environment variables) का एक अस्त-व्यस्त ढेर संभालना?

यह एक नाजुक कॉन्फिग का टावर था। `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`... या रुकिए, क्या यह `DB_USERNAME` था? क्या यह `DB_PASS` है या `DB_PWD`? क्या इस बार मुझे `PG_*` उपसर्गों की ज़रूरत है? और टाइमआउट सेटिंग कहाँ जाती है?

यह पत्तों का एक नाजुक घर था, तैयार था गिरने के लिए क्योंकि आपने `HOST` को कैपिटलाइज़ करना भूल गए।

फिर, किसी ने एक शानदार विचार किया कि बस एक URL¹ का उपयोग किया जाए:

```bash
postgres://user:pass@host:5432/dbname
```

एक स्ट्रिंग। आपको जो कुछ भी चाहिए। सार्वभौमिक रूप से पार्स करने योग्य। पोर्टेबल। क्या मैं कहूं... खूबसूरत?

तो फिर हम LLMs के साथ 1999 जैसा व्यवहार क्यों कर रहे हैं?

## Env Var का विस्फोट

अभी, मेरी `.env` फाइल एक परित्यक्त API कुंजियों का कब्रिस्तान लगती है। `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `MISTRAL_API_KEY`, `GROQ_API_KEY`। और Azure के बारे में तो मुझे शुरू ही मत कराइए—"hello" कहने के लिए भी आपको एक एंडपॉइंट, एक डिप्लॉयमेंट नाम, एक API संस्करण और एक कुंजी की ज़रूरत होती है।

यह सिर्फ बदसूरत नहीं है; यह घर्षण है। हर बार जब मैं एक मॉडल बदलना या एक नए प्रोवाइडर का परीक्षण करना चाहता हूँ, तो मैं इनिशियलाइज़ेशन कोड फिर से लिख रहा हूँ, विशिष्ट पैरामीटर नामों के लिए दस्तावेज़ीकरण खोज रहा हूँ, और अपने पर्यावरण कॉन्फिग में तीन और पंक्तियाँ जोड़ रहा हूँ।

क्या हो अगर हम बस... ~~चुरा~~ उधार लें DB URL का विचार?

## LLM कनेक्शन स्ट्रिंग्स का परिचय

कल्पना कीजिए कि अपने पूरे मॉडल इंटरफ़ेस को एक ही पंक्ति से कॉन्फिगर करना:

```bash
llm://api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7&max_tokens=1500
llm://api.z.ai/glm-4.7?top_p=0.9&cache=true
```

### LLM कनेक्शन स्ट्रिंग का शरीर-रचना (Anatomy)

![LLM कनेक्शन स्ट्रिंग के भाग](./inline-url-diagram-dark.svg)

स्कीम `llm://` है। होस्ट प्रोवाइडर का API बेस URL है। पथ मॉडल का नाम है। और क्वेरी पैरामीटर उन सभी रनटाइम विकल्पों को संभालते हैं जो आमतौर पर आपके कोड को अव्यवस्थित करते हैं।

## Auth चाहिए? बढ़िया, इसे जोड़ दें।

बिल्कुल `postgres://` की तरह, हम प्रमाणीकरण को सीधे इसमें बेक कर सकते हैं:

```bash
llm://app-name:sk-proj-123456@api.openai.com/gpt-5.2?reasoning_effort=none&temp=0.7
```

*नोट: हाँ, URL में क्रेडेंशियल्स डालना एक सुरक्षा जोखिम हो सकता है अगर आप उन्हें सार्वजनिक लॉग्स में पेस्ट कर रहे हैं। लेकिन आधुनिक लॉगिंग सेवाएँ इन पैटर्न को स्क्रब करने में काफी अच्छी हैं, और ईमानदारी से, क्या आप अपनी `.env` फाइल को बेहतर तरीके से संभाल रहे हैं? सत्यापित करें, सैनिटाइज़ करें, और सावधानी से उपयोग करें।*

## लचीलापन (Resiliency)? क्यों नहीं, भाई।

कई डेटाबेस लाइब्रेरी एकाधिक होस्ट निर्दिष्ट करके राउंड-रोबिन फेलओवर का समर्थन करती हैं। क्यों नहीं हमारे AI एजेंट्स को वही विश्वसनीयता मिले?

```bash
llms://primary.gpt,backup.gpt/gpt-6?temp=0.9
```

`llms://` में वह `s` टाइपो नहीं है। यह बहुवचन है। अगर `primary.gpt` लटक जाता है, तो क्लाइंट स्वचालित रूप से `backup.gpt` पर पुनः प्रयास करता है। कोई जटिल राउटर लॉजिक आवश्यक नहीं।

<blockquote class="inset">एक स्ट्रिंग में सब कुछ, आपके **auth** से लेकर **endpoint** तक, और **hyperparameters** तक।</blockquote>

## वैकल्पिक प्रारूप

मैं `llm://` से बंधा नहीं हूँ। विशिष्ट स्कीम कम महत्वपूर्ण है मानक से।

मैं एक ऐसी दुनिया की कल्पना कर सकता हूँ जहाँ हम संक्षिप्तता के लिए प्रोवाइडर-विशिष्ट स्कीम का उपयोग करें, जबकि मानक संरचना बनाए रखें:

```bash
ollama://localhost:11434/llama3
vercel://anthropic/sonnet-4.5?temp=0.8&web_search={"maxUses":3}
bedrock://us-west-2.aws/anthropic/sonnet-4.5?temp=0.8&cacheControl=ephemeral
```

सटीक सिंटैक्स के बावजूद, मुख्य लाभ निर्विवाद हैं:

1.  **पोर्टेबिलिटी:** अपनी पूरी कॉन्फिग को एक लोकल स्क्रिप्ट से क्लाउड वर्कर तक कॉपी और पेस्ट करें।
2.  **CLI अनुकूल:** अपनी स्क्रिप्ट्स को एक ही तर्क पास करें। `my-agent --model "llm://..."` इससे बेहतर है `my-agent --model gpt-4 --temp 0.7 --key $KEY --host ...`।
3.  **भाषा-agnostic:** हर प्रोग्रामिंग भाषा में एक मजबूत URL पार्सर होता है। हमें मुफ्त में सत्यापन, पार्सिंग और सैनिटाइज़ेशन मिलता है।

<blockquote class="ai-response inset">डेटाबेस की दुनिया को इसे समझने में दशकों लग गए।<br /><b>अच्छी खबर, AI टाइमलाइन में, यह लगभग आधा vibe-year पहले की बात है।</b></blockquote>

## निर्णय

हमें एक और जटिल कॉन्फिगरेशन मानक या YAML-आधारित मैनिफेस्ट फाइल की आवश्यकता नहीं है। हमें बस उस एक टूल का उपयोग करने की आवश्यकता है जो बाकी इंटरनेट के लिए पिछले 30 वर्षों से काम कर रहा है।

आइए पहिए का पुनः आविष्कार करना बंद करें और अपने LLM कनेक्शन को उसी सम्मान के साथ व्यवहार करना शुरू करें जो हम अपने डेटाबेस को देते हैं। आपकी `.env` फाइल (और आपकी सांस) आपका धन्यवाद करेगी।
