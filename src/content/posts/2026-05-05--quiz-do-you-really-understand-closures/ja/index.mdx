<p class="inset">クロージャは、すべてのJavaScript開発者が*理解了していると信じている*概念です — 理解できなくなる瞬間まで。 このクイズは、あなたが実際に知っていることの限界を見つけます。</p>

クロージャは、イベントハンドラ、ファクトリ関数、Reactフック、モジュールパターン、そしてほとんど至る所に現れます。それらはエキゾチックではありません。それらはJavaScriptが織り成されている構造そのものなのです。

質問は穏やかに始まり、随后、床板を取り外し始めます。

    `var`は関数スコープです。**1つの**`i`だけが存在します — 5つのコールバックはすべて同じバインディングを閉じ込めます。タイムアウトが発生的时候我頃には、ループは終了しており、`i`は`5`です。

    5つのコールバック。1つの`i`。値：`5`。5回ログ出力されます。

    修復方法は`let`（ブロックスコープ、反復ごとに新しいバインディング）またはIIFEでラップして現在の値をキャプチャすることです：
    ```javascript
    for (let i = 0; i < 5; i++) {
      setTimeout(() => console.log(i), 100); // 0, 1, 2, 3, 4 ✓
    }
    ```

    ```javascript
    { let i = 1; setTimeout(() => console.log(i), 100) }

    ```

    したがって、5つのコールバックは5つのDistinctな変数を閉じ込めます。結果：`0, 1, 2, 3, 4`。

    これは、クロージャが値を「冻结」すると想定する사람々に刺さります。それはしません。**参照**をキャプチャします。

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={5}
  group="アドバンスト"
  title="Reactのステールクロージャ"
  options={[
    {text: 'クリックごとに現在のcountをログ出力する'},
    {text: '常に0をログ出力する', isAnswer: true},
    {text: 'NaNをログ出力する'},
    {text: "これはコンパイルしない"},
  ]}
>
  <slot name="question">
  <div className="question">
    このReactコンポーネントは、ボタンが複数回クリックされたときどのような動作をしますか？
    ```jsx
    function Counter() {
      const [count, setCount] = useState(0);

      useEffect(() => {
        const id = setInterval(() => {
          console.log('count is', count);
        }, 1000);
        return () => clearInterval(id);
      }, []); // 空のdeps

      return <button onClick={() => setCount(c => c + 1)}>
        Click ({count})
      </button>;
    }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    古典的なステールクロージャ。`useEffect`は一度だけ実行され（空のdep配列）、マウント時の`count`を閉じ込める интервалを作成します： `0`。何回クリックしても интервал のクロージャは更新された値を永远に見ません。

    ボタンラベルは正しく更新されます — これは新鮮なレンダリングです。しかし`console.log`は永遠に`0`に閉じ込められています。

    修復方法：dep配列に`count`を追加する（интервалを再作成）またはrefを使用します：
    ```jsx
    const countRef = useRef(count);
    useEffect(() => { countRef.current = count; }, [count]);
    // then use countRef.current inside the interval
    ```
  </div>
  </slot>
</Challenge>

    待ってください — 注意深く読んでください。実際の結果は何ですか？
    ```javascript
    const bank = (() => {
      let balance = 0;
      return {
        deposit: (n) => (balance += n),
        withdraw: (n) => (balance -= n),
        getBalance: () => balance,
      };
    })();

    ```
  </div>

  <slot name='explanation'>
  <div className="explanation">
    答えは`60`です。

    これは**モジュールパターン**です — クロージャのオブジェクトを返すIIFE。3つのメソッドすべてが同じ`balance`バインディングを閉じ込めます。これは本物のプライベートです：`balance`を外部からアクセスすることはできません。

    ```javascript
    console.log(bank.balance); // undefined
    ```

    クラス以前は、これはJavaScriptでステートをカプセル化する方法*そのもの*でした。レガシーコードベースやユーティリティライブラリでまだ見かけるでしょう。

  ]}
>
  <slot name="question">
  <div className="question">
    このコードの問題は何ですか？
    ```javascript
    function attachHandler(element) {
      const data = new Array(100_000).fill('🔥');
      element.addEventListener('click', () => {
        console.log('clicked!', data.length);
      });
    }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    イベントリスナーは`data`を閉じ込めるので、リスナーが到達可能な間、配列も到達可能のままになります。`element`が長寿命でリスナーが決して削除されない場合、その大きな配列も保持され続けます。

    `element`はDOMにあります。リスナーが`data`を保持しています。GC 그것을 수집할 수 없습니다.

    修復方法：本当に必要な場合を除いて、長寿命のリスナーで大きなデータをキャプチャしないでください。また所有するUIが破棄されるときにリスナーを削除してください：
    ```javascript
    function attachHandler(element) {
      const data = new Array(100_000).fill('🔥');
      // do something with data...

      const onClick = () => {
        console.log('clicked!');
      };

      element.addEventListener('click', onClick);
      return () => element.removeEventListener('click', onClick);
    }
    ```

    最新のV8は以前よりもこれが上手になっています。しかし、多くのイベントリスナーを持つ長寿命のSPAでは、このパターンはまだ 人々を苦しめます。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={8}
  group="エキスパート"
  title="thisトラップ"
  options={[
    {text: 'Dan, Dan'},
    {text: 'Dan, undefined'},
    {text: 'undefined, Dan'},
    {text: 'undefined, undefined', isAnswer: true},
  ]}
>
  <slot name="question">
  <div className="question">
    何をログ出力しますか？
    ```javascript
    const user = {
      name: 'Dan',
      greetArrow: () => console.log(this?.name),
      greetRegular() {
        const inner = () => console.log(this?.name);
        inner();
      },
    };

    user.greetArrow();
    const fn = user.greetRegular;
    fn();
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    2つの微妙なトラップ：

    **`greetArrow`**：アロー関数には独自の`this`がありません。それらは周囲のレキシカルスコープから`this`をキャプチャします — 这里是モジュール/グローバルスコープであり、`user`ではありません。だから`this?.name`は`undefined`です。

    **`fn()`**：`greetRegular`はレシーバーなし呼ばれます（`fn()`而不是`user.greetRegular()`）。(strictモードでは、`this`は`undefined`です。非strictモードでは、グローバルオブジェクトです。どちらにせよ、`this?.name`は`undefined`です。

    もし`user.greetRegular()`を直接呼んでいたなら、`'Dan'`が得られます — その場合、`this`は`user`にバインドされるからです。

    結果：`undefined, undefined`。
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={9}
  group="ボーナス"
  title="クロージャによるメモ化"
  options={[
    {text: 'expensive()を3回呼び出す'},
    {text: 'expensive()を1回呼び出す', isAnswer: true},
    {text: 'expensive()を2回呼び出す'},
    {text: 'スロー：Maximum call stack exceeded'},
  ]}
>
  <slot name="question">
  <div className="question">
    `expensive()`実際に呼ばれる回数は？
    ```javascript
    function memoize(fn) {
      const cache = new Map();
      return (arg) => {
        if (cache.has(arg)) return cache.get(arg);
        const result = fn(arg);
        cache.set(arg, result);
        return result;
      };
    }

    let calls = 0;
    const expensive = (n) => { calls++; return n * 2; };
    const fast = memoize(expensive);

    fast(5);
    fast(5);
    fast(5);
    console.log(calls);
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    メモ化された関数は`cache`を閉じ込めます。最初の呼び出し（`fast(5)`）はキャッシュをミスし、`expensive`を呼び出して結果を保存します。2番目と3番目の呼び出しはキャッシュにヒットして即座に返ります。

    `calls`は`1`です。

    これはクロージャが実際の作業をしている 것입니다。`cache`はプライベート（`memoize`の外部からアクセスできない）、永続的（`fast`が存在する限り存続）、そして`fast`へのすべての呼び出しで共有されています。
  </div>
  </slot>

どうでしたか？

- **9-10点**：あなたは実際にクロージャを理解しています。ペアプログラミングするのは怖いでしょう。
- **6-8点**：堅実です。ステールクロージャ/`this`の質問で躓いたのは、正直なところです。
- **3-5点**：コンセプトは理解していますが、边缘が曖昧です。MDNのスコープチェーンのドキュメントを読みましょう。
- **0-2点**：クロージャを理解していると思っていました。今、理解しています。それが要点です。

Reactのステールクロージャ(#6)と`this`トラップ(#8)は、実際の本番バグを引き起こすものです。それらが親しくなかったら、復習する価値があります — それらは他のクロージャ関連の問題よりもコードレビューでより多く現れます。
