    फिक्स है `let` (block-scoped, हर इटरेशन में नया बाइंडिंग) या IIFE में बंद करना वर्तमान मान पकड़ने के लिए:
    ```javascript
    for (let i = 0; i < 5; i++) {
      setTimeout(() => console.log(i), 100); // 0, 1, 2, 3, 4 ✓
    }
    ```

  client:visible={{rootMargin: "150px"}}
  index={5}
  group="Advanced"
  title="Stale Closure in React"
  options={[
    {text: 'Logs the current count on every click'},
    {text: 'Always logs 0', isAnswer: true},
    {text: 'Logs NaN'},
    {text: "This won't compile"},
  ]}
>
  <slot name="question">
  <div className="question">
    बटन को कई बार क्लिक करने पर इस React कॉम्पोनेंट में क्या व्यवहार होता है?
    ```jsx
    function Counter() {
      const [count, setCount] = useState(0);

      useEffect(() => {
        const id = setInterval(() => {
          console.log('count is', count);
        }, 1000);
        return () => clearInterval(id);
      }, []); // empty deps

      return <button onClick={() => setCount(c => c + 1)}>
        Click ({count})
      </button>;
    }
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    क्लासिक stale closure। `useEffect` एक बार चलता है (empty dep array) और एक interval बनाता है जो वह `count` को बंद कर लेता है जैसा वह माउंट होने पर था: `0`। आप चाहे कितनी बार भी क्लिक करें, interval का क्लोज़र कभी अपडेटेड मान नहीं देखता।

    बटन लेबल सही अपडेट होता है — वह एक फ्रेश रेंडर है। लेकिन `console.log` हमेशा `0` पर फंसा है।

    फिक्स करें डिप्स अरे में `count` जोड़कर (interval को रीक्रिएट करने के लिए) या ref का उपयोग करके:
    ```jsx
    const countRef = useRef(count);
    useEffect(() => { countRef.current = count; }, [count]);
    // then use countRef.current inside the interval
    ```
  </div>
  </slot>
</Challenge>

    `element` DOM में है। लिसनर `data` को हैंडल करता है। GC इसे कलेक्ट नहीं कर सकता।

    फिक्स: लंबे-लाइफ़साइकल लिसनर्स में बड़ा डेटा को कैप्चर न करें जब तक आप वास्तव में नहीं चाहते, और जब UI गिरता है तो लिसनर्स हटाएं:
    ```javascript
    function attachHandler(element) {
      const data = new Array(100_000).fill('🔥');
      // do something with data...

      const onClick = () => {
        console.log('clicked!');
      };

      element.addEventListener('click', onClick);
      return () => element.removeEventListener('click', onClick);
    }
    ```

    आधुनिक V8 इसे पहले की तुलना में बेहतर रखता है। लेकिन लंबे-लाइफ़साइकल SPAs में जहां कई इवेंट लिसनर्स होते हैं, यह पैटर्न अभी भी लोगों को बुरा महसूस करवाता है।

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={8}
  group="Expert"
  title="The this Trap"
  options={[
    {text: 'Dan, Dan'},
    {text: 'Dan, undefined'},
    {text: 'undefined, Dan'},
    {text: 'undefined, undefined', isAnswer: true},
  ]}
>
  <slot name="question">
  <div className="question">
    यह क्या लॉग करेगा?
    ```javascript
    const user = {
      name: 'Dan',
      greetArrow: () => console.log(this?.name),
      greetRegular() {
        const inner = () => console.log(this?.name);
        inner();
      },
    };

    user.greetArrow();
    const fn = user.greetRegular;
    fn();
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    दो सूक्ष्म ट्रैप्स:

    **`greetArrow`**: Arrow functions का अपना `this` नहीं है। वे `this` को बंद कर लेते हैं जो उनके लिक्सिकल स्कोप में है — यहां यह मॉड्यूल/ग्लोबल स्कोप है, न कि `user`। तो `this?.name` `undefined` है।

    **`fn()`**: `greetRegular` को रिसीवर के बिना कॉल किया जाता है (`fn()` not `user.greetRegular()`)। Strict मोड में, `this` `undefined` है। Sloppy मोड में, यह ग्लोबल ऑब्जेक्ट है। दोनों ही मामले में `this?.name` `undefined` है।

    अगर आप `user.greetRegular()` directly कॉल करते, तो आप `'Dan'` पाते — तो `this` `user` को बाइंड होता है।

    रिजल्ट: `undefined, undefined`।
  </div>
  </slot>
</Challenge>

    `expensive()` वास्तव में कितनी बार कॉल हुआ?
    ```javascript
    function memoize(fn) {
      const cache = new Map();
      return (arg) => {
        if (cache.has(arg)) return cache.get(arg);
        const result = fn(arg);
        cache.set(arg, result);
        return result;
      };
    }

    let calls = 0;
    const expensive = (n) => { calls++; return n * 2; };
    const fast = memoize(expensive);

    fast(5);
    fast(5);
    fast(5);
    console.log(calls);
    ```
  </div>

  <slot name='explanation'>
  <div className="explanation">
    मेमोाइज्ड फंक्शन `cache` को बंद कर लेता है। पहला कॉल (`fast(5)`) कैश में नहीं है, `expensive` को कॉल करता है और रिजल्ट स्टोर करता है। दूसरा और तीसरा कॉल कैश में मिलते हैं और तुरंत लौटते हैं।

    `calls` `1` है।

    यह क्लोज़र्स द्वारा किया जाने वाला वास्तविक काम है। `cache` प्राइवेट है (बाहर से `memoize` तक नहीं पहुंच सकता), पर्सिस्टेंट (समय के साथ `fast` मौजूद रहता है), और सभी `fast` कॉल्स शेयर किया गया।
