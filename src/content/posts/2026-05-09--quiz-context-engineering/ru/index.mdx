    LLM не имеют постоянной памяти. Каждый вызов API без состояния — вы отправляете полный контекст, получаете ответ. «Память» разговора полностью зависит от истории сообщений, которые вы включаете в каждый запрос.

    Извлечение истории разговора из векторной БД теоретически привлекательно, но обычно избыточно для большинства чат-приложений. Сжатие — практичный стандарт.
