  options={[
    {text: 'width: 50%;'},

    {text: 'width: 250px;'},
    {text: 'width: 500px;'},
    {text: 'width: max(50%, 200px);'},
    {text: 'Sintaxis no válida'},
  ]}
>
  <slot name="question">
  <div className="question">
Si el contenedor padre mide 400px de ancho, ¿cuál será el ancho calculado real de este elemento?
```css
    ```css
    div {
      width: min(250px, 50%);
    }
    ```
```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
La función `min()` siempre se queda con el valor más pequeño entre `250px` y el `50%` del ancho del padre.

Para entender el valor calculado, hay que convertir las unidades relativas a píxeles:

- El `50%` de `400px` equivale a `200px`
- `250px` ya está en píxeles
```css
```css
/* This gets computed to */
width: min(250px, 200px);
/* -> 200px wins */
```
```
La función `min()` es especialmente útil para el diseño responsivo, ya que te permite garantizar que un componente (o el tamaño de la fuente) nunca supere un límite establecido.
  </div>
  </slot>
</Challenge>

```css

    }
    ```
```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
La función `max()` acepta dos o más valores y selecciona automáticamente el mayor. Si asumimos que el tamaño de fuente raíz es el predeterminado del navegador (`16px`), el ancho resultante es `96px`.

Para entender el valor calculado, debemos convertir las unidades relativas a píxeles:

- `50px` ya está en píxeles
- `10%` de `200px` equivale a `20px`
- `6rem` es `6 * 16px` (el tamaño de fuente predeterminado), lo que da `96px`
```css
```css
/* This gets computed to */
width: max(50px, 20px, 96px);
/* -> 96px wins */
```
```

  </slot>
