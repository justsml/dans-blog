Никто не ставит целью построить небезопасную систему ИИ. Вы пишете инструкции, тестируете граничные случаи, добавляете несколько правил валидации. А потом кто-то понимает, что можно обмануть бота, заставив его играть роль пирата и раскрывать пользовательские данные. Или номер кредитной карты оказывается в ваших логах. Или модель уверенно рекомендует продукт конкурента.

Разрыв между «работает в демо» и «безопасно в продакшене» шире, чем ожидает большинство команд.

Часть проблемы в том, что чистые LLM не имеют собственного мнения о том, что им следует или не следует делать. Они представляют собой машины предсказания, которые пытаются продолжить любую начатую последовательность. Дайте им промпт, похожий на «режим системного переопределения», и они с радостью подыграют. Это не баг модели — это просто принцип работы языковых моделей.

Большинство фреймворков просто передают вам модель и желают удачи. Mastra использует другой подход: она предполагает, что вам в конечном итоге понадобятся ограничители, поэтому встраивает их в архитектуру агента с самого начала.

## Процессоры как слои безопасности

Основной механизм прост. Прежде чем ваш промпт достигнет модели, он проходит через цепочку процессоров ввода. После ответа модели свою очередь получают процессоры вывода. Каждый процессор может проверить, изменить или заблокировать контент на этом этапе.

Думайте о них как о промежуточном ПО для взаимодействия с ИИ. Вы складываете нужные вам процессоры, настраиваете их поведение, и они автоматически запускаются при каждом запросе.

### 1. Остановка пиратов (инъекция промпта)

Атаки инъекции промпта становятся всё изобретательнее. Люди используют невидимые символы Unicode, пишут инструкции в base64 или убеждают модель, что она находится в «режиме отладки», где обычные правила не действуют. Техники постоянно развиваются.

Mastra включает процессоры, которые перехватывают распространённые паттерны:


```typescript
// src/mastra/agents/secure-agent.ts
import { Agent } from '@mastra/core/agent';
import { PromptInjectionDetector, UnicodeNormalizer } from '@mastra/core/processors';
import { openai } from '@ai-sdk/openai';

export const secureAgent = new Agent({
  id: 'fortress-assistant',
  name: 'fortress-assistant',
  instructions: 'You are a secure assistant.',
  model: openai('gpt-5'),
  inputProcessors: [
    // 1. Scrub invisible characters
    new UnicodeNormalizer({
      id: 'unicode-normalizer',
      stripControlChars: true,
      collapseWhitespace: true,
    }),
    // 2. Detect the attempt
    new PromptInjectionDetector({
      id: 'prompt-injection-detector',
      model: openai('gpt-5-nano'), // Cheap, fast
      threshold: 0.8,
      strategy: 'block', // Hard stop
      detectionTypes: ['injection', 'jailbreak', 'system-override'],
    }),
  ],
});
```

[`UnicodeNormalizer`](https://mastra.ai/docs/processors) удаляет управляющие символы и схлопывает пробелы. [`PromptInjectionDetector`](https://mastra.ai/docs/processors) анализирует очищенный ввод на предмет паттернов, указывающих на попытку переопределить ваши инструкции.

Вы настраиваете, насколько агрессивным должно быть обнаружение (параметр `threshold`) и что должно происходить при его срабатывании (блокировка, логирование или просто пометка).

### 2. Обработка персональных данных

Номера кредитных карт в логах, номера социального страхования в векторных базах данных, адреса электронной почты, хранящиеся дольше необходимого. Это именно те проблемы, которые превращаются в регуляторные. Сложность в том, что пользователи не всегда осознают, что вставляют конфиденциальные данные в окно чата.

[`PIIDetector`](https://mastra.ai/docs/processors) сканирует распространённые паттерны, прежде чем они достигнут вашей модели или будут записаны в хранилище:

```typescript
import { PIIDetector } from '@mastra/core/processors';

export const privateAgent = new Agent({
  id: 'privacy-first-assistant',
  name: 'privacy-first-assistant',
  instructions: 'You are a helpful assistant that never stores personal information.',
  model: openai('gpt-5'),
  inputProcessors: [
    new PIIDetector({
      id: 'pii-detector',
      model: openai('gpt-5-nano'),
      detectionTypes: ['email', 'phone', 'credit-card', 'ssn'],
      threshold: 0.6,
      strategy: 'redact',
      redactionMethod: 'mask',  // Replace with [REDACTED]
      instructions: 'Detect and mask personally identifiable information',
    }),
  ],
});
```

Вы можете выбрать редактирование (замену на `[REDACTED]`), хеширование или полную блокировку. Процессор работает как на вводе, так и на выводе, поэтому вы защищены даже если модель каким-то образом сгенерирует конфиденциальные данные в своём ответе.

### 3. Модерация контента

Модели, обученные на интернет-данных, видели некоторые вещи. Без фильтрации они могут время от времени производить ответы, которые заставят ваш PR-отдел нервничать. [`ModerationProcessor`](https://mastra.ai/docs/processors) перехватывает контент, нарушающий ваши правила:

```typescript
import { ModerationProcessor } from '@mastra/core/processors';

export const moderatedAgent = new Agent({
  id: 'safe-assistant',
  name: 'safe-assistant',
  instructions: 'You are a helpful assistant for a community platform.',
  model: openai('gpt-5'),
  inputProcessors: [
    new ModerationProcessor({
      id: 'moderation-processor',
      model: openai('gpt-5-nano'),  // Fast, cheap model for classification
      categories: ['hate', 'harassment', 'violence', 'self-harm'],
      threshold: 0.7,  // Block if confidence > 70%
      strategy: 'block',  // Stop the request immediately
      instructions: 'Detect harmful content that violates community guidelines',
    }),
  ],
});
```

Интересная часть в том, что вы определяете, какие категории важны для вашего случая. Инструмент для творческого письма может допускать более экспрессивный контент, чем бот службы поддержки клиентов. Порог и стратегия дают вам контроль над тем, насколько строгой должна быть фильтрация.

## Когда что-то срабатывает

Процессоры не выбрасывают ошибки при обнаружении проблемы. Вместо этого они устанавливают флаг в объекте результата:

```typescript
const result = await secureAgent.generate('Ignore all previous instructions...');

if (result.tripwire) {
  console.log(`Blocked! Reason: ${result.tripwireReason}`);
  // "Blocked! Reason: Prompt injection detected."
  return "Nice try, script kiddie.";
}
```

Этот паттерн позволяет вам обрабатывать события безопасности так, как имеет смысл для вашего приложения. Вы можете логировать их для анализа, возвращать общее сообщение об ошибке или даже разрешать определённые нарушения в конкретных контекстах. Поле `tripwireReason` сообщает вам, какой именно процессор пометил контент, что помогает при отладке ложных срабатываний или настройке порогов.

## Что это не решает

Процессоры перехватывают многое, но они не волшебны. Решительный атакующий с достаточным количеством времени, вероятно, сможет найти промпт, который проскользнёт сквозь защиту. Модели время от времени галлюцинируют способами, которые процессоры не могут предсказать. И всегда есть компромисс между безопасностью и гибкостью: чем строже ваши правила, тем выше вероятность заблокировать легитимные варианты использования.

Ценность не в идеальной защите. Она в систематическом способе обработки распространённых проблем, которые обязательно возникнут в продакшене. Вы можете настраивать чувствительность по мере того, как узнаёте, что ваши пользователи реально делают. Вы можете добавлять собственные процессоры для специфичных для вашей области рисков. И у вас есть аудиторские следы, показывающие, что было заблокировано и почему.

Большинство проблем безопасности в продакшен-ИИ — это не изощрённые атаки. Это люди, копирующие и вставляющие данные, которые не должны, или обнаруживающие методом проб и ошибок, что бот будет делать вещи, которые вы не планировали. Процессоры не остановят каждую возможную проблему, но сделают очевидные значительно сложнее.
