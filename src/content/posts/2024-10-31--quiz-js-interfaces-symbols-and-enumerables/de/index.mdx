    ```js
    const person = {};
    Object.defineProperty(person, 'age', {
      value: 25,
      enumerable: true
    });

    }
    ```
  </div>
  </slot>

  <div className="explanation">
    Die `enumerable`-Eigenschaft in `Object.defineProperty()` steuert, ob eine Eigenschaft von Aufzählungsmethoden wie `for...in` berücksichtigt wird. Da hier `enumerable: true` gesetzt ist, wird die `age`-Eigenschaft bei der Iteration tatsächlich aufgelistet.
  </div>
  </slot>

  <div className="question">
    Was ist der Standardwert für die Enumerierbarkeit einer Eigenschaft, wenn du `Object.defineProperty()` nutzt, ohne `enumerable` explizit anzugeben?
    ```js
    const car = {};
    Object.defineProperty(car, 'make', {
      value: 'Toyota'
    });
    console.log(Object.keys(car));
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Wenn du `Object.defineProperty()` aufrufst, ohne `enumerable` zu setzen, ist der Standardwert `false`. Deshalb wird die `make`-Eigenschaft von `Object.keys()` und ähnlichen Aufzählungsmethoden schlicht übergangen.
  </div>
  </slot>
</Challenge>


    Jeder Aufruf von `Symbol()` erzeugt einen einzigartigen, unveränderlichen Wert – selbst wenn die Beschreibung identisch ist. `sym1` und `sym2` sind also zwei völlig verschiedene Symbole, weshalb der Vergleich `false` zurückgibt.
  </div>
  </slot>

    Welche Methode extrahiert alle Symbol-Keys eines Objekts?

  <slot name='explanation'>
  <div className="explanation">
    Die Methode `Object.getOwnPropertySymbols()` ist dein Werkzeug, um gezielt die eigenen Symbol-Keys eines Objekts abzurufen. Sie filtert explizit nach Symbolen und ignoriert String-Keys sowie vererbte Eigenschaften – so isolierst du zuverlässig die direkt am Objekt definierten Symbole.
    ```js
    const sym1 = Symbol('id');
    const sym2 = Symbol('name');
    const obj = {
      [sym1]: 'symbol value',
      [sym2]: 'another symbol value'
    };
    console.log(Object.getOwnPropertySymbols(obj));
    // [Symbol(id), Symbol(name)]
    ```
  </div>
