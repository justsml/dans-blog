    ```js
      }
    });
    ```

    1. Se convierte en una cadena
    2. Se transforma a mayúsculas
    3. Se pasa a stdout

    Esto crea un pipeline que convierte toda la entrada a mayúsculas.

    ```js
      console.log(`${filename} was changed`);
    });
    // Then modify test.txt once
    ```
  </div>

  <slot name='explanation'>

    `fs.watch()` no garantiza que se dispare exactamente una vez por cada cambio lógico en el archivo. Es muy común que se dispare múltiples veces porque muchos editores de texto:
    1. Guardan en un archivo temporal
    2. Lo renombran al archivo final

    Para un monitoreo más robusto, te recomiendo:
    - Usar el paquete `chokidar`
    - Aplicar *debounce* a la callback
    - Optar por `fs.watchFile()` (aunque es menos eficiente)

  <div className="question">
    ¿Cuál es el propósito principal del backpressure en streams?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    El backpressure es un mecanismo que evita el desbordamiento de memoria al pausar la lectura cuando el extremo de escritura no puede seguir el ritmo.

    Ejemplo de backpressure manual:
    ```js
    readable.on('data', (chunk) => {
      const canContinue = writable.write(chunk);
      if (!canContinue) {
        readable.pause();
        writable.once('drain', () => readable.resume());
      }
    });
    ```
    `pipe()` se encarga de esto automáticamente.

    [Más información sobre backpressure](https://nodejs.org/en/learn/modules/backpressuring-in-streams#backpressuring-in-streams)
  </div>
  </slot>

    1. Modo binario (predeterminado): optimizado para manejar buffers y strings de forma eficiente

    ```js

      objectMode: true,
      transform(chunk, encoding, callback) {

      }
    });
    ```
