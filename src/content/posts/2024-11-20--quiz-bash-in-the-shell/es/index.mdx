    En Bash, las variables se declaran sin espacios alrededor del signo `=`. Por ejemplo:

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={4}
  group="Reemplazo de subcadenas"
  title="Reemplazar subcadena"
  options={[
    {text: 'meow meow'},
    {text: 'Meow meow'},
    {text: 'Bark meow', isAnswer: true},
    {text: 'Bark bark'},
    {text: 'Error'},
  ]}
>
  <slot name="question">
  <div className="question">
    ¿Qué está pasando aquí?
    ```bash
    str="Bark bark"
    echo ${str/bark/meow}
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La sintaxis `${var/pattern/replacement}` sustituye la primera coincidencia de `pattern` por `replacement`. En este caso, la salida es:
    ```plaintext
    Bark meow
    ```
    La búsqueda distingue entre mayúsculas y minúsculas. Para capturar tanto `bark` como `Bark`, utiliza un patrón como `${var/[Bb]ark/Bark}` o normaliza la cadena antes de reemplazar.

    Para sustituir todas las coincidencias, utiliza `${var//pattern/replacement}`.

    Para reemplazar solo si coincide al inicio de la cadena, usa `${var/#pattern/replacement}`.

    Para reemplazar solo si coincide al final de la cadena, usa `${var/%pattern/replacement}`.
  </div>
  </slot>
</Challenge>

<Challenge

>
  <slot name="question">
  <div className="question">
    ¿Qué imprime este script si el archivo `cats.txt` EXISTE?
    ```bash
    if [ -e cats.txt]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    ¿Notaste el espacio que falta antes del corchete de cierre?
    
    Bash es bastante estricto con esto: se requieren espacios dentro de las expresiones entre corchetes.

    Debido a que el espacio faltante hace que el comando `[` no vea un `]` de cierre, Bash imprime un mensaje de diagnóstico, considera que la prueba falló y continúa con la rama `else`.

    La sintaxis correcta es:
    ```bash
    if [ -e example.txt ]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
    Nota: Se **recomienda** usar dobles corchetes `[[ ]]` para expresiones condicionales. [Consulta el BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)
  </div>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="Trampas comunes"
  title="Sustitución de comandos"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    ¿Cuál de estas opciones ejecutará el comando `ls -l` y devolverá su salida?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La sintaxis `$(ls -l)` ejecuta el comando dentro de los **paréntesis** y reemplaza la llamada con su salida. Por ejemplo:
    ```bash
    echo "Today is $(date +%F)"
    # => Today is 2029-12-31
    ```
    La primera opción usa comillas simples `'`, **no backticks.** Esto bloquea la expansión, así que `'$(date +%F)'` solo imprimirá la cadena literal `$(date +%F)`.

    Aunque los backticks (`` `ls -l` ``) siguen siendo compatibles, se consideran un anti-patrón hoy en día. Usa siempre `$(comando)` para mayor legibilidad y compatibilidad entre distintas shells y versiones.

    Las llaves `${}` sirven para expandir variables, no para ejecutar comandos.

    El símbolo `%` no se usa para la sustitución de comandos.
  </div>
  </slot>
</Challenge>
