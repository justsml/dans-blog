  <div className="explanation">
    Die Syntax `${var/pattern/replacement}` ersetzt das erste Vorkommen von `pattern` durch `replacement`. Die Ausgabe lautet hier:
    ```plaintext
    Bark meow
    ```
    Die Ersetzung ist case-sensitiv. Um sowohl `bark` als auch `Bark` zu erfassen, verwende ein Muster wie `${var/[Bb]ark/Bark}` oder normalisiere den String vor der Ersetzung.

    Um vom Anfang des Strings zu ersetzen, verwende `${var/#pattern/replacement}`.

    Um vom Ende des Strings zu ersetzen, verwende `${var/%pattern/replacement}`.

    Was gibt dieses Skript aus, wenn die Datei `cats.txt` EXISTIERT?

    ```bash
    if [ -e cats.txt]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Hast du das fehlende Leerzeichen vor der schließenden Klammer bemerkt?

    Bash ist hier ziemlich pingelig: Leerzeichen innerhalb von Klammerausdrücken sind zwingend erforderlich.

    Da das fehlende Leerzeichen dazu führt, dass der `[`-Befehl keine schließende `]` erkennt, gibt Bash eine Fehlermeldung aus, wertet den Test als fehlgeschlagen und fährt mit dem `else`-Zweig fort.

    Die korrekte Syntax lautet:
    ```bash
    if [ -e example.txt ]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
    Hinweis: Doppelte Klammern `[[ ]]` sind für Bedingungsprüfungen **empfohlen**. [Siehe BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="Stolperfallen"
  title="Befehlssubstitution"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    Welche Option führt den Befehl `ls -l` aus und gibt dessen Ausgabe zurück?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Die Syntax `$(ls -l)` führt den Befehl innerhalb der **Klammern** aus und ersetzt ihn durch die Ausgabe. Zum Beispiel:
    ```bash
    echo "Today is $(date +%F)"
    # => Today is 2029-12-31
    ```
    Die erste Option verwendet einfache Anführungszeichen `'`, **keine Backticks.** Das verhindert die Expansion, sodass `'$(date +%F)'` einfach die Zeichenkette `$(date +%F)` wörtlich ausgibt.

    Zwar werden Backticks (`` `ls -l` ``) zur Befehlsausführung noch unterstützt, sie gelten jedoch zunehmend als Anti-Pattern (in manchen Kontexten). Die meisten empfehlen `$(Befehl)` für bessere Lesbarkeit und Konsistenz über verschiedene Shells und Versionen hinweg.

    Geschweifte Klammern `${}` dienen der Variablenexpansion, nicht der Befehlssubstitution.

    Das Zeichen `%` wird nicht für die Befehlssubstitution verwendet.
  </div>
  </slot>
</Challenge>
