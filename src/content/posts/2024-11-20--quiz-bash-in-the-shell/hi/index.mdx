  <slot name='explanation'>
  <div className="explanation">
    Bash में `(( ))` सिंटैक्स पूर्णांक गणना (integer math) करता है।

    इसका उपयोग साधारण गणनाओं के लिए किया जा सकता है:
    ```bash
    ((result = 2 + 2))
    echo $result # => 4
    ```
    या फिर इसे कंडीशनल एक्सप्रेशन (conditional expressions) में भी इस्तेमाल कर सकते हैं:
    ```bash
    if (( 2 > 1 )); then
      echo "2 is greater than 1"
    fi
    ```
    चूंकि Bash मूल रूप से फ्लोटिंग-पॉइंट नंबर्स को सपोर्ट नहीं करता, इसलिए दशमलव गणनाओं के लिए [`bc`](https://www.gnu.org/software/bc/manual/html_mono/bc.html) या [`awk`](https://www.gnu.org/software/gawk/manual/gawk.html) जैसे टूल्स का सहारा लें।
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="गोटचाज़"
  title="कमांड सब्स्टीट्यूशन"
  options={[

    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    कौन सा विकल्प `ls -l` कमांड को निष्पादित करेगा और उसका आउटपुट वापस करेगा?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    `$(ls -l)` सिंटैक्स **कोष्ठक** के अंदर लिखे कमांड को निष्पादित करता है और आउटपुट को वहीं प्रतिस्थापित कर देता है। उदाहरण के लिए:
    ```bash
    echo "Today is $(date +%F)"
    # => Today is 2029-12-31
    ```
    पहला विकल्प सिंगल कोट्स `'` का उपयोग करता है, **बैकटिक्स का नहीं।** यह एक्सपेंशन को रोक देता है, इसलिए `'$(date +%F)'` बस लिटरल स्ट्रिंग `$(date +%F)` ही प्रिंट करेगा।

    कमांड चलाने के लिए बैकटिक्स (`` `ls -l` ``) का उपयोग तकनीकी रूप से अभी भी काम करता है, लेकिन अब इसे कई संदर्भों में एक एंटी-पैटर्न माना जाता है। बेहतर पठनीयता और अलग-अलग शेल व वर्ज़न के साथ स्थिरता के लिए `$(command)` का ही उपयोग करना ज़्यादा मानक है।

    कर्ली ब्रेस `${}` का इस्तेमाल वेरिएबल एक्सपेंशन के लिए होता है, कमांड सब्स्टीट्यूशन के लिए नहीं।

    `%` कैरेक्टर का कमांड सब्स्टीट्यूशन से कोई लेना-देना नहीं है।
  </div>
  </slot>
</Challenge>
