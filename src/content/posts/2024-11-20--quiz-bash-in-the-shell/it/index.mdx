    Come si definiscono le variabili in Bash?

    In Bash, le variabili vengono dichiarate senza spazi attorno al segno `=`. Per esempio:
    ```bash
        name=Alice
    ```
    Questo assegna il valore `"Alice"` alla variabile `name`.

    Nota: `$name` viene usato per **riferirsi** o leggere il valore di una variabile.

    Aggiungere spazi fa sì che la shell interpreti il comando come un programma da eseguire, cosa che non vuoi quando imposti una variabile.

    Inoltre, Bash è sensibile alle maiuscole/minuscole, quindi `name`, `NAME` e `Name` sono variabili diverse.

    Infine, le variabili non possono contenere spazi o trattini (`-`) nei loro nomi. Usa trattini bassi (`_`) o camelCase.
  </div>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={4}
  group="Sostituzione di Sottostringhe"
  title="Sostituisci Sottostringa"
  options={[
    {text: 'meow meow'},
    {text: 'Meow meow'},
    {text: 'Bark meow', isAnswer: true},
    {text: 'Bark bark'},
    {text: 'Errore'},
  ]}
>
  <slot name="question">
  <div className="question">
    Cosa sta succedendo qui?
    ```bash
        str="Bark bark"
        echo ${str/bark/meow}
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La sintassi `${var/pattern/replacement}` sostituisce la prima occorrenza di `pattern` con `replacement`. Qui, l'output è:
    ```plaintext
        Bark meow
    ```
    È sensibile alle maiuscole/minuscole. Per gestire sia `bark` che `Bark`, usa un pattern come `${var/[Bb]ark/Bark}` o normalizza la stringa prima della sostituzione.

    Per sostituire tutte le occorrenze, usa `${var//pattern/replacement}`.

    Per sostituire dall'inizio della stringa, usa `${var/#pattern/replacement}`.

    Per sostituire dalla fine della stringa, usa `${var/%pattern/replacement}`.
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={6}
  group="Condizionali"
  title="If-Else di base"
  options={[
    {text: 'Il file esiste'},
    {text: 'Il file non esiste, dopo un messaggio diagnostico del test', isAnswer: true},
    {text: 'Solo errore'},
    {text: 'Doppie parentesi mancanti'},
    {text: 'Niente'},
  ]}
>
  <slot name="question">
  <div className="question">
    Cosa produce questo script se il file `cats.txt` ESISTE?
    ```bash
        if [ -e cats.txt]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Hai notato lo spazio mancante prima della parentesi di chiusura?

    Bash è molto esigente qui: gli spazi sono obbligatori all'interno delle espressioni tra parentesi.

    Poiché lo spazio mancante fa sì che il comando `[` non veda la `]` di chiusura, Bash stampa un messaggio diagnostico, considera il test fallito e continua con il ramo `else`.

    La sintassi corretta è:
    ```bash
        if [ -e example.txt ]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
    Nota: Le doppie parentesi `[[ ]]` sono **raccomandate** per le espressioni condizionali. [Vedi BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)
  </div>
  </slot>
</Challenge>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={14}
  group="Insidie"
  title="Sostituzione di comando"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    Quale opzione eseguirà il comando `ls -l` e restituirà l'output?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    La sintassi `$(ls -l)` esegue il comando all'interno delle **parentesi** e sostituisce l'output. Per esempio:
    ```bash
        echo "Today is $(date +%F)"
        # => Today is 2029-12-31
    ```
    La prima opzione usa gli apici singoli `'`, **non i backtick.** Questo impedisce l'espansione, quindi `'$(date +%F)'` stampa semplicemente la stringa letterale `$(date +%F)`.

    Sebbene sia ancora supportato l'uso dei backtick (`` `ls -l` ``) per l'esecuzione di comandi, recentemente è diventato un po' un anti-pattern (in alcuni contesti). La maggior parte consiglia di usare `$(comando)` per una migliore leggibilità e coerenza tra diverse shell e versioni.

    Le parentesi graffe `${}` sono usate per l'espansione delle variabili, non per la sostituzione di comando.

    Il carattere `%` non viene usato per la sostituzione di comando.
  </div>
  </slot>
