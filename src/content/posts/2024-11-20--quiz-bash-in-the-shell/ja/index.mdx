    `${var/pattern/replacement}` 構文は、`pattern` の最初の出現箇所を `replacement` に置き換えます。今回の出力は次の通りです：

    大文字・小文字は厳密に区別されます。`bark` と `Bark` の両方を処理したい場合は、`${var/[Bb]ark/Bark}` のようなパターンを使うか、置換前に文字列を正規化してください。

    文字列の先頭から置換するには、`${var/#pattern/replacement}` を使います。

    文字列の末尾から置換するには、`${var/%pattern/replacement}` を使います。

  <div className="question">
    ファイル `cats.txt` が存在する場合、このスクリプトの出力はどうなるか？
    ```bash
    if [ -e cats.txt]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    閉じ括弧の前のスペースが抜けているのに気づいただろうか？

    Bashはこの点に非常にうるさい。括弧式の内側にはスペースが必須だ。

    スペースが抜けているため、`[` コマンドは閉じ括弧 `]` を認識できず、Bashは診断メッセージを出力してテストを失敗と判定し、`else` ブランチへ進む。

    正しい構文は以下の通りだ：
    ```bash
    if [ -e example.txt ]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
    注：条件式には二重括弧 `[[ ]]` の使用が**推奨される**。[BashFAQを参照。](https://mywiki.wooledge.org/BashFAQ/031)
  </div>

<Challenge

  index={14}
  group="落とし穴"
  title="コマンド置換"
  options={[
    {text: '\'ls -l\''},
    {text: '% ls -l'},
    {text: '$ ls -l'},
    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    `ls -l` コマンドを実行し、その出力を返すのはどれ？
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    `$(ls -l)` 構文は、**括弧**内のコマンドを実行し、出力をその結果に置き換えます。例：
    ```bash
    echo "Today is $(date +%F)"
    # => Today is 2029-12-31
    ```
    1番目のオプションはシングルクォート `'` を使用しており、**バッククォートではありません。** これにより展開が抑制されるため、`'$(date +%F)'` は単にリテラル文字列 `$(date +%F)` を出力するだけです。

    コマンド実行にバッククォート（`` `ls -l` ``）を使用することは依然としてサポートされていますが、最近では（一部の文脈において）アンチパターンと見なされることが多くなりました。可読性の向上や、異なるシェル・バージョン間での一貫性を考慮し、多くの開発者が `$(command)` の使用を推奨しています。

    波括弧 `${}` は変数展開に使用されるものであり、コマンド置換ではありません。

    `%` 文字はコマンド置換には使用されません。
  </div>
  </slot>
</Challenge>
