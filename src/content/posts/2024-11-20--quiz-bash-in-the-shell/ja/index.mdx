>
  <slot name="question">
  <div className="question">
    ファイル `cats.txt` が存在する場合、このスクリプトは何を出力しますか？
    ```bash
        if [ -e cats.txt]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    閉じ括弧の前のスペースが抜けていることに気づきましたか？

    Bash はここでかなり敏感です：角括弧式の内部にはスペースが必要です。

    スペースが欠けていると `[` コマンドが閉じ `]` を見つけられず、Bash は診断メッセージを出し、テストを失敗とみなして `else` ブランチへ進みます。

    正しい構文は次の通りです:
    ```bash
        if [ -e example.txt ]; then
          echo "File exists"
        else
          echo "File does not exist"
        fi
    ```
    注: 二重角括弧 `[[ ]]` は条件式に **推奨** されています。 [BashFAQ を参照してください。](https://mywiki.wooledge.org/BashFAQ/031)
  </div>
  </slot>
</Challenge>

<Challenge

  index={14}

  options={[
    {text: '\'ls -l\''},

    {text: '$(ls -l)', isAnswer: true},
    {text: '${ls -l}'},
  ]}
>
  <slot name="question">
  <div className="question">
    どの選択肢がコマンド `ls -l` を実行し、出力を返しますか？
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    `$(ls -l)` 構文は **丸括弧** の中のコマンドを実行し、出力を置換します。例として:
    ```bash
        echo "Today is $(date +%F)"
        # => Today is 2029-12-31
    ```
    1番目の選択肢はシングルクォート `'` を使用しており、**バッククォートではありません**。これにより展開が抑止されるので、`'$(date +%F)'` は文字列 `$(date +%F)` をそのまま出力します。

    バッククォート（`` `ls -l` ``）でもコマンド実行はサポートされていますが、最近では（特定の文脈で）アンチパターンとみなされることが増えています。多くは可読性とシェル・バージョン間の一貫性のために `$(command)` の使用を推奨します。

    中括弧 `${}` は変数展開に使われ、コマンド置換には使用しません。

    `%` 文字はコマンド置換に使われません。
  </div>
  </slot>
</Challenge>
