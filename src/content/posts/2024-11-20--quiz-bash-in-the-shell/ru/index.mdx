    ```bash
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Заметили пропущенный пробел перед закрывающей скобкой?

    Bash здесь довольно придирчив: пробелы обязательны внутри выражений в квадратных скобках.

    Из-за пропущенного пробела команда `[` не видит закрывающую `]`, Bash выводит диагностическое сообщение, считает тест неудачным и переходит к ветке `else`.

    Правильный синтаксис:
    ```bash
    if [ -e example.txt ]; then
      echo "File exists"
    else
      echo "File does not exist"
    fi
    ```
    Примечание: Двойные скобки `[[ ]]` **рекомендуются** для условных выражений. [См. BashFAQ.](https://mywiki.wooledge.org/BashFAQ/031)
  </div>
  </slot>
</Challenge>

<Challenge

  ]}

  <slot name="question">
  <div className="question">
    Какой вариант выполнит команду `ls -l` и вернёт вывод?
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    Синтаксис `$(ls -l)` выполняет команду внутри **круглых скобок** и подставляет вывод. Например:
    ```bash
    echo "Today is $(date +%F)"
    # => Today is 2029-12-31
    ```
    Первый вариант использует одинарные кавычки `'`, **а не обратные кавычки.** Это предотвращает раскрытие, поэтому `'$(date +%F)'` просто выведет буквальную строку `$(date +%F)`.

    Хотя использование обратных кавычек (`` `ls -l` ``) для выполнения команд всё ещё поддерживается, в последнее время это стало своего рода антипаттерном (в некоторых контекстах). Большинство рекомендует использовать `$(command)` для лучшей читаемости и совместимости с разными оболочками и версиями.

    Фигурные скобки `${}` используются для раскрытия переменных, а не для подстановки команд.

    Символ `%` не используется для подстановки команд.
