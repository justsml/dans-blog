  <slot name='explanation'>
  <div className="explanation">
    Debido a errores de precisión en punto flotante, `0.1 + 0.2` no es exactamente igual a `0.3`. Por la forma en que los números de punto flotante se almacenan en memoria, el resultado es `0.30000000000000004`. El estándar IEEE 754 para manejar aritmética de punto flotante es el culpable: no puede representar algunos números con exactitud. Este es un problema común en todos los lenguajes de programación. Tarde o temprano te toparás con un decimal periódico y, sin importar el lenguaje, la computadora simplemente debe dejar de perseguir dígitos que se repiten infinitamente.

    Algunos lenguajes como Python y Java cuentan con `Decimal` o `BigDecimal` para manejar esto, pero no viene integrado en JavaScript. Puedes usar bibliotecas como `big.js` o `decimal.js` para resolver estos casos.

    (Nota: Algunos lenguajes están diseñados para manejar fracciones, números imaginarios, etc., a un nivel lógico superior, preservando expresiones literales. Pero, a nivel de hardware, siguen teniendo que lidiar con los mismos problemas de precisión de punto flotante.)
  </div>
  </slot>

<Challenge
  client:visible={{rootMargin: "150px"}}
  index={5}
  group="Comparación de tipos"
  title="Comparación de igualdad entre `parseInt` y `parseFloat`"
  options={[
    {text: 'true', isAnswer: true},
    {text: 'false'},
    {text: 'NaN'},
  ]}
>
  <slot name="question">
  <div className="question">
    ```jsx
    parseInt("42") === parseFloat("42")
    ```
  </div>
  </slot>
  <slot name='explanation'>
  <div className="explanation">
    En JavaScript, tanto `parseInt` como `parseFloat` interpretan la cadena `"42"` como el número `42`. Por lo tanto, la comparación `parseInt("42") === parseFloat("42")` se evalúa como `true`. Aunque `parseInt` detiene el análisis en el primer carácter que no es un dígito, `parseFloat` continúa hasta encontrar un carácter que no forme parte de un número de punto flotante. Sin embargo, dado que `"42"` no contiene puntos decimales ni otros caracteres no numéricos, ambas funciones devuelven exactamente el mismo valor.
  </div>
  </slot>
</Challenge>

  <slot name='explanation'>
  <div className="explanation">
    [`BigInt`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/BigInt) es un tipo diferente a `number`, por lo que `parseInt("42")` (un número normal) no es estrictamente igual a `BigInt("42")`. Para compararlos, debes convertir ambos al mismo tipo: `BigInt(parseInt("42")) === BigInt("42")`.
  </div>
  </slot>
