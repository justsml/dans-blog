## Requisitos del Proyecto

Antes de empezar a actualizar, establecí un conjunto de requisitos:

Ya que mi blog recibe una cantidad variable de visitas diarias, sentí que un sitio web estático pre-generado daría el rendimiento que deseaba sin la complejidad adicional. También necesitaba mantener contenido existente y características, incluyendo:

* Resaltado de código
* Comentarios
* Búsqueda del sitio
* Componentes React pre-existent: Quiz UI, Gist embeds
* Formulario de contacto
* Imágenes responsivas
* Tiempo de carga sub-1 segundo
* Compatible con navegadores desde 2018
* Implementación automática + PR- based

## Escoger mi Tecnología Correcta

En el transcurso de los años he trabajado con muchas herramientas de sitios web estáticos, desde Jekyll hasta Hugo y Slate. También he trabajado con muchos marcos de front-end: Ember, Knockout, Angular, Vue y por supuesto React.

Así que, tengo precisamente demasiadas opciones, lo que finalmente reduje a **Remix**, **Next.js** y **Astro**,

Podría escribir una serie entera de blog sobre mi proceso de evaluación, pero resumiré aquí:

< p class="breakout">Escogí [Astro](https://astro.build) porque puedo hacer cosas significativas muy rápidamente.</p>

Su diseño API es refrescadormente simple. Es un equilibrio perfecto entre flexibilidad y opiniones de diseño buenos.

Fue un poco reconfortante que Astro careciera de cualquier agenda o influencia nube obvia.

Astro no fue la única tecnología que utilicé, aquí hay una lista completa del stack:

* [Astro](https://astro.build): Un generador estático moderno.
* [ShadcnUI](https://ui.shadcn.com): Una colección de componentes reutilizables.
* [Tailwind CSS](https://tailwindcss.com): Un marco de CSS utilitario.
* [MDX](https://mdxjs.com): Contenido Markdown + componentes inline.
* [Pagefind](https://pagefind.app): Búsqueda estática y offline del sitio web. No necesito Algolia!
* [Utterances](https://utteranc.es): Sistema de comentarios basado en problemas de GitHub.
* [Netlify](https://www.netlify.com): Implementaciones automatizadas, formulario de contacto con captcha.

## Astro: Curva de Aprendizaje y Características Clave

< p class="breakout quote">Astro se convirtió rápidamente en el cimiento de mi actualización.</p>

Aquí hay algunas características clave que encontré especialmente útiles:

* Archivos `.astro`: Al principio, los componentes Astro pueden parecerse a componentes JSX de React, pero son mucho diferentes y sirven para un conjunto diferente de objetivos. (Ver tabla de comparación abajo.)
* Compiladores Golang: Astro se ejecuta con sus propios herramientas de compilación y Vite; solo funciona. Maneja ESM/CJS, TypeScript, empaquetado de código, estilos, imágenes, etc.
* No hay influencia ni agenda nube. (*Cough* Next.js, OpenNext)
* Modos de renderizado estático vs. híbrido: Astro proporciona flexibilidad para objetivos cloud y plataformas en la nube que van desde AWS hasta GCP, Firebase, Netlify, Vercel, Cloudflare Pages, Azure, Fly.io y muchos otros.
* Colecciones de contenido: El API `getCollection` simplifica el trabajo con archivos de contenido como fuente de datos.
* Ruteo basado en archivos: El sistema de ruteo basado en archivos de Astro, combinado con `getStaticPaths`, hace que la generación de páginas sea un placer.

## CSS Moderno: Wow

Viniendo al desarrollo frontend, me alegró ver los avances en el CSS nativo:

* Variables CSS: Disponibles durante algún tiempo y relativamente estables en navegadores desde 202\*.
* Nesting: Finalmente en especie, y sin la sintaxis incómoda anterior. Ahora es similar a Less o SCSS.
* Selectores nuevos como `:is()`, `:where()` y `:has()` ofrecen más precisión al objetivo de los elementos.
* Unidades modernas como `ch`, `vw` y funciones como `clamp()` proporcionan un control mejorado sobre diseños y tipografía.
* Espaciado natural con atributos `-inline` y `-block`. Establecer espacio o margen en cualquier eje horizontal o vertical. En lugar de `margin: 0 1rem 0 1rem` → `margin-inline: 1rem`.
* Diseños avanzados: Reaprendiendo CSS Grid. Wow, hay mucho mierda allí. Puedes obtenerse con la ayuda de figuras como [Kevin Powell's video: Learn CSS Grid the easy way](https://www.youtube.com/watch?v=rg7Fvvl3taU), [Responsive without media queries](https://ardilamorin.com/responsive-no-media-queries/), [Ten modern layouts in one line of CSS](https://web.dev/articles/one-line-layouts).

## Búsqueda: Pagefind

Implementar una **búsqueda del sitio** sin servicios de terceros o hosteo de bases de datos parecía un desafío divertido. Después de todo, no como 10,000 posts para indexar (aún).

Mientras navegaba por las [integraciones comunitarias de Astro](https://astro.build/integrations/?search=find) encontré una herramienta fantástica que deseaba haber conocido antes: [Pagefind](https://pagefind.app).

< p class="breakout quote">Pocos herramientas resuelven cualquier problema tan bien como Pagefind resuelve búsqueda local del sitio.</p>

La simplicidad de implementar Pagefind es un placer. Puede integrarse con cualquier contenido estático y elegir si desea una UI predeterminada o puede construir algo personalizado si lo prefiere.

## Comentarios: Utterances

Desafortunadamente, tuve que decir adiós a Disqus y los comentarios que había acumulado durante muchos años.

Quería un control mejorado/visibilidad en los scripts de terceros en mi sitio web.
También necesitaba algo simple y mantenible.
Esto me llevó a elegir el servicio fantástico [Utterances](https://utteranc.es/) con su sistema de comentarios basado en problemas de GitHub. Es fácil de configurar y gratuito.

## Tailwind: Regrets

Hay solo un pedazo de tecnología que estoy cada vez más arrepentido de utilizar: Tailwind.
Sobre tiempo, puedo sentir el costo diferencial entre escribir vs. mantener. Tailwind es tan rápido para escribir, pero una vez sea lo suficientemente complejo, puede convertirse en tedioso leer y extender.

## Conclusión

Actualizando mi vieja sitio web de Gatsby v1 a un stack moderno construido alrededor de Astro fue una experiencia divertida. 10/10 recomiendo.
Si estás considerando actualizar un sitio web antiguo o construir un nuevo sitio web estático (o híbrido), te recomiendo verificar Astro. La curva de aprendizaje puede ser empinada en veces, pero los beneficios en términos de rendimiento, experiencia del desarrollador y futuro-protección de tu proyecto valen la pena el esfuerzo.
