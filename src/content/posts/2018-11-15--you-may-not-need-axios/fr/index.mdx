### Comparaison des fonctionnalités

|                                                 | fetch    | axios    | request |
|-------------------------------------------------|:--------:|:--------:|:-------:|

| Intercepter la requête et la réponse                  |✅        |✅         |✅       |
| Transformer les données de la requête et de la réponse |✅        |✅         |✅       |
| Annuler les requêtes                                 |✅        |✅         |❌       |
| Transformations automatiques pour les données JSON    |aides manuelles |✅         |✅       |
| Prise en charge côté client pour la protection contre XSRF |✅        |✅         |✅       |

| Progression                                    |✅        |✅         |✅       |
| Streaming                                       |✅        |✅         |✅       |

<br /><br />

En commençant cet article (fin 2018, mis à jour en 2024), je pensais que je terminerais avec un tableau de cases à cocher mélangées. Il y a sûrement des _cas d'utilisation_ spéciaux qui justifiaient [`axios`](https://www.npmjs.com/package/axios), [`request`](https://www.npmjs.com/package/request), [`r2`](https://www.npmjs.com/package/r2), [`superagent`](https://www.npmjs.com/package/superagent), [`got`](https://www.npmjs.com/package/got), etc.

Eh bien, il s'avère que **j'ai surestimé le besoin de bibliothèques HTTP tierces**.

Bien que j'utilise `fetch` depuis plusieurs années (y compris pour des tâches non triviales : téléchargements de fichiers et gestion des erreurs/répétitions), j'avais encore des idées fausses sur les capacités et les limites de `fetch`.

Le `fetch` natif n'analyse pas automatiquement les réponses JSON ni ne sérialise les corps de requête JSON.

Vous appelez `response.json()` au retour et `JSON.stringify()` à l'envoi. Axios conserve cet avantage ergonomique ; l'argument en faveur de `fetch` est qu'un petit utilitaire comble souvent l'écart.

Bon, voyons ce que `fetch` peut faire...

### Compatibilité

Depuis 2022, l'API `fetch` est [largement prise en charge](https://caniuse.com/#feat=fetch) dans tous les navigateurs modernes et dans les versions plus récentes de NodeJS v18+.

Si vous devez prendre en charge IE, vous pouvez [polyfill fetch](https://github.com/github/fetch#browser-support) avec le package `github/fetch` (maintenu par une équipe géniale chez GitHub). Il est possible de remonter jusqu'à [IE8](https://github.com/camsong/fetch-ie8) - _Les résultats peuvent varier_.

Les versions antérieures de NodeJS peuvent tirer parti de l'API `fetch` avec le package [`node-fetch`](https://www.npmjs.com/package/node-fetch) :

```sh
npm install node-fetch
```

_Après polyfill+node-fetch : compatible à 99,99 %_ ✅
