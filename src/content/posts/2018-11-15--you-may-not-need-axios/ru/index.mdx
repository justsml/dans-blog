Когда я начинал эту статью (конец 2018 года, обновлено в 2024), я предполагал, что в итоге получу таблицу с перемешанными галочками.

Конечно, есть особые _Use Cases_, которые оправдывали [`axios`](https://www.npmjs.com/package/axios), [`request`](https://www.npmjs.com/package/request), [`r2`](https://www.npmjs.com/package/r2), [`superagent`](https://www.npmjs.com/package/superagent), [`got`](https://www.npmjs.com/package/got) и т.д.

Ну, как оказалось, **я переоценил необходимость в сторонних http-библиотеках.**

Несмотря на использование `fetch` в течение нескольких лет (в том числе для нетривиальных задач: загрузка файлов и поддержка ошибок/повторных попыток), у меня всё ещё были заблуждения относительно его возможностей и ограничений.

Встроенный `fetch` не выполняет автоматический парсинг JSON-ответов и не сериализует JSON-тела запросов.

Вы вызываете `response.json()` при получении ответа и `JSON.stringify()` при отправке запроса. Axios всё ещё выигрывает в этом аспекте эргономики; аргумент в пользу `fetch` в том, что небольшой хелпер часто закрывает этот пробел.

Что ж, давайте посмотрим, на что способен `fetch`...

### Совместимость

По состоянию на 2022 год, API `fetch` [широко поддерживается](https://caniuse.com/#feat=fetch) во всех современных браузерах и в более новых версиях NodeJS v18+.

Если вам необходимо поддерживать IE, вы можете [использовать полифилл fetch](https://github.com/github/fetch#browser-support) с помощью пакета `github/fetch` (поддерживается отличной командой из GitHub).

Можно уйти вплоть до [IE8](https://github.com/camsong/fetch-ie8) — _результат может отличаться_.

В более ранних версиях Node.js можно использовать API `fetch` с помощью пакета [`node-fetch`](https://www.npmjs.com/package/node-fetch):

```sh
npm install node-fetch
```

_После polyfill+node-fetch: совместимость 99.99%_ ✅
